# Project Architect

Use this skill when Porter needs to create or refine a project.

Core rules:
- Start from the user objective, not from internal implementation structure.
- Prefer one clear project over multiple thin projects.
- Reject project creation if the work is better handled as a single worker task.
- Produce a short project brief with name, objective, mode, success bar, and likely worker lanes.

Output format:
- `name`
- `objective`
- `mode`: `manual` or `autonomous`
- `why_project`
- `initial_workers`
- `risks`
