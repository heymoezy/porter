**Routes to implement:**

1. **GET /** — List templates
   - Query params: `category?: string`, `tag?: string`, `q?: string`, `include_internal?: string`, `limit?: string`, `offset?: string`
   - Default: exclude internal templates (`is_internal = 0`) unless `include_internal=true` is passed
   - Filter by category: `WHERE category = ?`
   - Filter by tag: Use `json_each(tags)` to check if tag value exists in the JSON array:
     ```sql
     WHERE id IN (SELECT at.id FROM agent_templates at, json_each(at.tags) jt WHERE jt.value = ?)
     ```
   - Search by q: `WHERE (name LIKE ? OR description LIKE ?)`
   - Parse JSON fields (tags, skills, tools, required_backends, required_tools) before returning
   - Return `{ templates: [...], total: N }`

2. **GET /:id** — Get single template
   - Return full template with all fields parsed from JSON
   - 404 if not found: `err('TEMPLATE_NOT_FOUND', 'Template not found')`

3. **POST /:id/instantiate** — Create agent from template
   - Parse body with `instantiateSchema`
   - Fetch template from DB: `SELECT * FROM agent_templates WHERE id = ?`
   - 404 if template not found
   - **Strict validation** (locked decision — 422 with specific items):
     - Parse `required_backends` JSON array from template
     - For each backend name, resolve to URL via `getBackendUrl()`, then `probeBackend(url)`
     - Parse `required_tools` JSON array from template
     - For each tool, check `workspace_connections` table: `SELECT id, status FROM workspace_connections WHERE provider = ? AND status = 'connected'`
     - Collect missing backends and tools
     - If any missing: return `reply.code(422).send(err('MISSING_DEPENDENCIES', JSON.stringify({ missing_backends: [...], missing_tools: [...] })))`
   - **Create persona row:**
     - ID: `'agent_' + crypto.randomUUID().replace(/-/g, '').slice(0, 12)`
     - name: body.name override or template.name
     - role: body.role override or template.category
     - config JSON blob: `{ description: body.description || template.description, skills: JSON.parse(template.skills), tools: JSON.parse(template.tools), template_id: template.id, project_id: body.project_id || null }`
     - templateId: template.id (the new column from Plan 01)
     - owner: `request.sessionUser!.username`
     - status: 'idle'
     - createdAt: `new Date().toISOString()`
   - **Write .md files to personas directory:**
     ```typescript
     const personaDir = path.join(process.env.HOME!, 'documents/porter/personas', agentId);
     await fs.mkdir(personaDir, { recursive: true });
     ```

   - **Rollback on file write failure:** If any fs.writeFile throws, DELETE the persona row and re-throw (no partial agents — locked decision)
   - **Auto-assign to project** if project_id provided: check project exists, set config.project_id
   - Return 201 with agent formatted using a local formatAgent (copy the relevant parts from agents.ts or import if exported)
   - Fetch the created persona row back and format it

```typescript
function insertTemplate(t: {
  id: string; name: string; category: string; description: string;
  tags: string[]; skills: string[]; tools: string[];
  required_backends: string[]; required_tools: string[];
  system_prompt: string; soul_text: string; role_card_text: string;
  identity_text: string; skills_text: string;
  is_internal: number; sort_order: number;
}): void {
  sqlite.prepare(`
    INSERT OR IGNORE INTO agent_templates
      (id, name, category, description, tags, skills, tools, required_backends, required_tools,
       system_prompt, soul_text, role_card_text, identity_text, skills_text, is_internal, sort_order)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  `).run(
    t.id, t.name, t.category, t.description,
    JSON.stringify(t.tags), JSON.stringify(t.skills), JSON.stringify(t.tools),
    JSON.stringify(t.required_backends), JSON.stringify(t.required_tools),
    t.system_prompt, t.soul_text, t.role_card_text, t.identity_text, t.skills_text,
    t.is_internal, t.sort_order,
  );
}
```
