Create `backend/src/routes/v1/feedback.ts`:

```typescript
// POST /api/v1/feedback/:dispatchId
// Body: { event_type: 'positive' | 'negative', note?: string }
// Auth: requireAuth (same as chat endpoints)
```

Implementation:
1. Import pg pool from the same pattern as chat.ts (fastify decoration or import)
2. Import { v4 as uuidv4 } from 'uuid'
3. Register route with `preHandler: [fastify.requireAuth]`
4. Validate event_type is one of: 'positive', 'negative', 'correction', 'retry', 'abandon', 'success'
5. Query bridge_dispatch_log for the dispatch_id:
   ```sql
   SELECT agent_id, skills_used FROM bridge_dispatch_log WHERE id = $1
   ```
6. If no row found or no agent_id, return 404 with `{ ok: false, error: { code: 'NOT_FOUND', message: 'Dispatch not found' } }`
7. Parse skills_used JSONB — read `.selected` array only (NOT candidates)
8. If selected is empty or null, return `{ ok: true, data: { created: 0 } }` (no skills to attribute)
9. For each selected skill, INSERT into skill_feedback_events:
   ```sql
   INSERT INTO skill_feedback_events (id, persona_id, skill_id, dispatch_id, event_type, note)
   VALUES ($1, $2, $3, $4, $5, $6)
   ```
10. After all inserts, batch-update persona_skills counters for all selected skills at once:
    - For 'positive' event_type: increment positive_feedback_count
    - For 'negative' event_type: increment negative_feedback_count
    - For all types: set times_completed = times_completed + 1 (acknowledges the user saw the result)
    - Recompute effectiveness_score:
      ```sql
      UPDATE persona_skills
      SET positive_feedback_count = CASE WHEN $3 = 'positive' THEN COALESCE(positive_feedback_count, 0) + 1 ELSE COALESCE(positive_feedback_count, 0) END,
          negative_feedback_count = CASE WHEN $3 = 'negative' THEN COALESCE(negative_feedback_count, 0) + 1 ELSE COALESCE(negative_feedback_count, 0) END,
          times_completed = COALESCE(times_completed, 0) + 1,
          effectiveness_score = CASE
            WHEN (COALESCE(positive_feedback_count, 0) + CASE WHEN $3 = 'positive' THEN 1 ELSE 0 END
                + COALESCE(negative_feedback_count, 0) + CASE WHEN $3 = 'negative' THEN 1 ELSE 0 END) > 0
            THEN (COALESCE(positive_feedback_count, 0) + CASE WHEN $3 = 'positive' THEN 1 ELSE 0 END)::float
                 / (COALESCE(positive_feedback_count, 0) + CASE WHEN $3 = 'positive' THEN 1 ELSE 0 END
                  + COALESCE(negative_feedback_count, 0) + CASE WHEN $3 = 'negative' THEN 1 ELSE 0 END)
            ELSE NULL
          END
      WHERE persona_id = $1
        AND COALESCE(skill_id, skill_name) = ANY($2)
      ```
    Parameters: [personaId, skillIds, eventType]
11. Return `{ ok: true, data: { created: selectedSkills.length } }`

Register the route in `backend/src/routes/v1/index.ts`:
- Import feedbackRoutes from './feedback.js'
- Register: `fastify.register(feedbackRoutes, { prefix: '/api/v1/feedback' })`

Use the `ok()` and `err()` envelope pattern if available in Brain v1, or use raw JSON responses matching the existing chat.ts pattern (check what chat.ts uses for consistency).

3. **Add feedback submission function** (inside the ChatPanel component, before the return):
   ```typescript
   async function sendFeedback(msg: ChatMessage, eventType: 'positive' | 'negative') {
     if (!msg.dispatchId || msg.feedbackSent) return;
     try {
       await fetch(`/api/v1/feedback/${msg.dispatchId}`, {
         method: 'POST',
         headers: { 'Content-Type': 'application/json' },
         credentials: 'include',
         body: JSON.stringify({ event_type: eventType }),
       });
       setMessages(prev => prev.map(m => m.id === msg.id ? { ...m, feedbackSent: eventType } : m));
     } catch (e) {
       console.error('Feedback submission failed:', e);
     }
   }
   ```

4. **Render thumbs-up/down buttons** on assistant messages. In the message rendering section (where messages are mapped), after the message content div for assistant messages, add:
   ```tsx
   {msg.role === "assistant" && msg.dispatchId && (
     <div className="flex items-center gap-1 mt-1">
       <button
         onClick={() => sendFeedback(msg, 'positive')}
         disabled={!!msg.feedbackSent}
         className={`p-0.5 rounded transition-colors ${
           msg.feedbackSent === 'positive' ? 'text-green-500' :
           msg.feedbackSent ? 'text-text3 opacity-30' :
           'text-text3 hover:text-green-500'
         }`}
         title="Good response"
       >
         <ThumbsUp size={14} />
       </button>
       <button
         onClick={() => sendFeedback(msg, 'negative')}
         disabled={!!msg.feedbackSent}
         className={`p-0.5 rounded transition-colors ${
           msg.feedbackSent === 'negative' ? 'text-red-500' :
           msg.feedbackSent ? 'text-text3 opacity-30' :
           'text-text3 hover:text-red-500'
         }`}
         title="Poor response"
       >
         <ThumbsDown size={14} />
       </button>
     </div>
   )}
   ```

5. **Import ThumbsUp, ThumbsDown** from lucide-react at the top of the file (add to existing lucide imports).

   - After done event: thumbs shown, clickable (dispatchId populated)
   - After feedback: clicked thumb highlighted (green/red), both disabled (feedbackSent set)
   - No dispatchId (no skills used): no thumbs shown (graceful absence)

**Proxy path resolution:** The admin frontend uses relative paths like `/api/v1/feedback/...`. In dev mode, `vite.config.ts` proxies ALL `/api` requests to Brain at `http://127.0.0.1:3001`. In production, the admin backend serves the frontend and has its own route handlers — but `/api/v1/feedback/` is a Brain route. The same mechanism that makes `/api/v1/chat/sessions` work in production (reverse proxy or direct Brain access) will handle `/api/v1/feedback/`. Use the relative path `/api/v1/feedback/${msg.dispatchId}` — do NOT hardcode `http://127.0.0.1:3001`.
