<objective>
Create the feedback telemetry database schema and surface dispatch_id through the SSE pipeline so chat clients can link user feedback to specific dispatches.

Purpose: Without the skill_feedback_events table and counter columns on persona_skills, no feedback can be stored. Without dispatch_id in the SSE done event, the client cannot link thumbs-up/down to the dispatch that produced the response. This plan builds the infrastructure that Plan 02 and 03 consume.

Output: Migration file, updated Drizzle schema, dispatch_id in SSE done events, times_selected counter increments on dispatch.
</objective>

**routing-engine.ts changes:**

1. In the `selectStreamWithFallback` method (or the wrapped stream generator at line ~627-655), change the fire-and-forget `logDispatch` to capture the returned dispatch ID. The generator is `async function*` so it can await:
   - Change: `self.logDispatch(decision, ctx, result).catch(() => {});`
   - To: `const dispatchId = await self.logDispatch(decision, ctx, result).catch(() => null as string | null);`
   - After the logDispatch call, yield a special metadata token that encodes the dispatch_id. Use a convention like yielding a JSON string prefixed with `__DISPATCH_META__` that the caller can detect:
     ```
     if (dispatchId) yield `__DISPATCH_META__${JSON.stringify({ dispatch_id: dispatchId })}`;
     ```
   - IMPORTANT: This yield happens AFTER the stream is fully consumed (after the for-await loop), so the caller sees it as the last token.

2. In `logDispatch`, after the INSERT succeeds, add a `times_selected` increment for each selected skill. After the existing RPG XP award block, add:
   ```typescript
   // Phase 34: Increment times_selected counter on persona_skills for each selected skill
   if (ctx.skillsUsed?.selected?.length && ctx.agentId) {
     const skillIds = ctx.skillsUsed.selected.map(s => s.skillId);
     await pool.query(
       `UPDATE persona_skills
        SET times_selected = COALESCE(times_selected, 0) + 1,
            last_used_at = EXTRACT(EPOCH FROM NOW())
        WHERE persona_id = $1
          AND COALESCE(skill_id, skill_name) = ANY($2)`,
       [ctx.agentId, skillIds]
     );
   }
   ```
   This is inside the async IIFE that already has try/catch, so failures are non-fatal.

**chat.ts changes:**

1. In the streaming loop (line ~318-323), detect and strip the `__DISPATCH_META__` token before sending to client:
   ```typescript
   let capturedDispatchId: string | null = null;
   // Inside the for-await loop:
   if (token.startsWith('__DISPATCH_META__')) {
     try {
       const meta = JSON.parse(token.slice('__DISPATCH_META__'.length));
       capturedDispatchId = meta.dispatch_id;
   ```

IMPORTANT: The `__DISPATCH_META__` approach avoids changing the StreamBackend interface or threading a ref object through multiple layers. It's a simple convention that only chat.ts needs to understand.
