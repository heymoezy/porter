**Route implementations:**

**GET /** — List all skills with agent assignment counts:
```sql
SELECT s.*,
  (SELECT COUNT(*) FROM template_skills ts WHERE ts.skill_id = s.id) AS template_count,
  (SELECT COUNT(*) FROM persona_skills ps WHERE ps.skill_name = s.id AND ps.enabled = 1) AS agent_count
FROM skills s
ORDER BY s.sort_order, s.name
```
Return: `ok({ skills: rows, total: rows.length })` — each row includes all skill fields plus template_count and agent_count.

**GET /categories** — Category summary:
```sql
SELECT category, COUNT(*) as count FROM skills GROUP BY category ORDER BY category
```
Return: `ok({ categories: rows })`

**GET /featured** — Featured skills:
```sql
SELECT * FROM skills WHERE featured = 1 ORDER BY featured_order, sort_order, name
```
Return: `ok({ skills: rows })`

**GET /:id** — Single skill with full agent details:
```sql
-- Get skill
SELECT * FROM skills WHERE id = $1
-- Get assigned templates
SELECT ts.template_id, at.name, at.category FROM template_skills ts JOIN agent_templates at ON at.id = ts.template_id WHERE ts.skill_id = $1 ORDER BY at.name
-- Get assigned agents (personas)
SELECT ps.persona_id, p.name, p.role, ps.enabled FROM persona_skills ps LEFT JOIN personas p ON p.id = ps.persona_id WHERE ps.skill_name = $1 ORDER BY p.name
```
Return: `ok({ skill: { ...skillRow, templates, agents } })` or 404 `err('NOT_FOUND', 'Skill not found')`

**POST /** — Create skill:
- Validate body with createSkillSchema
- Generate id: `parsed.data.id || crypto.randomUUID()`
- INSERT INTO skills with all fields
- Return 201 `ok({ skill: { id, ...parsed.data } })`

**PUT /:id** — Update skill:
- Validate body with updateSkillSchema
- Build dynamic SET clause from provided fields (only update what's passed)
- Always set `updated_at = EXTRACT(EPOCH FROM NOW())`
- Return `ok({ skill: updatedRow })` or 404

**DELETE /:id** — Delete skill:

- Return `ok({ deleted: true })` or 404 if no rows affected

**PUT /:id/toggle** — Toggle enabled state (backward compatibility):
