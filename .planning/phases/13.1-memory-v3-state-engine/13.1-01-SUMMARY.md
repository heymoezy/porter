## What Was Built

**Three new PostgreSQL tables:**

- `directives` — workspace and project-scoped operating rules injected into every prompt. Columns: id, scope ('workspace'|'project'), scope_id, content, priority (lower=higher), source_type, status, created_by, timestamps.
- `project_notes` — project-scoped notes categorized as state/decision/constraint. Columns: id, project_id, content, note_type, confidence_score, source_type, status, created_by, timestamps.
- `agent_notes` — persona-scoped learnings, directives, and constraints. Columns: id, agent_id (personas.id), content, note_type, confidence_score, source_type, status, superseded_by_id (for consolidation), created_by, timestamps.

**Migration (migrate-memv3.ts):**
- Idempotency key: `memory_v3` in schema_migrations
- Enables `pg_trgm` extension (trigram similarity for future consolidation)
- Adds `migrated_to_v3` column to concepts (IF NOT EXISTS — idempotent)
- Creates all three tables (IF NOT EXISTS)
- Creates 3 performance indexes + 1 bonus idx_concepts_scope
- Migrates accepted/active concepts: directive kind -> directives table; agent scope -> agent_notes (via template_id lookup); project scope -> project_notes
- Marks migrated concepts with migrated_to_v3=1 (not deleted — preserves V2 continuity)
- Wired into startup after migrateConsolidated

**Smoke test (smoke-phase13.1.sh):**
- 10 active MEMV3-01 schema checks (all pass: 10/10)
- 4 MEMV3-02..05 placeholders with clear skip messages for Plans 02/03
- Exit code 1 on any failure
