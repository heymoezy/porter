    Export function:
    ```typescript
    export async function consolidateAgentMemory(agentId: string): Promise<{
      before: number;
      after: number;
      merged: number;
      pairs: Array<{ kept: string; superseded: string; similarity: number }>;
    }>
    ```

    Implementation:
    - Count active agent_notes for this agent: `SELECT COUNT(*) AS cnt FROM agent_notes WHERE agent_id = $1 AND status = 'active'`
    - Store as `before`
    - Find similar pairs using pg_trgm:
      ```sql
      SELECT a.id AS a_id, a.content AS a_content, a.confidence_score AS a_score,
             b.id AS b_id, b.content AS b_content, b.confidence_score AS b_score,
             similarity(a.content, b.content) AS sim
      FROM agent_notes a
      JOIN agent_notes b ON a.id < b.id
      WHERE a.agent_id = $1
        AND b.agent_id = $1
        AND a.status = 'active'
        AND b.status = 'active'
        AND similarity(a.content, b.content) > 0.6
      ORDER BY sim DESC
      ```
    - For each pair, keep the one with higher confidence_score (if equal, keep the one with alphabetically smaller id for determinism)
    - Track which IDs to supersede in a Set to avoid double-superseding
    - In a transaction:
      - For each pair where neither has been superseded yet:
        - Determine winner (higher confidence) and loser
        - UPDATE agent_notes SET status = 'superseded', superseded_by_id = $winner, updated_at = EXTRACT(EPOCH FROM NOW()) WHERE id = $loser AND status = 'active' RETURNING id
        - If RETURNING returns 0 rows, skip (already modified by concurrent edit — use the pattern from research pitfall 4)
        - Track in pairs array
    - Count active agent_notes after consolidation as `after`
    - Return { before, after, merged: before - after, pairs }

    Also run consolidation on the `concepts` table for the same agent (scope='agent', scope_id mapping). Query `SELECT id FROM personas WHERE id = $1` to confirm the persona exists, then find the template_id via `SELECT template_id FROM personas WHERE id = $1`. Use template_id as scope_id for concepts:
      ```sql
      SELECT a.id AS a_id, a.confidence_score AS a_score,
             b.id AS b_id, b.confidence_score AS b_score,
             similarity(a.content, b.content) AS sim
      FROM concepts a
      JOIN concepts b ON a.id < b.id
      WHERE a.scope = 'agent' AND a.scope_id = $1
        AND b.scope = 'agent' AND b.scope_id = $1
        AND a.status = 'active' AND b.status = 'active'
        AND similarity(a.content, b.content) > 0.6
      ORDER BY sim DESC
      ```
    Same merge strategy for concepts — winner keeps, loser gets status='superseded', superseded_by_id=winner.

    Wrap everything in try/catch. On error, rollback and rethrow.

    Route: `fastify.get('/admin/overview', { preHandler: [fastify.requireAuth] }, ...)`

    Run aggregation query:
    ```sql
    SELECT
      p.id AS agent_id,
      p.name AS agent_name,
      COUNT(DISTINCT c.id) FILTER (WHERE c.status = 'active') AS concept_count,
      COUNT(DISTINCT an.id) FILTER (WHERE an.status = 'active') AS note_count,
      COALESCE(AVG(
        CASE WHEN an.status = 'active' THEN an.confidence_score
             WHEN c.status = 'active' THEN c.confidence_score
        END
      ), 50)::integer AS avg_confidence,
      COUNT(DISTINCT c.id) FILTER (WHERE c.review_state = 'pending' AND c.status = 'active') AS pending_review_count
    FROM personas p
    LEFT JOIN concepts c ON c.scope = 'agent' AND c.scope_id = p.template_id AND c.status = 'active'
    LEFT JOIN agent_notes an ON an.agent_id = p.id AND an.status = 'active'
    WHERE p.is_system = 0
    GROUP BY p.id, p.name
    ORDER BY p.name
    ```

    Compute health_score in application code: `Math.min(100, Math.max(0, avg_confidence - (pending_review_count * 5)))`

    ```sql
      (SELECT COUNT(*) FROM directives WHERE status = 'active') AS total_directives,
      (SELECT COUNT(*) FROM concepts WHERE review_state = 'pending' AND status = 'active') AS total_pending_review
    ```
