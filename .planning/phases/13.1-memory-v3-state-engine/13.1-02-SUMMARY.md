## What Was Built

**memory-injection.ts (new service):**

- `buildMemoryContext(opts)` — async function accepting `agentId`, `projectId`, `tokenBudget`, `searchQuery`
- Default token budget: 2000 tokens (overridable per-agent via `personas.config.memory_token_budget`)
- `estimateTokens(text)` — internal helper: `ceil(length / 4)` approximation
- 5-tier ordered pipeline with per-tier clipping:
  - **Tier 1 — Agent Identity**: queries `personas` for name + role; overrides budget if config has `memory_token_budget`; skipped if no agentId
  - **Tier 2 — Directives**: queries workspace directives + project-scoped directives if projectId provided; ordered by priority ASC (lower priority number = higher precedence)
  - **Tier 3 — Project Notes**: queries `project_notes` ordered by confidence_score DESC; skipped if no projectId
  - **Tier 4 — Agent Notes**: queries `agent_notes` ordered by confidence_score DESC; skipped if no agentId
  - **Tier 5 — Archival FTS**: queries `concepts` via `websearch_to_tsquery` + `ts_rank`; skipped if no searchQuery or remainingTokens ≤ 50
- Sections concatenated with `\n\n` separator; empty sections silently skipped
- Full try/catch — any DB error logs to console.error and returns `''`

**chat.ts (modified):**

- Added import: `import { buildMemoryContext } from '../../services/memory-injection.js'`
- Calls `buildMemoryContext({ agentId, projectId, searchQuery: message })` after identity prefix construction, before streaming
- Updated `augmentedMessage` construction:
  - If memoryContext present: `memoryContext + '\n\n---\n\n' + message`
  - If identityPrefix present: prepended first (runtime label before knowledge block)
  - Final order: `[identity prefix][memory context]\n\n---\n\n[user message]`
- Original `message` variable still persisted to `chat_messages` unchanged
