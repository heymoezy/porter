### Human Verification Required

#### 1. Injection quality in live chat

**Test:** Open a chat session with a configured agent (e.g., Porter) and send a message. Observe the AI response for evidence it received agent identity, directives, or agent notes.
**Expected:** AI response reflects the agent's configured role and any active directives.
**Why human:** Verifying actual prompt content passed to the model requires reading the AI's behavior; automated checks can only confirm the injection path is wired, not that the model acknowledges the context.

#### 2. pg_trgm consolidation effectiveness on real data

**Test:** Run `POST /api/v1/memory/consolidate` with a production agent_id that has accumulated notes. Verify the `merged` count in the response is non-zero when similar notes exist.
**Expected:** Pairs of similar notes with similarity > 0.6 are merged; loser gets `status='superseded'`.
**Why human:** pg_trgm similarity thresholds require real content to judge quality — whether 0.6 is the right threshold for this data is a judgment call.

#### 3. Admin overview health score accuracy

**Test:** Navigate to `GET /api/v1/memory/admin/overview` with a platform_admin account. Review the per-agent `health_score` values.
**Expected:** Health scores reflect actual agent memory quality; lower scores for agents with many pending review items.
**Why human:** Health score formula (avg_confidence minus 5*pending) requires business judgment to validate it's meaningful.
