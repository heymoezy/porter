Create backend/src/services/file-ingress.ts with these exports:

1. **File classification (pure function, no LLM):**

   Export type FileCategory = 'document' | 'code' | 'data' | 'media' | 'config' | 'archive' | 'other'

   Export function classifyFile(filename: string, mimeType: string): FileCategory
   - Uses extension-based lookup (no LLM calls — instant classification):
     - document: md, txt, pdf, doc, docx, rtf, odt, tex, html (when not code)
     - code: py, js, ts, tsx, jsx, sh, rs, go, java, c, cpp, h, rb, php, bash, sql, graphql, css, yaml, yml, toml, json (when in code context)
     - data: csv, tsv, xlsx, xls, sqlite, db, parquet, json (when data-shaped), xml (when data-shaped)
     - media: png, jpg, jpeg, gif, svg, webp, ico, mp3, mp4, webm, wav, ogg, avi, mov
     - config: env, ini, conf, cfg, properties, gitignore, dockerignore, editorconfig
     - archive: zip, gz, tar, rar, 7z, bz2
     - other: anything else
   - Ambiguous extensions (json, yaml, xml): default to 'config' if filename matches common config patterns (package.json, tsconfig.json, docker-compose.yml etc.), otherwise 'data'

2. **Category-to-directory mapping:**

   Export const CATEGORY_DIR_MAP: Record<FileCategory, string> = {
     document: 'context',
     code: 'work',
     data: 'context',
     media: 'outputs',
     config: 'work',
     archive: 'archive',
     other: 'intake',
   }

3. **Route file (move from uploads to project dir):**

   Export async function routeFile(opts: {
     diskPath: string;          // current location in uploads/
     filename: string;
     projectFsPath: string;     // project root dir from projects.fs_path
     category: FileCategory;
   }): Promise<string>
   - Target dir = projectFsPath + '/' + CATEGORY_DIR_MAP[category]
   - Ensure target dir exists (fs.mkdir recursive)
   - If file already exists at target, append _1, _2 etc. to filename (before extension)
   - Move file: fs.rename(diskPath, targetPath) — if cross-device, fall back to fs.copyFile + fs.unlink
   - Return the new absolute path

4. **Memory signal emission:**

   async function emitIngressSignal(opts: {
     projectId: string;
     filename: string;
     category: FileCategory;
     targetDir: string;
   }): Promise<void>
   - INSERT INTO concepts: memory_kind='concept', trust_tier='medium', scope='project', scope_id=projectId, content='File ingested: {filename} classified as {category}, routed to {targetDir}', source_type='file_ingress', status='active', review_state='accepted'
   - Use crypto.randomUUID() for id
   - Best-effort: catch errors and log, don't throw

5. **Project context update:**

   async function appendFileReference(opts: {
     projectFsPath: string;
     filename: string;
     category: FileCategory;
     targetPath: string;
   }): Promise<void>
   - Read _system/project.md
   - Check if "## Files" section exists; if not, append "\n## Files\n\n"
   - Append "- [{category}] {filename} -> {relative path from project root}\n"
   - Write back to _system/project.md
   - Best-effort: catch errors and log

6. **Main entry point:**

   Export async function processIngress(opts: {
     fileId: string;
     filename: string;
     diskPath: string;
     mimeType: string;
     projectId: string;
   }): Promise<{ category: FileCategory; newPath: string } | null>
   - Query projects.fs_path from DB for projectId
   - If no fs_path, log warning and return null (project not provisioned)
   - classifyFile -> category
   - routeFile -> newPath
   - Update files table: UPDATE files SET disk_path = $1 WHERE id = $2 (update to new location)
   - emitIngressSignal (fire-and-forget .catch)
   - appendFileReference (fire-and-forget .catch)
   - emitSSE('project:file-ingress', { projectId, fileId, filename, category })
   - Return { category, newPath }
   - Entire function wrapped in try/catch returning null on error (never blocks upload)
