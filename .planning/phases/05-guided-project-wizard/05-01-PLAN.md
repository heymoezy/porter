**1. Create `backend/src/routes/v1/wizard.ts`:**

Import: `FastifyInstance`, `FastifyPluginOptions`, `db`, `sqlite` from db/client, `ok`, `err` from envelope, `featureFlags` from config, `z` from zod, `crypto`, `dispatch` from ai-router, types from types/wizard, `fs` and `path` for reading personas directory.

**Feature flag guard:** Before all handlers, check `featureFlags.guidedWizard`. If false, return 503 with `err('FEATURE_DISABLED', 'Guided wizard is not enabled')`.

**Template loading helper:** On module load, scan the `personas/` directory (resolve relative to project root via `path.resolve(__dirname, '../../../../personas')`) to build a list of available agent templates. For each subdirectory, read the directory name as the templateId. Build a `AVAILABLE_TEMPLATES` array of `{ templateId: string, name: string }`. Cache this list (it only changes on deploy, not at runtime). Wrap in try/catch — if personas/ doesn't exist, use an empty array and log a warning.

```typescript
function loadAvailableTemplates(): Array<{ templateId: string; name: string }> {
  try {
    const personasDir = path.resolve(__dirname, '../../../../personas');
    if (!fs.existsSync(personasDir)) return [];
    return fs.readdirSync(personasDir, { withFileTypes: true })
      .filter(d => d.isDirectory())
      .map(d => ({ templateId: d.name, name: d.name.replace(/-/g, ' ').replace(/\b\w/g, c => c.toUpperCase()) }));
  } catch { return []; }
}
const AVAILABLE_TEMPLATES = loadAvailableTemplates();
```

**Zod schemas:**
```typescript
const detectSchema = z.object({
  action: z.literal('detect'),
  message: z.string().min(1).max(2000),
});

const proposeSchema = z.object({
  action: z.literal('propose'),
  goal: z.string().min(1).max(2000),
  answers: z.array(z.string()).default([]),
});

const approveSchema = z.object({
  action: z.literal('approve'),
  proposal: z.object({
    projectName: z.string().min(1).max(100),
    projectType: z.string(),
    agents: z.array(z.object({
      templateId: z.string(),
      name: z.string(),
      role: z.string(),
      portrait: z.string().optional().default(''),
      whyChosen: z.string(),
    })),
    milestones: z.array(z.string()),
    scopeLabel: z.string(),
    explanation: z.string().optional().default(''),
  }),
});

const wizardSchema = z.union([detectSchema, proposeSchema, approveSchema]);
```

**POST `/` handler** (with preHandler `[fastify.requireAuth]`):
1. Parse body with `wizardSchema.safeParse(req.body)`. If invalid, return 400 with `err('VALIDATION_ERROR', issues)`.
2. Switch on `parsed.data.action`:

**`detect` action:**
- Use heuristic-first detection:
  - Check message for project-intent keywords: matches pattern like "I need|want|build|create|make|start|launch" + noun phrase
  - If heuristic passes, use `dispatch()` to call Porter LLM with a classification prompt (below)
  - If heuristic fails (short greeting, question, status check), return `{ isProject: false, clarity: 'clear', suggestedQuestions: [] }`
- LLM classification prompt (injected as system context, NOT as full dispatch — keep under 200 tokens):
  ```
  Classify this message. Is the user requesting a new project? Respond with JSON only:
  {"isProject":true/false,"clarity":"clear"|"vague"|"ambiguous","projectType":"website|app|content|research|design|ops|custom","suggestedQuestions":[{"id":"q1","text":"...","options":[{"id":"o1","label":"..."}]}]}
  If vague, include 1-2 clarifying questions with 3-4 selectable options each.
  ```
- Parse LLM JSON response. If LLM call fails, fall back to heuristic-only result.
- Return `ok(detectResult)` conforming to `WizardDetectResult`.

**`propose` action:**
- Build a prompt for Porter LLM that includes:
  - The user's goal
  - Their answers (if any)
  - Available project types: website, app, presentation, research, content, design, ops, custom
  - **Available agent templates from personas/ directory** (per user decision: "template matching from 70 agent archetypes"):
    ```
    Available agent templates (select from these ONLY, do not invent agent names):
    ${AVAILABLE_TEMPLATES.map(t => t.templateId).join(', ')}
    ```
  - Instruction to respond with JSON matching `WizardProposal` shape
  - Instruction to select 2-4 agents from the available templates appropriate for the project type, explaining WHY each was chosen. Use the templateId from the list above.
  - Instruction to generate 3-5 milestones
  - Instruction to estimate scope as "Small (1-2 weeks)", "Medium (1 month)", or "Large (2+ months)"
- Call `dispatch()` with agentId set to the Porter master agent (query `personas` for `is_master = 1`, fallback to first agent)
- Parse LLM response JSON into `WizardProposal`
- Return `ok({ proposal })` conforming to `WizardProposeResult`

**`approve` action:**
- Use `sqlite.transaction()` (from better-sqlite3) for atomic commit:
  1. Generate `projectId = crypto.randomUUID()`
  2. Generate slug from projectName (lowercase, replace spaces with hyphens, strip non-alphanumeric)
  3. INSERT into `projects` table: id, name, slug, type, status='active', description=proposal.explanation, ownerId from session username, milestones as JSON array, metadata with `{ wizard: true }`, createdAt and updatedAt as `unixepoch('now')`
  4. For each agent in `proposal.agents`:
     - Generate `personaId = crypto.randomUUID()`
     - INSERT into `personas`: id, name, role, status='idle', is_temporary=1, config=JSON with `{ project_id: projectId, template_id: agent.templateId }`, created_at=ISO string
     - Generate `jobId = crypto.randomUUID()`
     - INSERT into `agent_jobs`: id, agent_id=personaId, project_id=projectId, trigger_type='wizard_start', prompt=`Project "${proposal.projectName}" just started. Your role: ${agent.role}. Review the milestones and begin your first task.`, status='pending', scheduled_for=`unixepoch('now')`, created_at=`unixepoch('now')`
     - Collect personaId and jobId into result arrays
  5. Log activity for each agent: INSERT into `agent_activity` with event_type='wizard_start', summary=`${agent.name} assigned to ${proposal.projectName}`
- Return `ok({ projectId, agentIds, jobIds })` conforming to `WizardApproveResult`

Export default `async function wizardV1Routes(fastify: FastifyInstance, opts: FastifyPluginOptions)`.
