## Summary

Phase 32 adds a VSCode-style pack explorer to the Porter Admin â€” a dedicated `/skills/:id/pack` route with a left file tree and right editor pane. It also computes quality tier diagnostics (scaffold/baseline/production/high-performing) that propagate as badges everywhere skills appear.

The backend work is small: one new PUT endpoint (`/api/admin/skills/:id/files/*`) for writing files back to disk, and a new `computePackDiagnostics()` function in `skill-library.ts` that scores each pack. The existing GET endpoint and `listSkillFiles()` enumeration already cover reading. The frontend work is larger: a new route file, CodeMirror 6 installation, a reusable `PackExplorer` component, quality badge component, and link wiring in agent-detail.tsx.

Skill packs on disk have a fixed structure: `SKILL.md`, `prompt.md`, `guides/qa-checklist.md`, `examples/README.md`, `meta/skill.json`. Many of the 209 packs are scaffold filler generated by `ensureSkillPack()` â€” all packs generated this way contain the boilerplate string "Operate as" in prompt.md and "none yet" / "- none" in SKILL.md when triggers are empty, making scaffold detection straightforward with phrase matching.

**Primary recommendation:** Use `@uiw/react-codemirror` 4.x as the React wrapper for CodeMirror 6 â€” it handles SSR, lazy loading, and the setup boilerplate. Add a new route `skills/:id/pack` and a new `PUT /:id/files/*` API endpoint. Compute quality scores server-side in `skill-library.ts` and return them on the existing skills list response so badges work site-wide without extra fetches.

### Pattern 1: Quality Tier Computation (Server-Side)
**What:** Compute `qualityTier` and `diagnostics` fields in `skill-library.ts` and attach them to `SkillRecord`. The skills list API already returns `SkillRecord[]`, so all consumers get badges for free.

**When to use:** Any time you need quality data â€” list view, marketplace, agent detail, pack explorer header.

**Example:**
```typescript
// In skill-library.ts

export type QualityTier = 'scaffold' | 'baseline' | 'production' | 'high-performing'

export interface PackDiagnostics {
  fileCount: number
  nonEmptyCount: number
  totalWords: number
  scaffoldPhraseMatches: number
  scaffoldPct: number
  missingFiles: string[]
  emptyFiles: string[]
  qualityTier: QualityTier
}

const SCAFFOLD_PHRASES = [
  'none yet',
  '- none',
  'Operate as ',          // every generated prompt.md starts with this
  'Porter-specific notes', // boilerplate footer block
  'Add examples',
  'TODO',
  'TBD',
  'placeholder',
  'When a project requires', // generated SKILL.md trigger boilerplate
  'When Porter delegates work', // generated trigger boilerplate
  'When specialized',       // generated trigger boilerplate
]

const SCAFFOLD_WORD_THRESHOLD = 300   // total words: below = scaffold
const BASELINE_WORD_THRESHOLD = 600   // above = baseline
const PRODUCTION_WORD_THRESHOLD = 1200 // above = production; else high-performing

function computePackDiagnostics(skillId: string, files: SkillFileSummary[]): PackDiagnostics {
  const dir = getSkillDir(skillId)
  const expectedFiles = ['SKILL.md', 'prompt.md', 'guides/qa-checklist.md', 'examples/README.md', 'meta/skill.json']
  const presentPaths = new Set(files.map(f => f.path))

  const missingFiles = expectedFiles.filter(p => !presentPaths.has(p))
  const emptyFiles: string[] = []
  let totalWords = 0
  let scaffoldPhraseMatches = 0

  for (const file of files) {
    const fullPath = path.join(dir, file.path)
    const text = safeReadText(fullPath)
    if (!text.trim()) { emptyFiles.push(file.path); continue }
    totalWords += text.split(/\s+/).filter(Boolean).length
    for (const phrase of SCAFFOLD_PHRASES) {
      if (text.includes(phrase)) scaffoldPhraseMatches++
    }
  }

  const scaffoldPct = files.length > 0 ? Math.round((scaffoldPhraseMatches / (files.length * SCAFFOLD_PHRASES.length)) * 100) : 100

  let qualityTier: QualityTier = 'scaffold'
  if (missingFiles.length === 0 && emptyFiles.length === 0) {
    if (totalWords >= PRODUCTION_WORD_THRESHOLD && scaffoldPhraseMatches <= 2) qualityTier = 'high-performing'
    else if (totalWords >= BASELINE_WORD_THRESHOLD && scaffoldPhraseMatches <= 4) qualityTier = 'production'
    else if (totalWords >= SCAFFOLD_WORD_THRESHOLD) qualityTier = 'baseline'
  }

  return {
    fileCount: files.length,
    nonEmptyCount: files.length - emptyFiles.length,
    totalWords,
    scaffoldPhraseMatches,
    scaffoldPct,
    missingFiles,
    emptyFiles,
    qualityTier,
  }
}
```

### Pattern 2: File Write Endpoint with Path-Traversal Guard
**What:** Mirror the existing GET `/:id/files/*` pattern exactly, adding write with `fs.writeFileSync`.

**When to use:** The PUT handler in `skills.ts`.

**Example:**
```typescript
// In skills.ts â€” mirrors existing GET /:id/files/* guard pattern
fastify.put('/:id/files/*', async (req, reply) => {
  const { id, '*': relativePath } = req.params as { id: string; '*': string }
  const { content } = (req.body ?? {}) as { content?: string }
  if (typeof content !== 'string') {
    reply.status(400)
    return err('INVALID_INPUT', 'content must be a string')
  }
  const base = path.resolve(SKILLS_ROOT, id)
  const target = path.resolve(base, relativePath)
  if (!target.startsWith(base)) {
    reply.status(403)
    return err('FORBIDDEN', 'Path traversal rejected')
  }
  fs.mkdirSync(path.dirname(target), { recursive: true })
  fs.writeFileSync(target, content, 'utf8')
  return ok({ saved: true, path: relativePath })
})
```

### File Tree Grouping Logic
The flat walk from `listSkillFiles()` returns relative paths. Group them for display:
- Root files: `SKILL.md`, `prompt.md` (no `/` in path)
- `guides/*`
- `examples/*`
- `meta/*`
- `assets/*` (usually empty, can be hidden if empty)

### Anti-Patterns to Avoid
- **Fetching file content in the tree render:** Only fetch content when a file is selected, not upfront for all files.
- **Re-computing diagnostics on every list call:** Diagnostics involve reading disk files. Fine for the detail view; acceptable for the list since `listSkillFiles()` already walks disk. Cache with React Query staleTime.
- **Mutating SKILLS_ROOT path** in the write handler: Always `path.resolve(base, relativePath)` and check `startsWith(base)` â€” never skip this guard.
- **Skipping dirty check on route change:** The unsaved changes warning must use React Router 7's `useBlocker()` hook, not `window.onbeforeunload` (won't fire on SPA navigation).

## Common Pitfalls

### Pitfall 1: CodeMirror SSR in React Router 7
**What goes wrong:** `@uiw/react-codemirror` uses browser APIs (`document`, `window`) and will crash on the server-side render pass that React Router 7 does.
**Why it happens:** React Router 7 is a full-stack framework that SSRs routes.
**How to avoid:** Lazy-import the component with `React.lazy()` + `Suspense`, or add `ssr: false` to Vite config for the module (not recommended â€” affects full bundle). Best approach: `const CodeMirror = React.lazy(() => import('@uiw/react-codemirror'))` in the explorer component.
**Warning signs:** "window is not defined" crash on first load.

### Pitfall 2: Stale Editor Content After File Switch
**What goes wrong:** User selects file A, edits it, selects file B â€” editor still shows file A's content with B's new fetch overlaid.
**Why it happens:** CodeMirror 6 manages its own internal document state; changing the `value` prop alone does not reset the editor.
**How to avoid:** Use a `key={selectedFile}` prop on the `<CodeMirror>` component to force a full remount on file switch. This resets internal CM6 state cleanly.
**Warning signs:** Content from previous file appears briefly when switching.

### Pitfall 3: PUT Endpoint Route Conflict
**What goes wrong:** `PUT /:id/files/*` conflicts with `PUT /:id` (the existing skill update endpoint) in Fastify's router.
**Why it happens:** Fastify matches routes in registration order with parameter wildcards. `/:id/files/*` must be registered BEFORE `/:id` in the route file.
**How to avoid:** Register the more specific wildcard route before the generic `/:id` handler. Confirmed â€” looking at current skills.ts, `PUT /:id` is at line 114; the new handler must go before it.
**Warning signs:** Save requests hitting the skill update handler instead of the file write handler, returning 400 INVALID_INPUT on `name`.

### Pitfall 4: Diagnostics Performance on 209 Skills
**What goes wrong:** The skills list endpoint walks 209 skill directories Ă— N files each and reads content for word counting â€” could take 2-5 seconds.
**Why it happens:** `computePackDiagnostics()` calls `safeReadText()` per file.
**How to avoid:** Two options: (a) compute diagnostics only on `GET /:id` (detail view), return a lighter `qualityTier` on the list using file size heuristics only; or (b) add simple in-process memoization (Map keyed by skillId + directory mtime). For Phase 32, option (a) is safer: list endpoint returns `qualityTier` from a fast file-size check, detail view returns full `diagnostics`.
**Warning signs:** Skills list taking >1s to load after adding diagnostics.

### Pitfall 5: useBlocker Dirty Guard Not Releasing
**What goes wrong:** After saving, the navigation blocker still fires when the user tries to leave.
**Why it happens:** `isDirty` state not reset after successful save mutation.
**How to avoid:** In the `useMutation` `onSuccess` callback, call `setIsDirty(false)`. Set `isDirty(true)` only in the CodeMirror `onChange` handler when `editorContent !== savedContent`.
