**Zod Schemas:**

```typescript
const createConversationSchema = z.object({
  scope_type: z.enum(['project', 'agent', 'contact', 'global']),
  scope_id: z.string().optional(),
  title: z.string().optional(),
  channel_type: z.string().default('internal'),
  external_id: z.string().optional(),
  metadata: z.record(z.unknown()).optional(),
});

const createMessageSchema = z.object({
  content: z.string().min(1),
  sender_type: z.enum(['user', 'agent', 'external', 'system']).default('user'),
  sender_id: z.string().optional(),
  sender_name: z.string().optional(),
  parent_id: z.number().int().optional(),
  channel_type: z.string().default('internal'),
  channel_metadata: z.record(z.unknown()).optional(),
});
```

**Endpoints:**

1. **GET /** — List conversations with optional filters.
   - `preHandler: [fastify.requireAuth]`
   - Query params: `scope_type`, `scope_id`, `q` (FTS search), `limit` (default 50), `offset` (default 0)
   - If `q` is provided (CHAT-03): Run FTS5 search:
     ```sql
     SELECT DISTINCT c.* FROM conversations c
     JOIN messages m ON m.conversation_id = c.id
     JOIN messages_fts ON messages_fts.rowid = m.id
     WHERE messages_fts MATCH ?
     ORDER BY c.updated_at DESC
     LIMIT ? OFFSET ?
     ```
   - If no `q`: Simple query with optional `scope_type` and `scope_id` WHERE clauses.
   - Return `ok({ conversations, total })`.

2. **POST /** — Create a conversation.
   - `preHandler: [fastify.requireAuth]`
   - Validate body with `createConversationSchema`.
   - Validate scope_id existence (if scope_type is 'project', check projects table; if 'agent', check personas table; if 'contact', check contacts table; 'global' requires no scope_id).
   - Generate `id` with `crypto.randomUUID()`.
   - INSERT into conversations table.
   - Return `reply.code(201).send(ok({ conversation }))`.

3. **GET /:id** — Get single conversation with metadata.
   - `preHandler: [fastify.requireAuth]`
   - SELECT from conversations WHERE id = :id.
   - Return 404 if not found.
   - Return `ok({ conversation })`.

4. **PATCH /:id** — Update conversation (title, metadata).
   - `preHandler: [fastify.requireAuth]`
   - Accept `title` (string optional) and `metadata` (object optional).
   - UPDATE conversations SET ... WHERE id = :id.
   - Also update `updated_at = unixepoch('now')`.
   - Return `ok({ conversation })` with the updated row.

5. **DELETE /:id** — Delete conversation and its messages.
   - `preHandler: [fastify.requireAuth]`
   - Delete all messages where conversation_id = :id first.
   - Delete the conversation.
   - Return `ok({ deleted: true })`.

6. **GET /:id/messages** — Get messages for a conversation (CHAT-01, CHAT-02).
   - `preHandler: [fastify.requireAuth]`
   - Optional query params: `flat` (boolean, default false), `limit` (default 100), `offset` (default 0).
   - SELECT all messages WHERE conversation_id = :id ORDER BY created_at ASC, LIMIT, OFFSET.
   - If `flat=true` or `flat=1`: return flat array.
   - Otherwise (default): Build tree structure in application code:
     ```typescript
     interface MessageNode {
       id: number;
       conversation_id: string;
       parent_message_id: number | null;
       sender_type: string;
       sender_id: string | null;
       sender_name: string | null;
       content: string;
       channel_type: string;
       channel_metadata: Record<string, unknown>;
       created_at: number;
       children: MessageNode[];
     }
     ```
     Group by parent_message_id: messages with null parent are roots, others are children of their parent. Build map by id, iterate once to attach children.
   - Return `ok({ messages, total })`.

7. **POST /:id/messages** — Create a message (CHAT-01, CHAT-02).
   - `preHandler: [fastify.requireAuth]`
   - Validate body with `createMessageSchema`.
   - Verify conversation exists (SELECT id FROM conversations WHERE id = :id), return 404 if not.
   - If `parent_id` is provided: verify parent message exists in same conversation, return 400 `INVALID_PARENT` if not.
   - Auto-populate `sender_id` from `request.sessionUser.username` if sender_type is 'user' and sender_id is not provided.
   - Auto-populate `sender_name` from `request.sessionUser.displayName ?? request.sessionUser.username` if sender_type is 'user' and sender_name is not provided.
   - INSERT into messages (FTS5 trigger handles indexing automatically).
   - UPDATE conversations SET updated_at = unixepoch('now') WHERE id = :id.
   - SELECT the inserted message back (using `last_insert_rowid()`).
   - Return `reply.code(201).send(ok({ message }))`.

8. **GET /search** — FTS5 search across all messages (CHAT-03).
   - `preHandler: [fastify.requireAuth]`
   - Query params: `q` (required), `limit` (default 20), `offset` (default 0).
   - If `q` is missing, return 400 `INVALID_INPUT`.
   - Run FTS5 query:
     ```sql
     SELECT m.id, m.conversation_id, m.content, m.sender_name, m.sender_type,
            m.channel_type, m.created_at, rank
     FROM messages m
     JOIN messages_fts ON messages_fts.rowid = m.id
     WHERE messages_fts MATCH ?
     ORDER BY rank
     LIMIT ? OFFSET ?
     ```
   - Return `ok({ results, total })` where total is a separate COUNT query.

**Important implementation notes:**
- Use `sqlite.prepare().all()` and `sqlite.prepare().run()` for all queries (raw better-sqlite3, not Drizzle) — matches existing pattern in chat.ts and collaborators.ts.
- Parse JSON fields with try/catch: `try { JSON.parse(row.metadata || '{}') } catch { {} }`.
- All new IDs via `crypto.randomUUID()` for conversations; messages use autoincrement.
- Wrap multi-statement operations (delete conversation + messages) in `sqlite.transaction()`.
