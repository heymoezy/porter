2. **Create `scripts/migrate-sqlite-to-pg.ts`:**

   This is a standalone script run with `tsx scripts/migrate-sqlite-to-pg.ts`.
   It reads from SQLite and writes to PostgreSQL.

   ```typescript
   #!/usr/bin/env tsx
   /**
    * One-shot SQLite â†’ PostgreSQL migration script.
    * Run with Porter STOPPED: tsx scripts/migrate-sqlite-to-pg.ts
    *
    * Reads: ~/.porter/porter.db (SQLite, read-only)
    * Writes: DATABASE_URL (PostgreSQL)
    *
    * Steps:
    * 1. Connect to both databases
    * 2. Run consolidated PG migration (create tables)
    * 3. For each table in FK order: read SQLite rows, transform types, INSERT into PG
    * 4. Backfill search_vector for messages and concepts
    * 5. Reset serial sequences to max(id) + 1
    * 6. Validate: compare row counts
    * 7. Print summary table
    */
   ```

   **Script structure:**
   - Import `Database` from `better-sqlite3` (read-only open of porter.db)
   - Import `Pool` from `pg`
   - Import `migrateConsolidated` from `../backend/src/db/migrate-consolidated.js`

   **Table migration order** (respects FKs):
   ```
   schema_migrations, users, sessions, tasks, chats, chat_messages, chat_attachments,
   projects, personas, agent_jobs, agent_activity, decision_log, token_usage_daily,
   workspace_connections, project_connections, calendar_events,
   subscriptions, auth_tokens, billing_events,
   project_collaborators, collaboration_events,
   companies, contacts, contact_emails, contact_phones, contact_social,
   conversations, messages, files, file_projects, file_contacts, file_conversations,
   contact_conversations, contact_projects,
   contact_analyses, agent_templates, concepts, learning_sessions
   ```

   **For each table, the migration function must:**
   a. Read all rows: `const rows = sqliteDb.prepare('SELECT * FROM tablename').all()`
   b. If 0 rows, skip with log
   c. Transform types per row:
      - JSON TEXT columns (metadata, config, trigger_data, tags, skills, tools, etc.) â†’ `JSON.parse(value)` before INSERT (pg driver serializes objects to jsonb)
      - Handle NULL and empty string JSON: if value is `null`, `''`, or not valid JSON, default to `{}` or `[]` depending on column
      - BLOB columns (chat_attachments.data): pass as `Buffer`
      - All other columns: pass through as-is
   d. Batch INSERT using multi-row VALUES:
      ```typescript
      // For efficiency, build multi-row INSERT
      const cols = Object.keys(rows[0]);
      const placeholders = rows.map((_, i) =>
        `(${cols.map((_, j) => `$${i * cols.length + j + 1}`).join(', ')})`
      ).join(', ');
      const values = rows.flatMap(row => cols.map(c => row[c]));
      await pool.query(
        `INSERT INTO ${tableName} (${cols.join(', ')}) VALUES ${placeholders} ON CONFLICT DO NOTHING`,
        values
      );
      ```
      For large tables (1000+ rows), batch in chunks of 500 to avoid parameter limit.

   e. Log: `console.log(`Migrated ${tableName}: ${rows.length} rows`);`

   **After all tables migrated:**

   3. **Reset serial sequences** for all tables with SERIAL PKs:
      ```sql
      SELECT setval(pg_get_serial_sequence('table_name', 'id'), COALESCE(MAX(id), 0) + 1, false)
      FROM table_name;
      ```
      Tables needing sequence reset: chat_messages, agent_activity, decision_log, token_usage_daily,
      project_connections, collaboration_events, contact_emails, contact_phones, contact_social,
      messages, file_projects, file_contacts, file_conversations, contact_conversations,
      contact_projects, auth_tokens, billing_events

   4. **Backfill search_vector** columns:
      ```sql
      UPDATE messages SET search_vector =
        setweight(to_tsvector('english', COALESCE(content, '')), 'A') ||
        setweight(to_tsvector('english', COALESCE(sender_name, '')), 'B') ||
        setweight(to_tsvector('english', COALESCE(channel_type, '')), 'C');

      UPDATE concepts SET search_vector =
        to_tsvector('english', COALESCE(content, ''));
      ```

   5. **Validate row counts:**
      ```typescript
      const tables = ['users', 'sessions', 'tasks', ...]; // all table names
      const results = [];
      for (const table of tables) {
        const sqliteCount = sqliteDb.prepare(`SELECT COUNT(*) as cnt FROM ${table}`).get().cnt;
        const pgResult = await pool.query(`SELECT COUNT(*) as cnt FROM ${table}`);
        const pgCount = parseInt(pgResult.rows[0].cnt);
        results.push({
          table,
          sqlite: sqliteCount,
          pg: pgCount,
          status: sqliteCount === pgCount ? 'OK' : 'MISMATCH'
        });
      }
      ```

   6. **Print summary table:**
      ```
      Table                     | SQLite | PG    | Status
      ========================= | ====== | ===== | ======
      users                     | 2      | 2     | OK
      projects                  | 5      | 5     | OK
      ...
      ```
      Use `console.table()` or manual formatting with ANSI colors (green OK, red MISMATCH).

   7. **Exit with code 0 if all OK, code 1 if any MISMATCH.**
