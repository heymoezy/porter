## Decisions Made

- **doublePrecision for timestamps (not timestamptz):** RESEARCH.md documented that the entire codebase stores timestamps as Unix epoch floats, with 45+ files using epoch arithmetic. Converting to timestamptz would require rewriting every timestamp comparison — massive scope and high breakage risk. Accepted deviation: doublePrecision preserves epoch semantics, EXTRACT(EPOCH FROM NOW()) replaces strftime/unixepoch defaults.
- **migrate-consolidated.ts in single transaction:** All-or-nothing schema creation. If any table fails, ROLLBACK ensures no partial state. Idempotency guard (consolidated_pg_v1 key) prevents double-application on restart.
- **customType tsvector:** Drizzle ORM has no native tsvector type. customType from drizzle-orm/pg-core is the correct pattern for unsupported PG types.
