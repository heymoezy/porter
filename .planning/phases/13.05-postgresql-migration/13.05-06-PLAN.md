# Plan 06 — Merge Admin Backend Into Brain

**Wave:** 3 (parallel with Plan 05)
**Goal:** Single backend, one DB, one schema, one auth, one envelope

## Summary

Port all admin routes + services from porter-admin/backend into Brain's Fastify server.
Admin frontend becomes a pure Vite app calling Brain's API. Admin's Fastify server gets deleted.

## Tasks

### 6.1 — Port admin route files into Brain
- Copy all 15 route files from porter-admin/backend/src/routes/ to porter/backend/src/routes/v1/admin/
- Update imports: `../db/client.js` → `../../../db/client.js` (adjust path depth)
- Replace sqlite.prepare() with pool.query() (same PG conversion as Plans 02-04)
- Use Brain's envelope (ok/err with trace_id) — replace admin's envelope import
- Use Brain's auth decorators — replace `fastify.requirePlatformAdmin` with `preHandler: [fastify.requireAuth]` + role check
- Normalize ALL response fields to camelCase (fix admin's inconsistent snake_case/camelCase)
- Fix raw error responses in chat.ts to use err() helper

### 6.2 — Port admin services
- Copy customer-intel.ts and forge.ts to porter/backend/src/services/
- Convert sqlite→pool (same PG conversion)
- Update imports to Brain's db/client path

### 6.3 — Register admin routes in Brain's index.ts

- Add requirePlatformAdmin decorator to Brain's auth plugin if not exists

### 6.4 — Port admin auth endpoint
- Admin has POST /api/v1/auth/login with slightly different logic
- Merge into Brain's existing auth route (keep Brain's version, add any missing admin features)

### 6.5 — Add SMTP config to Brain's config.ts

- Add SMTP_* env vars to Brain's config

### 6.6 — Delete admin's Fastify backend
- Remove porter-admin/backend/ directory (or mark as deprecated)
- Admin frontend is the only thing that needs to stay
