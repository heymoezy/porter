### `/home/lobster/.claude/hooks/porter-session-end.js` (final)

```javascript
#!/usr/bin/env node
// Porter Session End hook — SessionEnd event
//
// Two responsibilities:
//   1. Tells Porter's Intellect session-analyzer that the CLI session is ending
//      so it can synthesize an episode and persist it to the episodes table.
//   2. Phase 48.2 (Plan 04) belt-and-braces: spawn the Stop hook one final time,
//      DETACHED + unref()'d, so it survives this hook's exit and gets one last
//      tail-parse of transcript_path to catch any assistant turn that landed
//      after the previous Stop fire (research Risk 3 / Anthropic Issue #8564).
//
// Fire-and-forget. Never blocks CLI shutdown.

const http = require('http');

let input = '';
process.stdin.setEncoding('utf8');
process.stdin.on('data', (chunk) => (input += chunk));
process.stdin.on('end', () => {
  let data;
  try {
    data = JSON.parse(input);
  } catch {
    data = {};
  }

  const sessionId = data.session_id || process.env.CLAUDE_SESSION_ID || 'unknown';
  if (sessionId === 'unknown') process.exit(0);

  let project = null;
  const cwd = process.cwd();
  const match = cwd.match(/^\/home\/lobster\/projects\/([^/]+)/);
  if (match) project = match[1];

  const body = JSON.stringify({
    sessionId,
    project,
    gateway: 'claude_cli',
  });

  const req = http.request(
    {
      hostname: '127.0.0.1',
      port: 3001,
      path: '/api/v1/intellect/session-end',
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': Buffer.byteLength(body),
      },
      timeout: 3000,
    },
    () => {}
  );
  req.on('error', () => {});
  req.on('timeout', () => req.destroy());
  req.write(body);
  req.end();

  // ─── Phase 48.2 (Plan 04) — belt-and-braces final tail-parse ─────────────
  // Spawn porter-stop.js with the SessionEnd payload re-shaped as a Stop
  // payload. The child MUST be detached + unref'd so it survives this
  // SessionEnd hook's exit. Otherwise the parent's process group dies before
  // the Stop-hook child can finish its POST and the final assistant turn
  // lands nowhere. The Stop hook is idempotent (UNIQUE constraint + backend
  // content+timestamp dedup), so worst case is a no-op when the bookmark is
  // already at EOF.
  try {
    if (data && data.session_id && data.transcript_path) {
      const { spawn } = require('node:child_process');
      const stopPayload = JSON.stringify({
        session_id: data.session_id,
        transcript_path: data.transcript_path,
        cwd: data.cwd || null,
        hook_event_name: 'Stop',
      });
      const child = spawn('node', ['/home/lobster/.claude/hooks/porter-stop.js'], {
        stdio: ['pipe', 'ignore', 'ignore'],
        detached: true,
      });
      child.on('error', () => {});
      try { child.stdin.write(stopPayload); child.stdin.end(); } catch {}
      // Detach from parent so the child survives SessionEnd's exit and can
      // finish its POST asynchronously.
      try { child.unref(); } catch {}
    }
  } catch {
    // fire-and-forget; never throw on SessionEnd
  }
  // ─── end Phase 48.2 belt-and-braces block ────────────────────────────────
});
```

## Decisions Made

- **Config flag default = TRUE.** Capture-on is the default; privacy is opt-out via `/silo none` (per-session) or `INTELLECT_TRANSCRIPT_CAPTURE_ENABLED=false` (global). This mirrors Phase 48.1 (`/silo` default-on with opt-out), and the locked decision in `48.2-RESEARCH.md` ("Locked Decisions" section).
- **Gate placement BEFORE input validation.** A globally disabled instance must return the same neutral response regardless of payload shape — so the gate is the FIRST statement after destructuring. This keeps the response contract stable (callers can treat `skipped:'disabled'` exactly like `skipped:'silo_none'`) and the gate path is a single `if` (no branches to think about).
- **`/transcript/retention-run` namespaced under `/transcript/*`.** The endpoint sits adjacent to `/transcript/turn` rather than `/retention/run` or `/intellect/retention-run`. One prefix per subsystem, one auth posture across all endpoints in the prefix, smoke harness drops in via the existing curl POST pattern.
- **Belt-and-braces via spawn, not via a new `/transcript/flush` endpoint.** The plan offered both options; spawn won because (a) it reuses Stop's existing per-session byte-offset bookmark, (b) the backend content+timestamp dedup pre-check from 48.2-03 makes any re-fire a verified no-op, (c) zero new backend surface to maintain, (d) the Stop hook's existing 250ms flush-race delay is exactly what the SessionEnd flush race needs.
- **Detached + unref is non-negotiable.** Without `{detached:true}` + `child.unref()`, the spawned Stop child would die with the SessionEnd parent's process group before the HTTP POST completed. Verified by reading Node.js child_process docs and by parallel with `npm run dev` background patterns in the rest of the codebase.
