# Phase 48.2 Plan 03: Hook Wiring — UserPromptSubmit Extension + NEW Stop Hook + settings.json Stop Registration

**`UserPromptSubmit` captures every non-/silo user turn via a fire-and-forget POST to `/transcript/turn`. The new `Stop` hook tails `transcript_path` from a per-session byte-offset bookmark, filters strictly to assistant turns (skipping user/attachment/queue-operation/sidechain rows), POSTs each text turn synchronously, and advances the bookmark per-turn so partial-batch failure retries only the failed lines. TRC-08 idempotency is enforced at the backend via a (session_id, role, captured_at, content) dedup pre-check — the UNIQUE constraint alone could not catch bookmark-cleared re-parses because the backend assigns turn_index from MAX+1. All 8 TRC checks green; the smoke harness flips TRC-02/TRC-03/TRC-08 from `[skip]` to `[ ok ]`.**

```javascript
#!/usr/bin/env node
// Porter User Prompt hook — UserPromptSubmit event
//
// Three responsibilities:
//   1. /silo slash-command interception (Phase 48.1, Plan 03)
//      - Detects prompts starting with "/silo" BEFORE the length check.
//      - Synchronously POSTs to /api/v1/intellect/silo-command with a 3000ms
//        timeout, then emits a UserPromptSubmit `decision: "block"` JSON to
//        stdout so the raw /silo line never reaches the model. The backend's
//        confirmation message is echoed as `additionalContext` so the user
//        sees feedback inline.
//      - On HTTP failure / timeout the hook STILL emits a block decision (so
//        /silo never silently falls through to the model).
//   2. Transcript capture (Phase 48.2, Plan 03 — TRC-02)
//      - Fire-and-forget POST to /api/v1/intellect/transcript/turn with
//        role='user'. Skipped for /silo commands (those exit above).
//      - Skipped for prompts shorter than 12 chars (existing length gate).
//   3. Existing correction-detector forwarding (pre-existing behavior)
//      - Fire-and-forget POST to /api/v1/intellect/correction.
//      - Never writes to stdout for the correction path.

const http = require('http');

let input = '';
process.stdin.setEncoding('utf8');
process.stdin.on('data', (chunk) => (input += chunk));
process.stdin.on('end', async () => {
  let data;
  try {
    data = JSON.parse(input);
  } catch {
    process.exit(0);
  }

  const sessionId = data.session_id || process.env.CLAUDE_SESSION_ID || 'unknown';

  // ─── Phase 48.1: /silo slash command interception ──────────────────────
  const rawPrompt = (data.prompt || '').trim();
  if (rawPrompt.startsWith('/silo')) {
    const cmd = rawPrompt;

    let message = 'silo command backend unreachable';
    try {
      const ac = new AbortController();
      const timeout = setTimeout(() => ac.abort(), 3000);
      const resp = await fetch('http://127.0.0.1:3001/api/v1/intellect/silo-command', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ session_id: sessionId, command: cmd }),
        signal: ac.signal,
      });
      clearTimeout(timeout);
      const j = await resp.json().catch(() => ({}));
      if (j && typeof j.message === 'string') {
        message = j.message;
      } else if (!resp.ok) {
        message = `silo command failed (HTTP ${resp.status})`;
      } else {
        message = 'silo command accepted';
      }
    } catch (e) {
      try { process.stderr.write(`[porter-user-prompt] /silo failed: ${e && e.message ? e.message : e}\n`); } catch {}
    }

    // Block the raw /silo command from reaching the model + echo feedback as additionalContext
    const out = {
      decision: 'block',
      reason: message,
      hookSpecificOutput: {
        hookEventName: 'UserPromptSubmit',
        additionalContext: message,
      },
    };
    process.stdout.write(JSON.stringify(out));
    process.exit(0);
  }
  // ─── end /silo interception ────────────────────────────────────────────

  const userMessage = data.prompt || data.user_message || data.message || '';
  if (!userMessage || userMessage.length < 12) {
    process.exit(0);
  }

  // ─── Phase 48.2 TRC-02: capture user turn (fire-and-forget; never blocks) ──
  // The /silo branch above already `process.exit(0)`s before reaching here, so
  // any prompt we see here is a real user turn (length ≥ 12, not /silo). Both
  // this POST and the /correction POST below are fire-and-forget on http.request.
  try {
    const captureBody = JSON.stringify({
      session_id: sessionId,
      cwd: data.cwd || process.cwd(),
      role: 'user',
      content: userMessage.slice(0, 32 * 1024),
      captured_at: new Date().toISOString(),
    });
    const captureReq = http.request(
      {
        hostname: '127.0.0.1',
        port: 3001,
        path: '/api/v1/intellect/transcript/turn',
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Content-Length': Buffer.byteLength(captureBody),
        },
        timeout: 2000,
      },
      () => {}
    );
    captureReq.on('error', () => {});
    captureReq.on('timeout', () => { try { captureReq.destroy(); } catch {} });
    captureReq.write(captureBody);
    captureReq.end();
  } catch {
    // fire-and-forget; never throw on capture path
  }
  // ─── end Phase 48.2 TRC-02 capture block ──────────────────────────────────

  // Detect project from cwd (same logic as porter-session-start.js)
  let project = null;
  const cwd = process.cwd();
  const match = cwd.match(/^\/home\/lobster\/projects\/([^/]+)/);
  if (match) project = match[1];

  const body = JSON.stringify({
    sessionId,
    project,
    userMessage,
    gateway: 'claude_cli',
  });

  const req = http.request(
    {
      hostname: '127.0.0.1',
      port: 3001,
      path: '/api/v1/intellect/correction',
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': Buffer.byteLength(body),
      },
      timeout: 2000,
    },
    () => {}
  );
  req.on('error', () => {});
  req.on('timeout', () => req.destroy());
  req.write(body);
  req.end();
});
```

```javascript
#!/usr/bin/env node
// Phase 48.2 — Porter Stop hook (TRC-03 + TRC-08)
// Captures assistant turns from the JSONL transcript_path using a per-session
// byte-offset bookmark. NEVER writes to stdout (decision:block would force
// Claude to continue per Risk 2). Idempotent via UNIQUE(session_id, turn_index)
// + ON CONFLICT DO NOTHING at the backend.
//
// Bookmark protocol:
//   /tmp/porter-transcript-bookmark/<session_id>.offset holds the byte position
//   PAST the last successfully-handled line. "Handled" means either:
//     (a) the line was a valid assistant turn and postTurn returned 2xx, OR
//     (b) the line was blank / unparseable / non-assistant / sidechain / had
//         empty text content (nothing to post — count as handled and skip).
//   On per-turn postTurn failure: the loop BREAKS and the bookmark is left at
//   lastSuccessOffset so the next Stop fire retries the failed line.
//
// Lives outside the Porter repo at /home/lobster/.claude/hooks/porter-stop.js
// (deliberately uncommitted per 48.1-03 precedent; reproduced verbatim in the
// 48.2-03 SUMMARY for re-deployment).

const fs = require('node:fs');
const path = require('node:path');
const http = require('node:http');
const readline = require('node:readline');

const BOOKMARK_DIR = '/tmp/porter-transcript-bookmark';
const PORTER_HOST = '127.0.0.1';
const PORTER_PORT = 3001;
const CAPTURE_PATH = '/api/v1/intellect/transcript/turn';
const FLUSH_DELAY_MS = 250;
const MAX_CONTENT = 32 * 1024;
const MAX_LINE_BYTES = 32 * 1024; // skip + warn on lines > 32KB (defensive)

function readStdinJson(timeoutMs = 1000) {
  return new Promise((resolve) => {
    let buf = '';
    const t = setTimeout(() => resolve(null), timeoutMs);
    try {
      process.stdin.setEncoding('utf8');
    } catch {
      clearTimeout(t);
      return resolve(null);
    }
    process.stdin.on('data', (c) => { buf += c; });
    process.stdin.on('end', () => {
      clearTimeout(t);
      if (!buf || !buf.trim()) return resolve(null);
      try { resolve(JSON.parse(buf)); } catch { resolve(null); }
    });
    process.stdin.on('error', () => { clearTimeout(t); resolve(null); });
  });
}

function postTurn(payload) {
  // Resolves with {ok: boolean} so the caller can advance the bookmark only on
  // per-turn success. 2xx => ok:true; transport error / non-2xx / timeout => ok:false.
  return new Promise((resolve) => {
    const body = JSON.stringify(payload);
    const req = http.request({
      hostname: PORTER_HOST,
      port: PORTER_PORT,
      path: CAPTURE_PATH,
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': Buffer.byteLength(body),
      },
      timeout: 3000,
    }, (res) => {
      const ok = !!res.statusCode && res.statusCode >= 200 && res.statusCode < 300;
      // Drain to allow connection reuse / avoid leaks
      res.on('data', () => {});
      res.on('end', () => resolve({ ok }));
      res.on('error', () => resolve({ ok: false }));
    });
    req.on('error', () => resolve({ ok: false }));
    req.on('timeout', () => { try { req.destroy(); } catch {} resolve({ ok: false }); });
    req.write(body);
    req.end();
  });
}

async function main() {
  const data = await readStdinJson();
  if (!data) process.exit(0);

  const sessionId = data.session_id;
  const transcriptPath = data.transcript_path;
  const cwd = data.cwd || null;

  if (!sessionId || !transcriptPath) process.exit(0);
  if (!fs.existsSync(transcriptPath)) process.exit(0);

  // Flush-race breathing room (Pitfall 2; Anthropic Issue #8564)
  await new Promise((r) => setTimeout(r, FLUSH_DELAY_MS));

  try { fs.mkdirSync(BOOKMARK_DIR, { recursive: true }); } catch {}
  const bookmarkPath = path.join(BOOKMARK_DIR, `${sessionId}.offset`);
  let offset = 0;
  try { offset = parseInt(fs.readFileSync(bookmarkPath, 'utf8'), 10) || 0; } catch {}

  let stat;
  try { stat = fs.statSync(transcriptPath); } catch { process.exit(0); }
  if (stat.size <= offset) process.exit(0);

  const stream = fs.createReadStream(transcriptPath, { start: offset });
  const rl = readline.createInterface({ input: stream });

  // Per-turn bookmark: advance lastSuccessOffset only past lines whose postTurn
  // returned 2xx (or that were skipped as non-assistant / blank). On partial-batch
  // failure, leave bookmark at last-success offset so the next Stop fire retries
  // the failed lines.
  let lastSuccessOffset = offset;

  try {
    for await (const line of rl) {
      // Compute the byte length of this line + the trailing newline; the bookmark
      // advances past this if (and only if) the line is successfully handled.
      const lineBytes = Buffer.byteLength(line, 'utf8') + 1; // +1 for '\n'
      const offsetAfterLine = lastSuccessOffset + lineBytes;

      // Cap individual line size — JSONL rows that exceed MAX_LINE_BYTES are skipped
      // (defensive; treat as handled so the loop progresses).
      if (lineBytes - 1 > MAX_LINE_BYTES) {
        try { process.stderr.write(`[porter-stop] line > ${MAX_LINE_BYTES}B at offset ${lastSuccessOffset}; skipping\n`); } catch {}
        lastSuccessOffset = offsetAfterLine;
        continue;
      }

      if (!line.trim()) {
        // Blank line: treat as successfully handled (nothing to post).
        lastSuccessOffset = offsetAfterLine;
        continue;
      }

      let row;
      try { row = JSON.parse(line); } catch {
        // Unparseable line: treat as handled (skip) and continue.
        lastSuccessOffset = offsetAfterLine;
        continue;
      }

      // Filter rows that are NOT assistant turns. These are "handled" (skipped)
      // and the bookmark advances past them.
      if (!row || row.type !== 'assistant' || row.isSidechain) {
        lastSuccessOffset = offsetAfterLine;
        continue;
      }
      const msgRole = row.message && row.message.role;
      if (msgRole !== 'assistant') {
        lastSuccessOffset = offsetAfterLine;
        continue;
      }
      const blocks = (row.message && row.message.content) || [];
      const text = blocks
        .filter((b) => b && b.type === 'text' && typeof b.text === 'string')
        .map((b) => b.text)
        .join('\n')
        .trim();
      if (!text) {
        lastSuccessOffset = offsetAfterLine;
        continue;
      }

      // Attempt the POST. Only advance the bookmark on per-turn 2xx success.
      const res = await postTurn({
        session_id: sessionId,
        cwd,
        role: 'assistant',
        content: text.slice(0, MAX_CONTENT),
        captured_at: row.timestamp || new Date().toISOString(),
      });
      if (!res || !res.ok) {
        // Partial-batch failure: stop the loop and keep bookmark at lastSuccessOffset
        // so the next Stop fire retries this line.
        try {
          process.stderr.write(
            `[porter-stop] post failed at offset ${lastSuccessOffset}; will retry on next fire\n`,
          );
        } catch {}
        break;
      }
      lastSuccessOffset = offsetAfterLine;
    }
    // Advance bookmark to the last-successful offset (per-turn precision).
    try { fs.writeFileSync(bookmarkPath, String(lastSuccessOffset)); } catch {}
  } catch (e) {
    try { process.stderr.write(`[porter-stop] parse error: ${e && e.message ? e.message : e}\n`); } catch {}
    // Still persist whatever progress was made.
    try { fs.writeFileSync(bookmarkPath, String(lastSuccessOffset)); } catch {}
  }

  process.exit(0);
}

main().catch(() => process.exit(0));
```

## Decisions Made

- **TRC-08 fix at the backend, not the hook.** The original 48.2 design ("UNIQUE(session_id, turn_index) + ON CONFLICT DO NOTHING catches re-parse dupes") only works when the **hook supplies turn_index** — but per 48.2-02 the backend assigns it from MAX+1. So a re-fire of the same JSONL with cleared bookmark just allocates a fresh index and inserts a duplicate. The smoke harness (Plan 05) explicitly tests this path. The correct fix is at the backend: pre-check `(session_id, role, captured_at, content)` before INSERT. captured_at from the JSONL `timestamp` is deterministic per turn, so two passes of the same JSONL match against the same (timestamp, content) and the second insert is correctly skipped with `{inserted:false, skipped:'duplicate'}`.
- **User-prompt path correctly bypasses the dedup.** The user-prompt hook supplies `captured_at: new Date().toISOString()` — a fresh ISO timestamp per submit. Distinct user submits = distinct timestamps = no dedup match = correct INSERT behaviour. No special-casing needed.
- **Per-turn bookmark advance (not end-of-file write).** The original research draft (and the early version of this hook) wrote `stat.size` to the bookmark after the loop. That works for the all-success case but corrupts the retry semantics on partial-batch failure — if line 5 of 10 fails, the bookmark would jump to end-of-file and lines 5-10 would be lost. The per-turn `lastSuccessOffset` advance pattern guarantees that the next fire retries only the failed line forward.
- **Stop hook stdout must stay empty.** Anything Stop writes to stdout could be interpreted as a `decision:block` payload, forcing Claude to continue. The hook uses `process.stderr.write(...)` for diagnostics and `process.exit(0)` always. Verified with both populated and empty-stdin invocations.
- **Length gate before capture in user-prompt hook.** Prompts < 12 chars exit early — the existing `/correction` path already does this and the capture block sits AFTER the gate, so accidental keystroke-submits don't pollute transcripts.
- **Hook files deliberately uncommitted.** They live in the global Claude Code hook directory at `/home/lobster/.claude/hooks/`, not the Porter repo. Same precedent as 48.1-03. Full contents reproduced verbatim above for re-deployment.

**1. [Rule 1 - Bug] TRC-08 idempotency contract not satisfied by UNIQUE constraint alone**

- **Found during:** Task 2 live-fire test
- **Issue:** The plan's `must_haves` entry says "Stop hook re-fire on same JSONL is idempotent (UNIQUE constraint catches dupes - TRC-08)". The smoke harness (Plan 05) clears the bookmark and fires the Stop hook twice, asserting exactly 1 row remains. First live-fire produced **2 rows** because:

  - Second fire (bookmark cleared, re-parse): `MAX(turn_index)+1 = 1` → INSERTs at turn_index=1 (also unique against existing row at 0) → duplicate **content** at a **different index**.

The pre-check fires only when `args.captured_at` is supplied (Stop hook), so the user-prompt path with fresh ISO timestamps per call sees it as a no-op — distinct user submits correctly produce distinct rows.
- **Files modified:** `backend/src/services/intellect/transcript-capture.ts` (+29 LOC including the union extension `skipped: 'duplicate'`)
