**Step 2 - Extend /home/lobster/.claude/hooks/porter-session-end.js with the belt-and-braces final tail-parse.**

Read the current hook in full. After its existing POSTs (and before exit), add a block that re-invokes the Stop hook logic on the transcript_path:

```javascript
// Phase 48.2 -- belt-and-braces final tail-parse on session end (research Risk 3).
// The Stop hook may have missed the last assistant turn if it fired before the
// JSONL flush landed. Run one more tail-parse against transcript_path using the
// same bookmark, then exit.
//
// IMPORTANT: the spawned child MUST be detached + unref'd so it survives the
// parent SessionEnd hook's exit. Otherwise the parent's process group dies
// before the Stop-hook child can finish its POST, and the final assistant
// turn lands nowhere.
try {
  const { spawn } = require('node:child_process');
  if (data && data.session_id && data.transcript_path) {
    const stopPayload = JSON.stringify({
      session_id: data.session_id,
      transcript_path: data.transcript_path,
      cwd: data.cwd || null,
      hook_event_name: 'Stop',
    });
    const child = spawn('node', ['/home/lobster/.claude/hooks/porter-stop.js'], {
      stdio: ['pipe', 'ignore', 'ignore'],
      detached: true,
    });
    child.on('error', () => {});
    try { child.stdin.write(stopPayload); child.stdin.end(); } catch {}
    // Detach the child from the parent so it survives SessionEnd's exit and
    // can finish its POST asynchronously.
    try { child.unref(); } catch {}
  }
} catch {
  // fire-and-forget
}
```

Notes:
- This is fire-and-forget. The child process exits independently. SessionEnd does NOT wait for it.
- The Stop hook is idempotent (UNIQUE constraint), so worst case is a no-op when the bookmark is already at EOF.
- The variable `data` must be the parsed stdin payload from porter-session-end.js. Read the file to confirm the variable name; adapt if needed.
