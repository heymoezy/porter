## Nyquist Coverage

Every Wave≥1 task has an automated `<verify>` block. This was established by the plan-checker pass — 100% of implementation tasks are sampled by the Wave 0 smoke harness (Plan 48.2-05):

- **All 4 implementation plans (01, 02, 03, 04) have automated `<verify>` blocks on every task.**
- **All 8 TRC requirements are exercised by `tests/smoke-48.2.sh`** (Plan 48.2-05).
- **Wave 0 ships in Wave 1** alongside Plan 01 because the smoke script is independent of any other plan's files (the smoke script's `tests/smoke-48.2.sh` path is disjoint from `backend/src/...` and `~/.claude/hooks/...`).
- **Graceful skip is built-in**: when the smoke script runs in Wave 1 (before Plan 03 ships the hooks), TRC-02/TRC-03/TRC-08 are SKIPPED (not failed) so Plan 05 itself can exit 0. As subsequent plans ship, additional TRCs flip from skipped → green.
- **Poll-not-sleep**: TRC-02/TRC-03/TRC-08 use a polling loop (up to 10s @ 0.5s interval) instead of a fixed sleep, eliminating flakiness on slow hook+backend round-trips.

The Nyquist criterion (sample at twice the highest frequency of state change) is met: every state-changing task (any task that creates files or modifies code) has an automated verify, and every TRC requirement has at least one assertion in the Wave-0 fixture-driven smoke harness.
