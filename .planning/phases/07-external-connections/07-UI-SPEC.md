### Accent Reserved For

Accent (`--accent-porter` / `--primary`) is reserved exclusively for:

1. "Connect" OAuth primary button (filled)
2. Active/focused form input ring
3. Tab underline on active Connections tab
4. Keyboard focus ring on interactive elements (via `--ring`)

Accent is NOT used for: card borders, body text, icons at rest, status dots (those use success/warning/danger/text3).

## Layout Contract

### Connections Page Structure

```
ConnectionsPage
├── Page Header
│   ├── Title: "Connections"                    [Display 20px/600]
│   ├── Subtitle: "Connect external services"   [Body 14px/400, text2]
│   └── "Admin only" Badge (if viewer/operator)
├── Tabs: "Workspace" | "Projects"
└── Workspace Tab
    └── Service Grid (auto-fill columns, min 280px per card)
        ├── ServiceCard: GitHub
        ├── ServiceCard: Email
        ├── ServiceCard: Google Calendar
        └── ServiceCard: WhatsApp
```

### ServiceCard Structure

```
ServiceCard (--surface bg, 1px --border, radius-lg, shadow-card)
├── Card Header
│   ├── Service icon (32×32, provider brand color or --text2)
│   ├── Service name                    [Heading 16px/600]
│   └── ConnectionStatusBadge
│       ├── Status dot (8×8px)
│       └── State label                 [Label 12px/500]
├── Card Body
│   ├── Description (1 sentence)        [Body 14px/400, --text2]
│   └── Last sync: "2 hours ago"        [Label 12px/500, --text3]
└── Card Footer
    └── Action button (full-width)
        ├── State: disconnected → "Connect" (accent/filled)
        ├── State: needs_reauth  → "Reconnect" (warning/outline)
        ├── State: connected     → "Manage"   (ghost)
        └── Admin only: operators see button disabled with tooltip
```

### OAuth Flow States (OAuthConnectButton)

1. Default: "Connect GitHub" — accent fill button, GitHub icon left
2. Loading: spinner replaces icon, button disabled, "Connecting..."
3. Success: toast fires via SSE `connection:status`, card re-renders connected
4. Error: inline error below button — "Authorization failed. Try again."

### API Key Form States (non-OAuth providers)

1. Default: masked input placeholder "Paste API key...", "Save" button disabled
2. Typing: "Save" button enabled
3. Saving: "Save" spins, input disabled
4. Saved: checkmark animation, input re-masks, "Saved" toast via SSE

## Interaction Contracts

### Admin vs Operator/Viewer Behavior

- Admin: all cards fully interactive — connect, disconnect, manage
- Operator/Viewer: cards render with status but action buttons disabled + tooltip "Only admins can manage connections"
- Source: CONTEXT.md — "Admin-only connection management" decision

### Connection Failure (needs_reauth) Flow

1. SSE `connection:status` fires with `status: "needs_reauth"`
2. `ReauthBanner` appears inside the affected ServiceCard (amber, inline — not a blocking modal)
3. Card status dot switches to warning amber
4. Any agent jobs in 'blocked' state show "Waiting for reconnect" in the activity feed
5. User clicks "Reconnect" — triggers the same OAuth flow as initial connect
6. Source: CONTEXT.md — auto-retry / needs_reauth / amber SSE toast decision

### Disconnect Confirmation

Uses `DisconnectDialog` (shadcn Dialog):
- Trigger: "Disconnect" button in Manage state
- Title: "Disconnect [Service]?"
- Body: "Connected agents will lose access. Active jobs will be paused."
- Actions: "Cancel" (ghost) | "Disconnect" (destructive red)
- On confirm: DELETE /api/v1/connections/:id, SSE fires connection:status disconnected, card re-renders

### Project-Level Override

- Located in project settings panel (not on the Connections page)
- Dropdown: "Use workspace connection" (default) | "[Connection Name]"
- Only workspace connections of matching provider type appear in dropdown
- Operators CAN change project-level overrides (read from CONTEXT.md: operators use connections via agents)
- Source: CONTEXT.md — "Project-level overrides via dropdown" decision

### WhatsApp Setup Prerequisites

- Before showing Connect button for WhatsApp: check `PORTER_PUBLIC_URL` is set
- If unset: show inline callout — "WhatsApp requires a public HTTPS URL. Set PORTER_PUBLIC_URL first."
- Callout uses warning color (`--warning`), not destructive
- Source: RESEARCH.md Pitfall 3

| Element | Copy | Notes |

| Page subtitle | Connect services to give agents real-world reach | Source: phase goal paraphrase |
| Primary CTA (OAuth) | Connect [Service] | e.g. "Connect GitHub", "Connect Google Calendar" |
| Primary CTA (API key) | Save Key | Short. Verb + noun. |
| CTA reauth state | Reconnect | Single word — clear intent. |
| CTA manage state | Manage | Opens credential detail/edit panel. |
| Empty state heading | No connections yet | Simple, factual. |
| Empty state body | Connect GitHub, email, or calendar to let your agents take action in the real world. | One sentence. No questions. |
| Connection success toast | [Service] connected | e.g. "GitHub connected" — SSE-driven toast |
| Reauth banner | [Service] needs reconnecting — your agents are paused. | Inline, inside ServiceCard. |
| Error state (OAuth fail) | Authorization failed. Check your credentials and try again. | Below button, --danger color. |
| Last sync label | Last synced [relative time] | e.g. "Last synced 2 hours ago". Use --text3. |
| Never synced label | Never synced | --text3, not an error. |
| Disconnect dialog title | Disconnect [Service]? | Direct question, service name in title. |
| Disconnect dialog body | Connected agents will lose access. Active jobs will be paused. | Two sentences. No fluff. |
| Disconnect confirm button | Disconnect | Same word as action. Destructive red. |
| Admin-only tooltip | Only admins can manage connections | Short. Shows on disabled button hover. |
| WhatsApp prerequisite | WhatsApp requires a public HTTPS URL. Set PORTER_PUBLIC_URL first. | Inline callout, not a toast. |
| API key input placeholder | Paste API key | Short placeholder. No ellipsis. |

## Accessibility

- All status dots include `aria-label` with state text (not just color)
- OAuth popup window is `role="dialog"` at OS level — no extra ARIA needed in parent
- Form inputs have associated `<label>` elements, not just placeholders
- Disabled action buttons include `title` tooltip explaining why disabled
- Focus ring uses `--ring` token (indigo, visible in both modes)
- Min touch target: 44px height on all buttons (WCAG 2.1 AA 2.5.5)
