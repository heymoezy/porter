      ```typescript
      const auth = new google.auth.OAuth2(
        process.env.GOOGLE_CLIENT_ID,

      auth.setCredentials({
      ```

    **syncCalendarEvents(connectionId?: string): Promise<number>**
    - Get calendar client
    - Fetch events from primary calendar: `calendar.events.list({ calendarId: 'primary', timeMin: new Date().toISOString(), timeMax: new Date(Date.now() + 30*86400000).toISOString(), maxResults: 100, singleEvents: true, orderBy: 'startTime' })`
    - For each event:
      - Extract: id (google event ID), summary (title), start.dateTime or start.date, end.dateTime or end.date, all-day flag
      - Try to match to a project: search project name or ID in event description/extendedProperties
      - Upsert into calendar_events: `INSERT OR REPLACE INTO calendar_events (id, connection_id, project_id, google_event_id, title, start_at, end_at, all_day, synced_at) VALUES (...)`
      - Use `crypto.randomUUID()` for id (or use google_event_id as primary key to avoid duplicates — actually use google_event_id directly as id since it's unique)
    - Return count of synced events
    - On 401: mark connection needs_reauth, emit SSE

    **pushMilestoneToCalendar(params: { title: string; date: string; projectName: string; connectionId?: string }): Promise<string>**
    - Get calendar client
    - Create event: `calendar.events.insert({ calendarId: 'primary', requestBody: { summary: 'Porter: ' + params.title, description: 'Project: ' + params.projectName, start: { date: params.date }, end: { date: params.date } } })`
    - Return google event ID

    **getProjectCalendarEvents(projectId: string): { id: string; title: string; startAt: string; endAt: string | null; allDay: boolean }[]**
    - Query: `SELECT id, title, start_at, end_at, all_day FROM calendar_events WHERE project_id = @projectId ORDER BY start_at ASC`
    - Return mapped rows

    **checkCalendarDeadlines(): number**
    - Query calendar_events where start_at is within next 24 hours
    - For each event with a linked project_id: use the same event trigger mechanism as `checkDeadlineTriggers` in event-triggers.ts
    - Import `getEventSubscribers` from event-triggers and call for 'deadline-approaching' trigger
    - Insert agent_jobs for subscribed agents with trigger_data containing calendar event details
    - Return count of jobs created

    ```typescript
    // Calendar sync — every 60 seconds
    if (featureFlags.externalConnections && tickCount % CALENDAR_SYNC_INTERVAL === 0) {
      try {
        // Only sync if a calendar connection exists
        const hasCalendar = sqlite.prepare(
          `SELECT 1 FROM workspace_connections WHERE provider = 'google_calendar' AND status = 'connected' LIMIT 1`
        ).get();
        if (hasCalendar) {
          await syncCalendarEvents();
          checkCalendarDeadlines();
        }
      } catch (e) {
        console.error('[scheduler] calendar sync error', e);
      }
    }
    ```

    This ensures calendar sync only runs when:
    1. The externalConnections feature flag is enabled
    2. A connected Google Calendar exists
    3. The 60-second interval has elapsed
