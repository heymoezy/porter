    **getWhatsAppCredentials(connectionId?: string): { phoneNumberId: string; accessToken: string }**
    - Query workspace_connections WHERE provider='whatsapp' AND status='connected'
    - Decrypt meta_json
    - Return { phoneNumberId, accessToken }
    - Throw if no connection

    **sendWhatsAppMessage(params: { to: string; text: string; agentName?: string; agentEmoji?: string; connectionId?: string }): Promise<{ messageId: string }>**
    - Get credentials
    - Prefix message with agent identity: if agentName, format as `"${agentEmoji ?? '🤖'} ${agentName}: ${params.text}"`, else just params.text
    - Send via Meta Cloud API:
      ```typescript
      const resp = await fetch(`https://graph.facebook.com/v21.0/${creds.phoneNumberId}/messages`, {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${creds.accessToken}`,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          messaging_product: 'whatsapp',
          to: params.to,
          type: 'text',
          text: { body: messageText },
        }),
      });
      ```
    - If resp not ok: throw with status code. If 401: mark connection needs_reauth, emit SSE.
    - Return `{ messageId: result.messages[0].id }`
    - NEVER log the access token

    **routeInboundWhatsApp(from: string, message: string, groupId?: string): string | null**
    - Check for @mention routing: regex `/@(\w+)/` in message text
    - If @mention found: query personas WHERE name ILIKE mentionedName AND status != 'retired' LIMIT 1
    - If agent found: insert agent_job with trigger_type='whatsapp_message', trigger_data containing `{ from, message, group_id: groupId }`, prompt = message text
    - If no @mention or agent not found: insert agent_job for Porter (SELECT id FROM personas WHERE is_master = 1 LIMIT 1) for AI-based dispatch
    - If groupId provided: look up project linked to this group. Query workspace_connections meta_json for group_mappings: `{ groupId: projectId }`
    - Return agent_id that was dispatched to

    **verifyWebhookSignature(signature: string, body: string): boolean**
    - Uses crypto.createHmac('sha256', process.env.WHATSAPP_APP_SECRET ?? '').update(body).digest('hex')
    - Compare with signature (after stripping 'sha256=' prefix)
    - Return true if match

    Export all 4 functions.

  <action>
    Create `backend/src/routes/v1/webhooks-whatsapp.ts`:

    **GET /** — Meta webhook verification
    - Meta sends: `hub.mode`, `hub.verify_token`, `hub.challenge` as query params
    - Check `hub.mode === 'subscribe'` AND `hub.verify_token === process.env.WHATSAPP_VERIFY_TOKEN`
    - If valid: return `hub.challenge` as plain text (200)
    - If invalid: return 403

    **POST /** — Inbound message handler
    - Read raw body for signature verification
    - Check `X-Hub-Signature-256` header via `verifyWebhookSignature()`
    - If signature invalid: return 403
    - Parse body JSON. Meta Cloud API payload structure:
      ```
      body.entry[0].changes[0].value.messages[0]
      ```
    - Extract: from (phone number), text.body (message text), metadata
    - Check for group context in `value.metadata` or `value.contacts`
    - Call `routeInboundWhatsApp(from, messageText, groupId)`
    - Always return 200 to Meta (acknowledge receipt even if routing fails — Meta retries on non-200)
    - Log routing result

    **Important:** This route must NOT use requireAuth — Meta sends requests without Porter auth cookies. The webhook signature verification is the authentication mechanism.

    Update `backend/src/routes/v1/index.ts`:
    - Add import: `import webhookWhatsAppRoutes from './webhooks-whatsapp.js';`
    - Add: `fastify.register(webhookWhatsAppRoutes, { prefix: '/webhooks/whatsapp' });`
