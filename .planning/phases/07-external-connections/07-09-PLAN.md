    **export function queueExternalCall(agentId: string, projectId: string | null, service: 'github' | 'email' | 'calendar' | 'whatsapp', action: string, params: Record&lt;string, unknown&gt;): string**
    - Creates an agent_job row with trigger_type='external_call'
    - Uses the pattern from RESEARCH.md Pattern 3:
      ```typescript
      const jobId = crypto.randomUUID();
      sqlite.prepare(`
        INSERT INTO agent_jobs
          (id, agent_id, project_id, trigger_type, trigger_data, status, scheduled_for)
        VALUES (?, ?, ?, 'external_call', ?, 'pending', unixepoch('now'))
      `).run(jobId, agentId, projectId, JSON.stringify({ service, action, ...params }));
      return jobId;
      ```
    - This is the canonical way for any route handler or service to queue an external call without blocking

    **export async function dispatchExternalCall(triggerData: string): Promise&lt;string&gt;**
    - Parse triggerData JSON: `const data = JSON.parse(triggerData)`
    - Route based on `data.service`:
      ```typescript
      switch (data.service) {
        case 'github':
          return await dispatchGitHub(data);
        case 'email':
          return await dispatchEmail(data);
        case 'calendar':
          return await dispatchCalendar(data);
        case 'whatsapp':
          return await dispatchWhatsApp(data);
        default:
          throw new Error(`Unknown external service: ${data.service}`);
      }
      ```

    **dispatchGitHub(data):**
    - Route based on `data.action`: 'list_repos', 'read_file', 'create_branch', 'create_pr'
    - For 'list_repos': call `listRepos()`, return JSON.stringify(repos)
    - For 'read_file': call `readFile(data.owner, data.repo, data.path, data.ref)`, return content
    - For 'create_branch': call `createBranch(data.owner, data.repo, data.branch_name, data.from_ref)`, return branch name
    - For 'create_pr': call `createPullRequest(data.owner, data.repo, { title: data.title, body: data.body, head: data.head, base: data.base })`, return JSON.stringify({ number, html_url })
    - Throw on unknown action

    **dispatchEmail(data):**
    - For 'send': call `sendEmail({ to: data.to, subject: data.subject, body: data.body, agentName: data.agent_name })`, return `Email sent: ${result.messageId}`

    **dispatchCalendar(data):**
    - For 'push_milestone': call `pushMilestoneToCalendar({ title: data.title, date: data.date, projectName: data.project_name })`, return event ID

    **dispatchWhatsApp(data):**
    - For 'send': call `sendWhatsAppMessage({ to: data.to, text: data.text, agentName: data.agent_name, agentEmoji: data.agent_emoji })`, return `Message sent: ${result.messageId}`

    **export function checkConnectionHealth(service: string): 'ok' | 'blocked'**
    - Map service to provider: github->'github', email->'email', calendar->'google_calendar', whatsapp->'whatsapp'
    - Query: `SELECT status FROM workspace_connections WHERE provider = ? LIMIT 1`
    - If no row found: return 'blocked' (no connection configured)
    - If status is 'needs_reauth' or 'error': return 'blocked'
    - If status is 'connected': return 'ok'
    - Otherwise: return 'blocked'

    In `executeJob()` function, add a branch BEFORE the aiRouterDispatch call:
    ```typescript
    if (job.trigger_type === 'external_call') {
      // CONN-05 locked decision: jobs depending on broken connections get 'blocked' status
      const triggerData = JSON.parse(job.trigger_data);
      const healthStatus = checkConnectionHealth(triggerData.service);
      if (healthStatus === 'blocked') {
        // Set job to 'blocked' — it will be retried when the connection is restored
        sqlite.prepare(
          `UPDATE agent_jobs SET status = 'blocked', result = ? WHERE id = ?`
        ).run(`Connection ${triggerData.service} is not available — waiting for reauth`, job.id);
        logActivity(job.agent_id, job.id, job.project_id, 'job_blocked',
          `Blocked: ${triggerData.service} connection needs reauth`,
          JSON.stringify({ service: triggerData.service, reason: 'connection_unavailable' }));
        emitSSE('agent:activity', {
          agent_id: job.agent_id,
          project_id: job.project_id,
          event_type: 'job_blocked',
          summary: `Waiting for ${triggerData.service} reconnection`,
        }).catch(() => {});
        return;
      }

      const result = await dispatchExternalCall(job.trigger_data);
      markJobComplete(job.id, result);
      logActivity(job.agent_id, job.id, job.project_id, 'job_complete',
        `External call completed (${triggerData.service})`,
        JSON.stringify({ service: triggerData.service }));
      emitSSE('agent:activity', {
        agent_id: job.agent_id,
        project_id: job.project_id,
        event_type: 'job_complete',
        summary: `External call completed`,
      }).catch(() => {});
      return;
    }
    ```

    Also add a periodic unblock check in the scheduler tick (after the calendar sync block):
    ```typescript
    // Unblock jobs whose connections have been restored
    if (featureFlags.externalConnections && tickCount % 15 === 0) {
      const blockedJobs = sqlite.prepare(
        `SELECT id, trigger_data FROM agent_jobs WHERE status = 'blocked' AND trigger_type = 'external_call'`
      ).all() as { id: string; trigger_data: string }[];
      for (const bj of blockedJobs) {
        try {
          const td = JSON.parse(bj.trigger_data);
          if (checkConnectionHealth(td.service) === 'ok') {
            sqlite.prepare(`UPDATE agent_jobs SET status = 'pending' WHERE id = ?`).run(bj.id);
          }
        } catch {}
      }
    }
    ```

    This ensures external_call jobs bypass the AI router and go directly to the appropriate service module, AND jobs targeting broken connections are blocked (not failed) and automatically unblocked when the connection is restored.
