    **At the top of index.ts**, add import:
    ```typescript
    import { startImapIdle, stopImapIdle } from './services/email.js';
    ```

    **After the Fastify server starts listening** (after `fastify.listen(...)` resolves), add an auto-start block:
    ```typescript
    // Auto-start IMAP IDLE if a connected email connection exists
    try {
      const emailConn = sqlite.prepare(
        "SELECT id FROM workspace_connections WHERE provider = 'email' AND status = 'connected' LIMIT 1"
      ).get() as { id: string } | undefined;
      if (emailConn) {
        startImapIdle(emailConn.id).catch((err) => {
          console.error('[email] IMAP IDLE auto-start failed:', err.message);
        });
        console.log('[email] IMAP IDLE auto-started for existing connection');
      }
    } catch (err) {
      console.error('[email] Failed to check email connection on startup:', err);
    }
    ```

    This is a fire-and-forget call: `startImapIdle` runs its own reconnect loop internally. The `.catch()` prevents unhandled rejection if credentials are expired (it will mark the connection `needs_reauth` internally).

    **Register shutdown hook** (using Fastify's `onClose` hook, add near other close hooks):
    ```typescript
    fastify.addHook('onClose', async () => {
      stopImapIdle();
    });
    ```

    This ensures the IMAP connection is cleanly terminated when the server shuts down (e.g., `systemctl --user restart porter`), preventing orphaned TCP connections to Gmail's IMAP server.

    **Do NOT** modify any other server startup logic. This is purely additive — two imports, one startup block after listen, one close hook.
