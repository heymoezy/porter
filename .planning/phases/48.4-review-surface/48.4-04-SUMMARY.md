## Failure-Code → Toast Message Mapping

| HTTP | Code           | Toast (accept)                                                        | Toast (reject)                              |
| ---- | -------------- | --------------------------------------------------------------------- | ------------------------------------------- |
| 404  | NOT_FOUND      | "Proposal not found"                                                  | "Proposal not found"                        |
| 409  | INVALID_STATE  | "Proposal status changed — list refreshed"                            | "Proposal status changed — list refreshed"  |
| 410  | TARGET_GONE    | "Target directive no longer exists — proposal removed"                | (reject path doesn't hit this)              |
| 422  | SEALED_SEED    | "Seed directives are protected — cannot accept this proposal"         | (reject path doesn't hit this)              |
| 422  | SILO_MISMATCH  | "Proposal targets wrong silo — review needed"                         | (reject path doesn't hit this)              |
| 500  | ACCEPT_FAILED  | "Accept failed: {server message}"                                     | (reject uses REJECT_FAILED fallback path)   |
| —    | UNKNOWN/other  | "Accept failed: {message or code}" (fallback)                         | "Reject failed: {message or code}" (fallback) |
| 200  | success        | "Proposal accepted — directive landed"                                | "Proposal rejected"                         |

All accept-error toasts trigger `qc.invalidateQueries({ queryKey: ['admin', 'dreams', 'proposals'] })` so the stale row clears immediately (relevant for NOT_FOUND / INVALID_STATE / TARGET_GONE).

Playwright RVS-12 asserts on `/sealed|seed|protected/i` — `Seed directives are protected — cannot accept this proposal` matches all three keywords.

## DiffBlock Approach

**v1: set-difference (not LCS).** Pseudocode:

```
beforeSet = Set(before.split('\n'))
afterSet = Set(after.split('\n'))

for each line in beforeLines:
  if line not in afterSet: render with red bg + "−" prefix
  else:                    render plain (unchanged)

for each line in afterLines:
  if line not in beforeSet: render with green bg + "+" prefix
```

**Why not LCS?** For dream-worker proposals where the model produces wholesale rewrites of directive content, grouping removed lines at the top and added lines at the bottom is readable. True LCS/Myers diff would let interleave inline (better for line-level edits inside a single block), but at the cost of a npm dep (`diff` or `diff2html`) and ~40 more LOC. Deferred to a follow-up phase if proposal previews show fine-grained edits frequently.

**Where rendered:** Only for `proposal_kind === 'supersede'` AND `targetQ.data` is non-null. The target fetch (`GET /api/admin/intelligence/directives/:id`) currently returns 404 — endpoint does NOT exist in `backend/src/routes/admin/intelligence.ts` (verified by `grep -rn directives/:id backend/src/routes/admin/`). So in practice DiffBlock is dormant code until the directives detail endpoint ships. The fallback (showing target_directive_ids list only) is the live behavior.

| User action                       | State change                       | Effect                                                                                |
| --------------------------------- | ---------------------------------- | ------------------------------------------------------------------------------------- |
| Page load                         | runFilter = null                   | Proposals query: no dream_run_id param, returns all matching silo+status              |
| Click "view" on a run row         | runFilter = run.id                 | queryKey changes → query refetches with `&dream_run_id=<id>` → table narrows to that run |
| Click "Clear run filter" in header | runFilter = null                   | queryKey reverts → table returns to full silo+status view                              |
| Active filter visual cue          | `isFiltered` check inside runs map | Filtered row gets `bg-surface/60` background, "Clear run filter" button appears in header |

queryKey: `['admin', 'dreams', 'proposals', silo, status, runFilter]` — runFilter is now part of the cache key, so cached results don't bleed across filter changes.

## Decisions Made

- **Sheet over Drawer** — `sheet.tsx` exists, `drawer.tsx` does not. Shadcn names are interchangeable; both wrap Radix Dialog. Sheet is what the admin already uses elsewhere.
- **Set-difference DiffBlock, NOT LCS** — readable enough for proposal-style wholesale rewrites; defer LCS (and npm dep) until proposal previews actually need fine-grained inline diffing.
- **Target fetch graceful degradation** — `GET /api/admin/intelligence/directives/:id` does NOT exist (verified). useQuery returns null on any error; DiffBlock renders only when target fetch succeeds. Acceptable v1 — the target_directive_ids list is shown either way.
- **Sonner toast lib** — installed and mounted in root.tsx. Single dep, ships own types, ~7KB gzipped. NO fallback to alert() ever (feedback_porter_alive.md + Playwright assertions on in-page DOM text).
- **Delete-kind two-step accept** — `handleAccept` branches on `proposal_kind === 'delete'`: opens Dialog instead of mutating directly. Other kinds skip the Dialog. Same accept mutation runs in both paths after the Dialog's Archive button.
- **Per-route window listener (not global)** — `useEffect` on `dreams:run-completed` lives inside dreams.tsx so toasts fire only while a reviewer is on the page. Cleanup on unmount prevents leaks when navigating away.
- **runFilter as queryKey member** — adding `runFilter` to queryKey ensures cache doesn't bleed between filtered and unfiltered views.
