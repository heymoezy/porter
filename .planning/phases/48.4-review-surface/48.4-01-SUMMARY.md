## Decisions Made

- **Playwright shape = CommonJS not ESM.** `tests/skill-evolution.spec.js` (the canonical scaffold predecessor) and `tests/setup-auth.js` both use `require()`. `tests/package.json` declares no `"type": "module"`. ESM imports would break `playwright.config.js` resolution. Matched the existing convention.
- **Login helper = UI selectors (loginAdmin) for Playwright + cookie jar POST for smoke.** Playwright tests use `#uname`/`#pw`/`.login-btn` (matches setup-auth.js). Smoke harness uses `curl -c COOKIE_JAR -X POST /api/auth/login` — when login envelope differs (e.g., username vs email field), smoke gracefully skips the dependent RVS section rather than failing.
- **SILO_MISMATCH probe DOES reference a real software-silo directive (silo-sw-design-system).** Necessary to exercise the accept-handler 422 cross-silo guard. Smoke asserts both `status_code == 422` AND `directive content unmutated` after the attempt — defense in depth. Without this probe, accidental cross-silo accept would never be caught.
- **SSE assertions are best-effort.** Background curl -N captures events for 6 seconds; grep on the captured log decides pass/skip. On race (event fires before curl connects, or curl closes before grep reads), harness reports `[skip] RVS-07a` — never `[FAIL]`. Plan 05 live human-verify is the definitive SSE check.
- **Auto-expiry handler invocation has two paths.** First try POST `/api/admin/workflows/run-by-action/memory_proposals_expire` (synchronous endpoint Plan 02 may add for testability). On 404, fall back to direct SQL UPDATE — verifies the data flip but not the broadcast or audit-log write. The workflow ROW presence (BUILTIN_WORKFLOWS seed) is asserted unconditionally — Plan 02 must register it.
- **40 fail lines, not 15 (the spec minimum).** Every RVS assertion has its own explicit `[FAIL] RVS-NN: <reason>` message for fast debugging. Aggregating into fewer fail lines would slow root-cause when something breaks downstream.

### Contracts Plan 02 must honor (DEFINED here, ASSERTED by smoke)

**SSE event topics (colon-namespaced, post-commit only):**
- `proposals:created` — fired by `runDreamWorker` after the transactional INSERT block; payload `{dream_run_id, silo_id, count}`
- `proposals:resolved` — fired by accept handler / reject handler / auto-expiry handler. Single-row payload: `{proposal_id, status, silo_id, dream_run_id}`. Bulk payload (expiry): `{event:'expired', count}`
- `dreams:run-completed` — fired by `runDreamWorker` after the UPDATE-status block; payload `{dream_run_id, silo_id, status, proposals_extracted}`

**Auto-expiry workflow:**
- BUILTIN_WORKFLOWS seed row: `{ trigger_value: 'every_24h', action_type: 'memory_proposals_expire', action_config: {} }`
- Handler signature: `UPDATE memory_proposals SET status='expired' WHERE status='pending' AND expires_at < EXTRACT(EPOCH FROM NOW()) RETURNING id, silo_id`
- Handler post-actions: write one `intellect_events` row with `event_type='proposals_expired'` + swept count; broadcast `proposals:resolved` with `{event:'expired', count}` payload

**Accept-handler matrix (smoke asserts each):**
- `new_directive`: INSERT new row into directives table; proposal status flips to 'accepted'
- `supersede`: UPDATE target directive content; proposal status flips to 'accepted'
- `delete`: UPDATE target directive `status='archived'` (soft-delete, NOT hard DELETE); proposal status flips to 'accepted'
- `merge`: UPDATE all targets `status='archived'` + INSERT new combined directive; proposal status flips to 'accepted'
- `SILO_MISMATCH` pre-flight: 422 when target's scope_id ≠ proposal.silo_id; proposal stays pending; target unmutated
- `SEALED_SEED` pre-flight: 422 when target's source_type='moe-direct'; proposal stays pending; seed unmutated
- Re-accept on already-accepted proposal: 409 INVALID_STATE
- Re-reject on already-rejected proposal: 409 INVALID_STATE
- Accept of non-existent proposal: 404 NOT_FOUND
- Accept with missing target directive: 410 TARGET_GONE
