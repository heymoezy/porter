## Accept Handler Matrix

| proposal_kind   | DB outcome                                                                         | touched ids                |
| --------------- | ---------------------------------------------------------------------------------- | -------------------------- |
| `new_directive` | INSERT into directives (scope='silo', source_type='dream_worker', status='active') | `[new_directive_id]`       |
| `supersede`     | UPDATE single target.content + .priority + .updated_at                             | `[target_id]`              |
| `merge`         | INSERT new combined directive + UPDATE all targets SET status='archived'           | `[new_id, ...target_ids]`  |
| `delete`        | UPDATE target SET status='archived' (soft-delete; preserves audit join chain)      | `[target_id]`              |

All paths flip `memory_proposals.status='accepted'`, set `reviewed_at` and `reviewed_by = sessionUser?.username ?? 'admin'`, and INSERT an `intellect_events` row with `event_type='proposal_accepted'`, `source_type='review_surface'`, payload `{proposal_id, dream_run_id, silo_id, proposal_kind, target_directive_ids_touched, reviewer}`. Post-commit `broadcast('proposals:resolved', {proposal_id, status:'accepted', silo_id, dream_run_id})`.

## Error Codes (HTTP status)

| Code           | HTTP | Trigger                                                                              |
| -------------- | ---- | ------------------------------------------------------------------------------------ |
| NOT_FOUND      | 404  | Proposal id (or run id) does not exist                                               |
| INVALID_STATE  | 409  | Proposal status != 'pending' (idempotent re-accept / re-reject)                      |
| TARGET_GONE    | 410  | One or more `target_directive_ids` no longer in directives                           |
| SILO_MISMATCH  | 422  | Any target has scope != 'silo' OR scope_id != proposal.silo_id                       |
| SEALED_SEED    | 422  | proposal_kind ∈ {delete, supersede, merge} AND any target source_type = 'moe-direct' |
| ACCEPT_FAILED  | 500  | Unexpected DB error after ROLLBACK (catch-block path)                                |
| REJECT_FAILED  | 500  | Unexpected DB error after ROLLBACK on reject (symmetric)                             |

All pre-flight failures ROLLBACK before returning. The `directive_immutable_moe_direct` trigger remains as defense-in-depth.

## SSE Broadcast Topics

| Topic                      | Emit site                                                                                 | Payload                                                                                                |
| -------------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| `proposals:created`        | `dream-worker.ts` post-`insertProposalsTransactionally` COMMIT (gated on count > 0)       | `{dream_run_id, silo_id, count}`                                                                       |
| `dreams:run-completed`     | `dream-worker.ts` × 3 — success path, empty-corpus path, failure (catch) path             | `{dream_run_id, silo_id, status: 'completed'\|'failed', proposals_extracted}`                          |
| `proposals:resolved`       | `admin/dreams.ts` accept post-COMMIT, reject post-COMMIT, workflow-engine expire handler  | accept/reject: `{proposal_id, status:'accepted'\|'rejected', silo_id, dream_run_id}`; expire: `{event:'expired', count}` |

All broadcasts are post-COMMIT or post-UPDATE — never inside an open transaction (Pitfall 1). The failure-path broadcast in dream-worker is wrapped in its own try/catch so a secondary broadcast failure cannot mask the original error.

Documented in Task 4 commit.
- Reject handler stays symmetric-atomic even though a single UPDATE would technically work — keeps the audit-event INSERT inside the same COMMIT so the proposal flip cannot land without its audit row.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Drizzle `$inferSelect` camelCase types don't match raw pg row shape**
- **Found during:** Task 1 (initial tsc pass after writing dreams.ts)
- **Issue:** The plan's action block imported `MemoryProposalRow`/`DreamRunRow` from `schema.ts` and accessed properties as camelCase (`proposal.proposalKind`, `proposal.targetDirectiveIds`, `proposal.siloId`, etc.). These types come from Drizzle's `$inferSelect`, which renames columns to camelCase ONLY when using the Drizzle ORM query builder. Raw `pool.query<T>(sql, params)` returns rows with the actual SQL column names (snake_case). The code would have compiled (type system trusts the assertion) but failed at runtime with `proposal.proposalKind === undefined`.
- **Fix:** Replaced the schema.ts imports with inline interface declarations using snake_case property names (`proposal_kind`, `target_directive_ids`, `silo_id`, etc.) matching the actual DB column names. Updated all 16 property accesses in the accept handler and the audit-event payload to snake_case.
- **Files modified:** `backend/src/routes/admin/dreams.ts`
- **Verification:** `npx tsc --noEmit` clean; live curl against `/proposals/:id/accept` with a seeded proposal returned the expected `directive_ids_touched` array and the proposal flipped correctly (proves runtime property access works).
- **Committed in:** `78b21c3` (Task 1 commit — caught and fixed before commit)

**2. [Plan scope expansion - not a deviation] Added 4th SSE broadcast for empty-corpus success path**
- **Found during:** Task 4
- **Issue:** Plan listed 3 insertion points (A: post-INSERT-COMMIT, B: post-UPDATE-completed, C: post-UPDATE-failed). The empty-corpus path in `dream-worker.ts` (parsed.proposals.length === 0 quiet-week branch) also UPDATEs `dream_runs.status='completed'` and returns, but the plan didn't list it as an insertion point. Without a broadcast there, the UI's "running" pill would stick on legitimate quiet weeks.
- **Fix:** Added a 4th `broadcast('dreams:run-completed', {..., status:'completed', proposals_extracted:0})` at the empty-corpus return path for UX symmetry.
- **Files modified:** `backend/src/services/intellect/dream-worker.ts`
- **Verification:** `grep -c 'broadcast(' backend/src/services/intellect/dream-worker.ts` returns 4.
- **Committed in:** `2c5d297` (Task 4 commit)
