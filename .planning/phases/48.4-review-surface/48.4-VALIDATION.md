## Nyquist Coverage

Every Wave≥1 task has an automated `<verify>` block. The contract is established up-front by Plan 48.4-01 shipping the smoke harness + Playwright scaffold in Wave 1 alongside Plan 48.4-02.

- **All 5 implementation plans (01..05) have automated `<verify>` blocks on every task.**
- **All 14 RVS requirements are exercised by `tests/smoke-48.4.sh` + `tests/dreams.spec.js`** (Plan 48.4-01 creates both).
- **Wave 0 ships in Wave 1** alongside Plan 02 because the smoke script's `tests/smoke-48.4.sh` path + Playwright scaffold + fixture SQL are disjoint from `backend/src/...` and `admin/frontend/...`.
- **Graceful skip is built-in at three layers**:
  1. Backend unreachable → schema gate (memory_proposals + dream_runs table existence) still runs via psql; the rest skip cleanly.
  2. `backend/src/routes/admin/dreams.ts` missing → RVS-01..RVS-07 SKIP via `SKIP_ADMIN_DREAMS=1` (Wave 1 runs complete green on the schema-only criteria).
  3. Playwright tests `.skip()` per case; un-skipped progressively by Plan 03 (RVS-08/09), Plan 04 (RVS-10/10b/11/12), Plan 05 (RVS-13).
- **Poll-not-sleep**: RVS-02 SSE capture uses background `timeout 6 curl -sN` + grep; RVS-06 expiry uses a direct UPDATE invocation fallback when the synchronous run-by-action endpoint is absent; RVS-13 polls `dream_runs.status` (up to 30s @ 500ms).
- **Live unmocked verification gated to Plan 05 human-verify**: smoke uses 7 hand-crafted fixture proposals (deterministic, version-controlled, no Bridge dependency); the real-Sonnet full-loop proof happens once in Plan 05's checkpoint.

The Nyquist criterion (sample at twice the highest frequency of state change) is met: every state-changing task (any task that creates files, registers endpoints, alters worker behavior, or wires SSE) has an automated verify, and every RVS requirement has at least one assertion in the Wave-0 fixture-driven harness OR Playwright scaffold.

## Sampling Rate

- **After every task commit:** `cd backend && npx tsc --noEmit` (zero new errors) AND `cd admin/frontend && npx react-router typecheck` (if frontend touched) — Porter's canonical "did it compile" gate.
- **After every plan wave:** `bash tests/smoke-48.4.sh` + relevant Playwright tests un-skipped progressively.
- **Phase gate:** All four smoke scripts (48.1 + 48.2 + 48.3 + 48.4) exit 0 AND Playwright dreams.spec.js fully un-skipped + all green AND live human-verify approved (Plan 05) AND `/health` returns v6.17.0.
- **Max feedback latency:** ~90 seconds (smoke ~30s + Playwright ~60s).
