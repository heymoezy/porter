## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot gaps, inline padding |
| sm | 8px | Between portrait and text label, badge padding |
| md | 16px | Card internal padding, stat tile padding |
| lg | 24px | Section padding (px-6), column gaps |
| xl | 32px | Major layout gaps |
| 2xl | 48px | (reserved — not used this phase) |
| 3xl | 64px | (reserved — not used this phase) |

Exceptions:
- Agent supervisor card row: px-2 py-2 (8px / 8px) — compact density using standard sm token
- Stat tiles: px-4 py-2 (16px / 8px) — matches md horizontal padding
- Activity feed row: py-2 px-2 (8px / 8px) — high-density list using standard sm token
- Touch targets for interactive rows: minimum 32px height

Source: existing components confirmed — do not change spacing values.

## Stat Tiles Data Contract

| Tile | Label | API source | Sub-label |
|------|-------|-----------|-----------|
| Projects | "Projects" | `GET /api/v1/projects` — `meta.total` | "{active count} active" |
| Agents | "Agents" | `GET /api/v1/agents` — array length | "{working count} working" |
| Jobs / Tasks | "Tasks" | `GET /api/v1/jobs?status=running` — count | "running" |
| Decisions | "Decisions" | `GET /api/v1/decisions` — `meta.total` today | "today" |
| Tokens | "Tokens" | `GET /api/v1/health` — token sum | "today" |

Polling interval: 30 seconds for all stat tiles. Animate number change with `AnimCount` on each update.

Loading state: existing skeleton shimmer using `--shimmer-from` / `--shimmer-via` gradient. Replace stat value with a `w-12 h-5 rounded animate-pulse bg-raised` placeholder per tile.

Error state: show last known value with a faint danger-colored dot indicator on the tile icon. Do NOT clear values on error — stale data beats empty tiles.

## Activity Feed Data Contract

### SSE Connection

- Connect to `/api/events` (SSE endpoint, proxied by Fastify to porter.py hub)
- Filter events where `event_type === "agent_activity"`
- Each event payload shape:

```typescript
interface AgentActivityEvent {
  agent_id: string
  agent_name: string
  event_type: "agent_activity"
  summary: string            // action text shown in feed
  status: "working" | "complete" | "error" | "queued"
  appearance_spec: {
    skin: string
    hair: string
    eyes: string
    shirt: string
    hair_style: string
  }
  timestamp: number          // unix epoch
}
```

- New events prepended to feed, existing items shift down
- Maximum 10 items visible at once — older items drop off bottom
- New item entrance: `animate-slide-down` (exists in app.css)
- Stale items below: `transition-all duration-[var(--duration-slow)] ease-out`
- Appearance: map `appearance_spec` fields to `PixelPortrait` props

### SSE Connection States

| State | Visual treatment |
|-------|-----------------|
| Connecting | Live dot pulses amber (`bg-warning animate-pulse-badge`) |
| Connected | Live dot is accent (`bg-accent-porter animate-pulse-badge`) |
| Disconnected | Live dot turns danger (`bg-danger`), no animation. Banner: "Activity feed offline — reconnecting" |
| Reconnecting | Live dot pulses amber. Auto-retry with exponential backoff (1s, 2s, 4s, 8s max) |

Connection indicator sits in the Activity section header next to the "Activity" label — existing `h-1.5 w-1.5 rounded-full` dot.

### Hero Stats Portraits

The `HeroStats` component renders up to 4 agent portraits in a row. Replace the hardcoded portrait array with the first 4 agents from `GET /api/v1/agents` sorted by `created_at` ascending (oldest first = founding team). If fewer than 4 agents exist, render only the available ones. If 0 agents, render 4 ghost placeholder shapes (grayscale, opacity-20, same 48px box).

## States and Edge Cases

### Zero Agents

| Surface | Treatment |
|---------|-----------|
| StatTiles "Agents" | Shows `0` with sub-label "none yet" |
| Activity feed | Empty state (see Copywriting) |
| HeroStats portraits | 4 ghost placeholder boxes |
| All 4 supervisor cards | Ghost state — grayscale, "pending" |

### One Agent

- That agent fills the matching supervisor slot (if role matches), others remain ghost
- HeroStats: 1 portrait + 3 ghost boxes
- Activity feed: any events from that agent appear live

### 20+ Agents

- StatTiles shows real count with AnimCount
- Supervisor cards: each slot takes the first role-matched agent
- HeroStats portraits: first 4 by created_at — no overflow, no carousel
- Activity feed: capped at 10 items, newest first

### API Unavailable (network error / 5xx)

- All stat tiles: show last known value + faint danger dot on icon
- Supervisor cards: retain last known state
- No full-page error screen — inline indicators only
- Console.error logged, no user-visible stack trace

## Data Freshness Policy

| Data | Method | Interval |
|------|--------|----------|
| Agent list (supervisor cards, hero portraits) | React Query polling | 60s |
| Stat tiles (projects, agents, jobs, decisions, tokens) | React Query polling | 30s |
| Activity feed | SSE push | Real-time (event-driven) |
| HeroStats portraits | Derived from agent list query | Refreshes with agent list |

All queries use React Query (`useQuery` via TanStack Query v5). SSE managed with `useSSEActivity` custom hook using native `EventSource`. On SSE disconnect, hook retries with exponential backoff (1s → 2s → 4s → 8s cap).
