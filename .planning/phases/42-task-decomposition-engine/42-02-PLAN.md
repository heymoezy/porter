  <action>
    Create `backend/src/services/task-decomposition/dag-executor.ts`:

    1. **getTreeStats(rootId: string): Promise<DAGStats>**
       - Single query: `SELECT status, COUNT(*)::int AS cnt FROM task_nodes WHERE root_id = $1 AND depth > 0 GROUP BY status`
       - Map to DAGStats object (default 0 for missing statuses)

    2. **markReadyTasks(rootId: string): Promise<TaskNode[]>**
       - Use the ready-task query from research spec (lines 128-142):
         SELECT tasks where status='pending' AND all dependencies are 'completed'
       - UPDATE matched rows to status='ready'
       - Return the updated rows

    3. **dispatchSubtask(task: TaskNode): Promise<TaskResult>**
       - Load completed dependency results: query task_nodes WHERE id = ANY(task.dependencies) AND status='completed'
       - Build context string from dep results: `[dep.description]: dep.result`
       - Build prompt: `task.description + "\n\nContext from prior work:\n" + contextStr`
       - Use routingEngine.select() with context (agentId: task.assignedAgentId, projectId: task.projectId)
       - Dispatch via routingEngine.dispatchWithQueue()
       - Return TaskResult { response, model, tokens }

    4. **handleFailure(task: TaskNode, error: Error): Promise<void>**
       - If task.attempt < task.maxAttempts:
         - UPDATE task_nodes SET status='pending', attempt=attempt+1, error=error.message
         - Broadcast SSE: `task:retry` event
       - Else:
         - UPDATE task_nodes SET status='failed', error=error.message, completed_at=now
         - Broadcast SSE: `task:failed` event
         - Check tree health: if failed > total * 0.5, cancel entire tree (UPDATE all pending/ready to 'cancelled')

    5. **executeTaskTree(rootId: string): Promise<void>**
       - Main loop:
         a. Call markReadyTasks(rootId)
         b. Call getTreeStats(rootId) — break if pending=0 AND ready=0 AND running=0
         c. Batch ready tasks, cap at MAX_CONCURRENT (3)
         d. For each batch task: UPDATE status='running', started_at=now, broadcast SSE `task:started`
         e. `Promise.allSettled(batch.map(t => dispatchSubtask(t)))` — THIS IS THE PARALLEL DISPATCH
         f. For fulfilled: UPDATE status='completed', result=value, completed_at=now, broadcast `task:completed`, propagateResult to downstream context
         g. For rejected: call handleFailure
         h. Broadcast `decomposition:progress` with stats
         i. Safety: check TREE_TIMEOUT_MS — if exceeded, cancel remaining and break
       - After loop: update root node status based on tree outcome

    6. **propagateResult(taskId: string, result: TaskResult): Promise<void>**
       - Find all tasks with this taskId in their dependencies array
       - UPDATE their context JSONB to include this task's result (merge into existing context)
       - This makes dependency outputs available to downstream tasks

    Export: executeTaskTree, markReadyTasks, getTreeStats

    Import broadcast from '../../services/sse-hub.js' for SSE events.
    SSE event types: 'decomposition:started', 'task:started', 'task:completed', 'task:failed', 'task:retry', 'decomposition:progress'
