    Create `backend/src/services/task-decomposition/task-classifier.ts`:

    1. `classifyFast(message: string): 'simple' | 'complex' | 'uncertain'`
       - Count words: `message.split(/\s+/).length`
       - Check conjunctions: `/\b(and then|after that|also|plus|additionally|as well as)\b/i`
       - Check multi-step: `/\b(first|second|third|step \d|phase|stage|next)\b/i`
       - Check lists: `(message.match(/^[-*\d+\.]\s/gm) || []).length >= 2`
       - Check code blocks: `(message.match(/```/g) || []).length >= 2` (likely paste, not decompose)
       - Simple: words < 25 AND no conjunctions AND no multi-step AND no lists
       - Complex: words > 80 OR hasMultiStep OR hasList OR (words > 40 AND hasConjunctions)
       - Uncertain: everything else

    2. `classifyWithLLM(message: string): Promise<ClassificationResult>`
       - Use routingEngine to select cheapest gateway (forceGatewayType: 'ollama' preferred, fallback to any)
       - Dispatch with system prompt asking for JSON: `{"complexity": "simple"|"complex", "reason": "...", "estimated_subtasks": N}`
       - Parse JSON response, return ClassificationResult
       - On parse failure or dispatch error, default to 'simple' (fail-safe: never decompose by accident)
       - Wrap in try/catch — classifier errors must NEVER block normal chat flow

    3. `classify(message: string): Promise<ClassificationResult>`
       - Call classifyFast first
       - If 'simple': return `{ classification: 'simple', reason: 'Fast path: short direct question' }`
       - If 'complex': return `{ classification: 'complex', reason: 'Fast path: multi-step indicators detected' }`
       - If 'uncertain': call classifyWithLLM, return result
       - Export both classify and classifyFast (classifyFast for testing, classify for production)

    Use `import { routingEngine } from '../bridge/routing-engine.js'` for LLM fallback.
    Temperature 0 for deterministic classification. Max 200 tokens response.
