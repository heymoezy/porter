    **Part A: Create backend/src/routes/v1/decomposition.ts**

    Follow the exact route plugin pattern from tasks.ts.

    Endpoints:

    1. **GET /api/v1/decomposition** — List root-level decomposed tasks
       - Auth required (requireAuth)
       - Query params: `status` (optional filter), `limit` (default 50, max 200), `offset` (default 0)
       - Query: `SELECT id, root_id, description, status, created_at, started_at, completed_at, project_id, chat_id, result FROM task_nodes WHERE depth = 0 ORDER BY created_at DESC LIMIT $1 OFFSET $2`
       - Apply status filter if provided
       - For each root, include a subquery count: `(SELECT COUNT(*)::int FROM task_nodes WHERE root_id = t.id AND depth > 0) AS subtask_count`
       - Also: `(SELECT COUNT(*)::int FROM task_nodes WHERE root_id = t.id AND depth > 0 AND status = 'completed') AS completed_count`
       - Also: `(SELECT COUNT(*)::int FROM task_nodes WHERE root_id = t.id AND depth > 0 AND status = 'failed') AS failed_count`
       - Total count for pagination
       - Return via `ok({ roots, total })`

    2. **GET /api/v1/decomposition/:rootId/tree** — Get full DAG tree for a root task
       - Auth required
       - Query: `SELECT * FROM task_nodes WHERE root_id = $1 ORDER BY depth ASC, created_at ASC`
       - Return root node + child nodes separately:
         ```typescript
         const root = nodes.find(n => n.id === rootId);
         const children = nodes.filter(n => n.id !== rootId);
         ```
       - Include stats: `{ total, completed, failed, running, pending }`
       - Return via `ok({ root, children, stats })`
       - 404 if root not found

    3. **GET /api/v1/decomposition/:rootId/nodes/:nodeId** — Get single node detail
       - Auth required
       - Query: `SELECT * FROM task_nodes WHERE id = $1 AND root_id = $2`
       - Include dependency details: for each dep ID in dependencies[], fetch `SELECT id, description, status, result FROM task_nodes WHERE id = ANY($1)`
       - Return via `ok({ node, dependencies: depDetails })`
       - 404 if node not found

    For all endpoints:
    - Use `pool.query` (raw SQL) matching existing patterns
    - Use `ok()` and `err()` from `../../lib/envelope.js`
    - camelCase the response fields (use SQL AS aliases or JS mapping)
    - Map `created_at`, `started_at`, `completed_at` as epoch numbers
    - Parse `result`, `context`, `dependencies` as JSON objects (they're JSONB in PG, come as objects already)

    **Part B: Register route in index.ts**

    In `backend/src/routes/v1/index.ts`:
    - Add import: `import decompositionV1Routes from './decomposition.js';`
    - Add registration: `fastify.register(decompositionV1Routes, { prefix: '/decomposition' });`
    - Place after the tasks route registration (line ~58)
