  export interface RpgStats {

  **Component structure (build these subsections in order):**

  1. **Rarity border wrapper** — outer `<div>` with `className={\`border-2 rounded-xl \${RARITY_CLASS[rarity]}\`}`. RARITY_CLASS map: `{ common: 'rarity-common', rare: 'rarity-rare', epic: 'rarity-epic', legendary: 'rarity-legendary', mythic: 'rarity-mythic' }`.

  2. **Header row** — agent name, rarity Badge (text = rarity capitalized, color map: common=text3, rare=blue-400, epic=purple-400, legendary=yellow-400, mythic=red-400), Elo rating badge (`ELO ${elo}`), level badge (`LV ${level}`).

  3. **Level + XP bar** — `<Progress>` component showing `(xp % (level * 100)) / (level * 100) * 100` percent. Label: `"${xp % (level*100)} / ${level*100} XP"`. Use className `xp-bar-fill` on the indicator via `indicatorClassName` prop (shadcn Progress supports this).

  4. **Star display** — render 5 star icons. Stars <= current `stars` prop = filled yellow (`Star` icon, fill="currentColor", className="text-yellow-400"). Stars > current = empty gray (`Star` icon, strokeWidth=1, className="text-text3/40"). After the 5th slot, render a small SVG arc showing progress to the next star gate. Arc formula:
     ```
     const GATE = [0, 50, 200, 500, 1000]  // dispatch counts for each star
     const nextGate = GATE[Math.min(stars, 4)]
     const prevGate = GATE[Math.max(stars - 1, 0)]
     const progress = Math.min((dispatchCount - prevGate) / (nextGate - prevGate + 1), 1)
     const circumference = 2 * Math.PI * 10  // r=10 circle
     const dashoffset = circumference * (1 - progress)
     ```
     Render SVG (24x24): `<circle r=10 cx=12 cy=12 className="star-progress-track" strokeWidth=2>` (background) + `<circle r=10 cx=12 cy=12 className="star-progress-fill" strokeWidth=2 strokeDasharray={circumference} strokeDashoffset={dashoffset} transform="rotate(-90 12 12)">` (progress). If `stars === 5`, render a crown emoji instead of the arc.
     Below stars: `"${dispatchCount} dispatches"` in text-2xs text-text3.

  5. **Stat Pentagon (recharts RadarChart)** — 200px height container. Data:
     ```typescript
     const pentagonData = [
       { stat: 'QTY', value: rpg?.quality ?? 0, fullMark: 100 },
       { stat: 'SPD', value: rpg?.speed ?? 0, fullMark: 100 },
       { stat: 'EFF', value: rpg?.efficiency ?? 0, fullMark: 100 },
       { stat: 'REL', value: rpg?.reliability ?? 0, fullMark: 100 },
       { stat: 'COMBO', value: rpg?.combo ?? 0, fullMark: 100 },
     ]
     ```
     Use `<ResponsiveContainer width="100%" height={200}>`. Inside: `<RadarChart data={pentagonData} margin={{ top: 10, right: 20, bottom: 10, left: 20 }}>`. Children:
     - `<PolarGrid gridType="polygon" stroke="var(--border)" />`
     - `<PolarAngleAxis dataKey="stat" tick={{ fontSize: 10, fill: 'var(--text3)' }} />`
     - `<Radar name="stats" dataKey="value" stroke="var(--accent-porter)" fill="var(--accent-porter)" fillOpacity={0.25} dot={false} />`
     CRITICAL: If `rpg` is null, still render the RadarChart with all values=0. Never conditionally skip the chart.

  6. **Shell + Intelligence section** — two-column grid. Left: Shell badge (icon + shell name). Right: intelligence.primary_model (or "Unassigned"), temperature if set. Small text-xs labels.

  7. **Skills section** — label "SKILLS", then map workshop.skills (max 4 shown, or skill_slots count). Each skill row: `skill_id` as label, `success_rate_30d` as a mini Progress bar (0-100), total_uses count in text-2xs. If workshop is null, show "No workshop data" in text-2xs text-text3.

  8. **Supports section** — label "SUPPORTS", map workshop.supports. Each support: id label, measured_impact in text-2xs text-success. If empty: "None equipped".

  9. **Equipment section** — label "EQUIPMENT", map workshop.equipment_slots. Show each slot key + equipped value (or "Empty" in text-text3/40). Max 6 slots shown.

  10. **Specialties section** — map rpg.specialties (max 3). Each as a Badge variant="outline" with the label. If empty, omit section entirely.

  **Empty/null safety rules:**
  - Every section guards with `?.` and `?? []` defaults
  - If both `rpg` and `workshop` are null, render a skeleton card with muted placeholder text: "Forge this agent to unlock its character sheet"
  - Never throw on null/undefined data
