  **Part A — Create `admin/frontend/app/components/vitals-bar.tsx`:**

  Props:
  ```typescript
  export interface VitalsProps {
    templateId: string;   // used to scope the query key for future SSE invalidation
    reliability: number;  // 0-100 from RpgStats (rpg.reliability)
    dispatchCount: number; // from RpgStats, to estimate focus if no session data
  }
  ```

  The component fetches bridge capacity and derives 3 vitals:

  ```typescript
  const { data: capacityData } = useQuery({
    queryKey: ['bridge', 'capacity'],
    queryFn: () => api<{ gateways: GatewayCapacity[] }>('/api/admin/bridge/capacity'),
    staleTime: 30_000,
    refetchInterval: 60_000,
  })
  ```

  **Vital 1 — Tokens (blue):**
  Derivation: Find the `daily` token limit across all gateways. Sum current_value / limit_value. Invert to get "remaining": `tokenPct = 100 - (totalUsed / totalLimit * 100)`. Clamp 0-100. If no capacity data, default to 85.
  Color: `bg-blue-400` (good >= 50), `bg-yellow-400` (20-50), `bg-red-400` (< 20).
  Label: "Tokens" | value: `${Math.round(tokenPct)}%`

  **Vital 2 — Health (green/red):**
  Derivation: `healthPct = props.reliability` (already 0-100 from rpg-engine). 100 = zero errors.
  Color: `bg-green-400` (>= 80), `bg-yellow-400` (50-80), `bg-red-400` (< 50).
  Label: "Health" | value: `${Math.round(healthPct)}%`

  **Vital 3 — Focus (purple):**
  Derivation: This is context window pressure. Without live session data (session registry is Phase 29), derive a proxy: `focusPct = Math.max(0, 100 - (dispatchCount % 50) * 2)`. This estimates context fill based on dispatch recency within a 50-dispatch window. Clamp 0-100. Default 90 if no data.
  Color: `bg-purple-400` (>= 70), `bg-yellow-400` (40-70), `bg-red-400` (< 40).
  Label: "Focus" | value: `${Math.round(focusPct)}%`

  **Render each vital bar as:**
  ```tsx
  <div className="flex flex-col gap-0.5">
    <div className="flex items-center justify-between">
      <span className="text-2xs font-semibold uppercase tracking-wide text-text3">{label}</span>
      <span className="text-2xs font-mono text-foreground">{value}</span>
    </div>
    <div className="h-1.5 w-full rounded-full bg-raised overflow-hidden">
      <div
        className={`h-full rounded-full transition-all duration-700 ${colorClass}`}
        style={{ width: `${pct}%` }}
      />
    </div>
  </div>
  ```

  Wrap all 3 in a `<div className="flex flex-col gap-2">` with a section header `<span className="text-2xs font-bold uppercase tracking-widest text-text3">Vitals</span>`.

  Export: `export function VitalsBar({ templateId, reliability, dispatchCount }: VitalsProps)`

  Create `admin/frontend/app/components/passive-tree-view.tsx`.

  This is a compact visual representation of passive tree nodes. It does NOT need a full graph library — render it as a responsive grid of nodes connected by subtle lines.

  Props:
  ```typescript
  export interface PassiveTreeViewProps {
    nodes: Array<{ node_id: string; unlocked: boolean; active: boolean; unlocked_at?: string }>;
    agentLevel: number;
  }
  ```

  **Node display logic:**
  Each node renders as a small circle + label. The design doc defines 8 known nodes with unlock levels:

  ```typescript
  const NODE_DEFS: Record<string, { label: string; effect: string; unlockLevel: number }> = {
    'autonomous-routing':  { label: 'Auto Route',    effect: '+12% efficiency',    unlockLevel: 5 },
    'context-anchor':      { label: 'Ctx Anchor',    effect: 'No quality decay',   unlockLevel: 10 },
    'fail-fast-guard':     { label: 'Fail Fast',     effect: '-18% hallucination', unlockLevel: 15 },
    'cost-optimizer':      { label: 'Cost Opt',      effect: 'Cheapest viable',    unlockLevel: 20 },
    'deep-memory':         { label: 'Deep Recall',   effect: 'Full history',       unlockLevel: 25 },
    'team-synergy':        { label: 'Team Synergy',  effect: '+10% bonded',        unlockLevel: 30 },
    'battle-hardened':     { label: 'Hardened',      effect: '+5% all in arena',   unlockLevel: 0 },  // 10 battle wins
    'specialist-focus':    { label: 'Specialist',    effect: '+15% top domain',    unlockLevel: 0 },  // 50 domain dispatches
  }
  ```

  For nodes passed in `props.nodes`, look up NODE_DEFS by node_id. For nodes NOT in props.nodes (the remaining NODE_DEFS keys), treat as locked.

  **Render in a 4x2 grid** (`grid grid-cols-4 gap-2`):
  - Iterate NODE_DEFS keys in definition order
  - For each node, determine state: `active` (in props, active=true), `unlocked` (in props, unlocked=true), `locked` (not in props OR unlocked=false)
  - Active: filled circle `bg-accent-porter` with white label
  - Unlocked: filled circle `bg-raised border border-success` with normal label
  - Locked: `bg-raised/40 border border-border/30 opacity-40` with muted label

  Each node cell:
  ```tsx
  <div
    key={node_id}
    className={`flex flex-col items-center gap-0.5 p-1 rounded cursor-default ${nodeClass}`}
    title={`${def.label}: ${def.effect}${def.unlockLevel > 0 ? ` (Level ${def.unlockLevel})` : ''}`}
  >
    <div className={`size-3 rounded-full ${circleClass}`} />
    <span className="text-2xs text-center leading-tight">{def.label}</span>
  </div>
  ```

  **Connecting lines:** Between nodes, render subtle horizontal lines using `absolute` positioned `<div>` elements OR simply use a CSS border-bottom on the container row — keep it minimal. A 1px dashed border-text3/20 between nodes is sufficient. Do not use SVG for connections — too complex.

  If `nodes` is empty or undefined, render all 8 nodes in locked state (this is valid for new agents).
  If `nodes` contains node_ids not in NODE_DEFS, skip them without crashing.

  Section header: `<span className="text-2xs font-bold uppercase tracking-widest text-text3">Passive Tree</span>`
