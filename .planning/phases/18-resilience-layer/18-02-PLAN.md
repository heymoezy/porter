    Define `GatewayProbeRow` interface matching raw DB columns:
    ```typescript
    interface GatewayProbeRow {
      id: string;
      type: string;
      name: string;
      url: string | null;
      auth_method: string;

    }
    ```

    Export `async function runHealthProbe(): Promise<void>`:
    1. `SELECT * FROM gateways WHERE enabled = 1` — get all enabled gateways
    2. For each row, map to GatewayRow using same mapGatewayRow pattern from routing-engine.ts
    3. `const adapter = createAdapter(row)` — skip if null (unknown type)
    4. Try `const health = await adapter.health()` with a 10s timeout (AbortController)
    5. Determine new status:
       - `!health.healthy` -> `'unavailable'`
       - `health.healthy && health.latencyMs && health.latencyMs > 5000` -> `'stale'`
       - else -> `'active'`
    6. Get circuit state: `const circuitState = getBreakerState(row.id) ?? 'closed'`
    7. Update DB: `UPDATE gateways SET status = $1, circuit_state = $2, last_health_at = EXTRACT(EPOCH FROM NOW()) WHERE id = $3`
    8. If old status !== new status: `emitSSE('bridge:health', { gateway_id: row.id, gateway_type: row.type, old_status: oldStatus, new_status: newStatus, latency_ms: health.latencyMs ?? null }).catch(() => {})`
    9. Wrap each gateway's probe in try/catch so one failure does not stop other probes — log error with `console.error('[health-probe] gateway %s failed: %s', row.id, err.message)`

    Include the `mapGatewayRow` helper (private function, same logic as routing-engine.ts — map snake_case DB columns to camelCase GatewayRow fields).

    **Step 2: Wire into scheduler.ts**
    In `backend/src/services/scheduler.ts`:
    1. Add import at top: `import { runHealthProbe } from './bridge/health-probe.js';`
    2. Add constant: `const HEALTH_PROBE_INTERVAL = 15; // 15 × 2000ms = 30s`
    3. In the `tick()` function, after the existing `tickCount++` line and BEFORE the deadline check, add:
       ```typescript
       // Bridge health probe — every 30s (skip first interval to avoid startup thundering herd)
       if (tickCount > HEALTH_PROBE_INTERVAL && tickCount % HEALTH_PROBE_INTERVAL === 0) {
         runHealthProbe().catch(err => console.error('[scheduler] health probe error', err));
       }
       ```
       The `tickCount > HEALTH_PROBE_INTERVAL` guard skips the first 30s window to avoid probing before adapters stabilize after boot.
