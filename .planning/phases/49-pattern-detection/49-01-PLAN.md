**Step 1 — Add FRUSTRATION_REGEX constant** immediately after the existing IMPERATIVE_REGEX (around line 60). The regex set is verbatim from 49-RESEARCH.md → Frustration-Marker Patterns:

```typescript
const FRUSTRATION_REGEX = new RegExp(
  [
    // 1. "every (single) time" — canonical recurring-failure marker
    '\\bevery (single )?time\\b',
    // 2. "i told you" / "i keep telling you" / "i've told you" — explicit repetition complaint
    "i('ve| keep)? (told|telling|asked|asking|said) you",
    // 3. "you keep" / "you always" / "you never" — pattern-of-behavior complaint
    'you (keep|always|never|still|just) ',
    // 4. "stop doing" / "stop trying" / "stop using" — explicit stop-this-pattern
    'stop (doing|trying|using|making|asking|repeating|freehanding|reinventing)',
    // 5. "why are you" / "why do i" / "why did" — frustration interrogatives
    'why (are|do|did) (you|i|we) ',
    // 6. "still broken" / "still wrong" / "still missing" — recurrence-by-state
    "still (broken|wrong|missing|the same|not|doesn'?t)",
    // 7. ALL-CAPS RUN — 4+ consecutive uppercase words (overlap with imperative lane acceptable; same-turn dedupe via selected.has())
    '\\b[A-Z]{4,}(\\s+[A-Z]{2,}){2,}\\b',
    // 8. "the same mistake" / "the same fucking" — anaphoric reference to recurrence
    'the same (mistake|fucking|error|bug|problem|issue|thing)',
    // 9. "i have to repeat" / "do i have to" / "how many times" — meta-complaint about repetition cost
    '(i have to repeat|do i have to|how many times)',
  ].join('|'),
  'i',
);
```

NOTE: Use a JS string array + `.join('|')` so each marker is independently readable. The trailing `'i'` flag makes ALL markers case-insensitive — and yes, that means the ALL-CAPS RUN pattern matches lowercase too. That is INTENTIONAL: a complaint phrased in lowercase ("every single time") should match without requiring caps. The ALL-CAPS RUN pattern uses literal `[A-Z]` ranges so it still only fires on actually-uppercase runs even with the `i` flag — `[A-Z]` with /i becomes effectively `[A-Za-z]`. THIS IS A KNOWN OVERLAP with marker 7. Document in the regex comment that the marker 7 pattern's intent is to catch ALL-CAPS emphasis but with /i it also catches multi-word lowercase runs of letter-only tokens — acceptable false-positive rate per Risk 1 analysis (10% budget cap absorbs noise).

ALTERNATIVE if marker 7 false-positives become an issue: drop the `i` flag and switch the lowercase-friendly markers (1-6, 8, 9) to use their own per-pattern `(?i:...)` inline flags. Ship the simple `i`-flag version in v1; tune later if `frustration_forced_examples` shows persistent garbage matches.

```typescript
// ── 4a-prime. Pass A0 (NEW — LRN-01): frustration force-include @ 10% budget,
//             user-role only, recency-first. Inserted BEFORE the imperative
//             lane so a fresh frustration turn beats a long old "always X"
//             imperative when budgets compete via counters.total clamp below.
const frustrations = all
  .filter(t => t.role === 'user' && t.is_frustration)
  .sort((a, b) => b.captured_at.getTime() - a.captured_at.getTime());
const frustrationForcedExamples: number[] = [];
for (const t of frustrations) {
  if (counters.frustration_forced + t.byte_size > budgetFrustration) continue;
  if (counters.total + t.byte_size > maxBytes) continue;
  if (selected.has(t.id)) continue;
  selected.add(t.id);
  counters.frustration_forced += t.byte_size;
  counters.total += t.byte_size;
  if (frustrationForcedExamples.length < 3) frustrationForcedExamples.push(t.id);
}
```
