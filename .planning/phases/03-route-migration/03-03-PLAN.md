    **Step 1: Create backend/src/routes/v1/agents.ts**

    CRITICAL: Read the porter.py GET /api/personas handler FIRST to understand the exact JSON shape returned. The embedded frontend JS parses specific fields from this response. The Fastify route must return the same shape wrapped in the ok() envelope.

    The porter.py handler likely does something like:
    - Query all personas from DB
    - Parse the config JSON blob
    - Merge config fields into the top-level response object
    - Return a list of agent objects

    The agents route must replicate this shape. Key fields to include per agent:
    - id, name, role, status, agent_group
    - description (from config.description)
    - appearance_spec (parsed JSON), appearance_style
    - personality, communication_style, archetype
    - skills (from config.skills or empty array)
    - tools (from config.tools or empty array)
    - knowledge_domains (parsed or empty array)
    - sort_order
    - created_at, updated_at

    Routes:

    GET / — List agents:
    - Query: db.select().from(schema.personas).where(not equal to 'retired' status).orderBy(schema.personas.sortOrder).all()
    - Map each row: parse config JSON, appearance_spec JSON, knowledge_domains
    - Build response objects matching porter.py shape
    - Return ok({ agents: [...], count: N })

    POST / — Create agent:
    - Validate body: z.object({ name: z.string().min(1), role: z.string().optional(), agent_group: z.string().optional(), description: z.string().optional() })
    - Generate id: 'agent_' + crypto.randomUUID().replace(/-/g, '').slice(0, 12)
    - Build config JSON with description
    - Insert into personas table
    - Return reply.code(201).send(ok({ agent }))

    GET /:id — Get single agent:
    - Query by id, parse all JSON fields
    - If not found: reply.code(404).send(err('AGENT_NOT_FOUND', 'Agent not found'))
    - Return ok({ agent }) with full detail

    PUT /:id — Update agent:
    - Accept partial updates to name, role, description (stored in config), status
    - Merge description into existing config JSON (read-modify-write)
    - Set updatedAt
    - Return ok({ agent })

    DELETE /:id — Retire agent (soft delete by setting status to 'retired'):
    - Update status to 'retired', set updatedAt
    - Return ok({ retired: true })
