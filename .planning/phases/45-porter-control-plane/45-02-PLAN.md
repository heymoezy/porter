<objective>
Implement approval gates for high-risk actions in the Porter control plane.

Purpose: When Porter delegates a task that involves code mutation, external API calls, file deletion, or system configuration changes, the action must pause pending explicit user approval. This prevents autonomous execution of potentially destructive operations. The gate integrates into the delegation pipeline so that all agent-dispatched work passes through risk classification.

Output: approval_requests table, approval-gate.ts service, REST endpoints for approve/reject/list, agent-delegation.ts wired with pre-dispatch risk check.
</objective>

export async function migrateAjqV1(pool: pg.Pool): Promise<void> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const check = await client.query(`SELECT 1 FROM schema_migrations WHERE id = 'ajq_v1'`);
    if (check.rowCount && check.rowCount > 0) { await client.query('COMMIT'); return; }
    // ... DDL statements ...
    await client.query(`INSERT INTO schema_migrations (id) VALUES ('ajq_v1')`);
    await client.query('COMMIT');
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

**2. Create `backend/src/services/control-plane/approval-gate.ts`:**

This service classifies task risk and manages the approval lifecycle.

**Types to export:**

```typescript
export type RiskLevel = 'none' | 'low' | 'high';

export interface RiskAssessment {
  level: RiskLevel;
  reasons: string[];
}

export interface ApprovalRequest {
  id: string;
  correlationId: string | null;
  sourceAgent: string;
  targetAgent: string | null;
  task: string;
  riskLevel: string;
  riskReasons: string[];
  status: 'pending' | 'approved' | 'rejected' | 'expired';
  requestedAt: number;
  resolvedAt: number | null;
  resolvedBy: string | null;
  rejectionReason: string | null;
}
```

**Functions to export:**

`classifyRisk(task: string): RiskAssessment` — Pure synchronous function:

High-risk patterns (any match -> `level: 'high'`):
- Code mutation: `/\b(rm\s+-rf|delete\s+file|remove\s+file|unlink|write\s+file|overwrite|truncate|modify\s+source|edit\s+code|patch\s+file|git\s+push\s+--force|git\s+reset\s+--hard)\b/i`
- External API calls: `/\b(curl\s+.*-X\s+(POST|PUT|DELETE|PATCH)|fetch\s*\(|http(s)?:\/\/.*\/(api|webhook)|send\s+email|send\s+webhook|post\s+to\s+api|call\s+external)\b/i`
- File deletion: `/\b(delete|remove|drop|destroy|wipe|purge)\s+(file|directory|folder|database|table|schema|collection|bucket)\b/i`
- System config: `/\b(systemctl|service\s+(restart|stop)|kill\s+-9|chmod|chown|iptables|env\s+set|config\s+change|modify\s+config)\b/i`
- Dangerous operations: `/\b(DROP\s+TABLE|DROP\s+DATABASE|TRUNCATE\s+TABLE|DELETE\s+FROM\s+\w+\s*;|ALTER\s+TABLE.*DROP)\b/i`

Each matched pattern adds a descriptive string to the `reasons` array (e.g., "Code mutation detected: rm -rf", "External API call: POST request").

If no high-risk patterns match, return `{ level: 'none', reasons: [] }`.

`createApprovalRequest(opts: { task: string; correlationId?: string; sourceAgent?: string; targetAgent?: string; riskAssessment: RiskAssessment; delegationRequest?: DelegationRequest }): Promise<ApprovalRequest>` — Inserts a row into approval_requests with status='pending', logs to msg_bus_events with intent='approval_requested', returns the created request.

`approveRequest(id: string, approvedBy: string): Promise<ApprovalRequest | null>` — Updates status to 'approved', sets resolved_at and resolved_by. Returns null if not found or not pending. Logs to msg_bus_events with intent='approval_granted'.

`rejectRequest(id: string, rejectedBy: string, reason?: string): Promise<ApprovalRequest | null>` — Updates status to 'rejected', sets resolved_at, resolved_by, rejection_reason. Returns null if not found or not pending. Logs to msg_bus_events with intent='approval_rejected'.

`listPendingApprovals(): Promise<ApprovalRequest[]>` — Returns all rows with status='pending', ordered by requested_at DESC.

`getApprovalRequest(id: string): Promise<ApprovalRequest | null>` — Fetches a single request by id.

Import `pool` from `../../db/client.js`, `crypto` from `node:crypto`, `logMsgBusEvent` from `../msg-bus.js`. Use `crypto.randomUUID()` for id generation.

In `delegateToAgent()`, AFTER the depth enforcement check (added in Plan 01) and BEFORE Step 1 (msg_bus logging), add the risk gate:

```typescript
// PCP-03: Approval gate for high-risk actions
const riskAssessment = classifyRisk(opts.task);
if (riskAssessment.level === 'high') {
  // Create approval request and pause execution
  const approval = await createApprovalRequest({
    task: opts.task,
    correlationId: opts.correlationId,
    sourceAgent: source,
    targetAgent: opts.targetAgent,
    riskAssessment,
    delegationRequest: opts,
  });
  
  // Do NOT execute — throw a typed error the caller can handle
  const error = new Error(
    `Action requires approval (id=${approval.id}): ${riskAssessment.reasons.join(', ')}`
  );
  (error as any).code = 'APPROVAL_REQUIRED';
  (error as any).approvalId = approval.id;
  (error as any).riskReasons = riskAssessment.reasons;
  throw error;
}
```

This ensures the delegation is blocked until approval. The DAG executor's handleFailure will catch this error. The user can approve via the REST endpoint, and the action can be re-queued.

Note: We are using a typed error with `.code = 'APPROVAL_REQUIRED'` so callers can distinguish approval blocks from real failures. The DAG executor already has retry logic that will handle this gracefully (it will mark the task as failed with the approval message, and the user can re-trigger after approval).
