**2. Create `backend/src/services/control-plane/delegation-doctrine.ts`:**

This is the core decision engine. It takes a message and optional context, and returns which strategy Porter should use.

Types to export:
```typescript
export type DispatchStrategy = 'direct' | 'delegate' | 'parallel' | 'escalate';

export interface DoctrineDecision {
  strategy: DispatchStrategy;
  reason: string;
  /** Whether the classifier was consulted (false for fast-path decisions) */
  classifierUsed: boolean;
}
```

Export function `decideDoctrine(message: string, opts?: { hasAgent?: boolean; isDecompositionContext?: boolean }): DoctrineDecision`:

Decision logic (pure synchronous, no LLM calls — mirrors classifyFast pattern):

1. If `opts.isDecompositionContext` is true, return `{ strategy: 'direct', reason: 'Already in decomposition context' }` — subtasks always execute directly.

2. Use classifyFast(message) from task-classifier.ts:
   - `'simple'` -> `{ strategy: 'direct', reason: 'Simple request — direct answer' }`
   - `'complex'` -> `{ strategy: 'delegate', reason: 'Complex request — decomposition' }`
   - `'uncertain'` -> Check additional heuristics:
     - If message contains question words (what, how, why, when, where, who, which, is, are, can, do, does, will, should) as the first word -> `{ strategy: 'direct', reason: 'Question — direct answer' }`
     - If message length > 200 chars AND contains action verbs (create, build, implement, design, analyze, compare, migrate, refactor, deploy, set up, configure) -> `{ strategy: 'delegate', reason: 'Action request with detail — decomposition' }`
     - Otherwise -> `{ strategy: 'escalate', reason: 'Ambiguous intent — requesting clarification' }`

3. Set `classifierUsed: true` in all cases (since classifyFast is always called).

This is a pure function with no side effects. The caller is responsible for logging and routing.
