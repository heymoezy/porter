## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding, badge padding |
| sm | 8px | Compact element spacing, icon-label gap |
| sm+ | 12px | Nav item vertical padding (py-3) |
| md | 16px | Default element spacing, card padding, input padding |
| lg | 24px | Section padding, modal header padding |
| xl | 32px | Layout gaps, panel padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Sidebar width: 220px (--sidebar token, existing) when expanded; 56px when collapsed
- Minimum touch target for icon-only nav items (collapsed sidebar): 44px height
- Nav item vertical padding: py-3 (12px top + 12px bottom) — replaces previous py-2.5 (10px is not a multiple of 4)

## Typography

System font stack. No web font loading — avoids dependency and matches product character (fast, no FOUT).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Main content, nav labels, descriptions |
| Label | 12px | 600 | 1.3 | Badges, chips, metadata, timestamps, version string |
| Heading | 16px | 600 | 1.3 | Section headers, modal titles, panel titles |
| Display | 20px | 600 | 1.2 | Page titles, wizard headings, landing headline |

**Weight vocabulary: 400 (regular) and 600 (semibold) only.** No intermediate weights.

**Letter spacing:**
- Nav labels, version string, and logo wordmark: `tracking-widest` (0.1em) — matches existing Sidebar.tsx `.tracking-widest`
- All other text: default (0)

**60/30/10 split (dark mode):**
- 60% dominant: `--bg` (#111827) — all page backgrounds, empty areas
- 30% secondary: `--surface` + `--raised` (#1E2736, #28344A) — nav sidebar, cards, panels, modals
- 10% accent: `--accent` (#6366F1) — see reserved list below

### Accent Reserved For

Accent (`--accent` / `#6366F1`) is reserved exclusively for:
1. Active nav item left-border indicator (2px right border on sidebar active item)
2. Active nav item background highlight
3. Primary CTA buttons (e.g. "Start setup", "Save Profile", "Change Password")
4. Form focus rings (outline on focused inputs/textareas)
5. Selected tab underline indicator
6. Toggle switch active state (theme toggle, feature flags)
7. Boot sequence step completion checkmark
8. Login page primary action button

Accent is NOT used on: body text, icons in default state, borders in default state, nav item icons, card titles.

### Danger Reserved For

Danger (`--danger`) is reserved exclusively for:
1. Delete confirmation buttons
2. Destructive action text labels
3. Error state messages
4. Form validation error borders

## CSS Variable Architecture

**Single source of truth: `:root`**

This is the contract for `frontend/src/index.css` (to be rewritten in Phase 1):

```css
/* SINGLE SOURCE: :root defines all design tokens */
:root {
  /* Surfaces */
  --bg: #111827;
  --surface: #1E2736;
  --raised: #28344A;
  --border: #374259;
  --border2: #4A5770;

  /* Text */
  --text: #F1F5F9;
  --text2: #94A3B8;
  --text3: #64748B;

  /* Brand */
  --accent: #6366F1;
  --accent-h: #818CF8;

  /* Semantic */
  --danger: #EF4444;
  --success: #22C55E;
  --warning: #F59E0B;

  /* Layout */
  --radius: 8px;
  --sidebar: 220px;
}

/* Light mode — explicit override */
[data-theme="light"] {
  --bg: #FFFFFF;
  --surface: #F3F4F8;
  --raised: #E8EBF2;
  --border: #D1D5DF;
  --border2: #B8BFD0;
  --text: #0F172A;
  --text2: #475569;
  --text3: #94A3B8;
  --accent: #4F46E5;
  --accent-h: #4338CA;
  --danger: #DC2626;
  --success: #16A34A;
  --warning: #D97706;
}

/* System preference — default before any explicit override */
@media (prefers-color-scheme: light) {
  :root:not([data-theme]) {
    --bg: #FFFFFF;
    --surface: #F3F4F8;
    --raised: #E8EBF2;
    --border: #D1D5DF;
    --border2: #B8BFD0;
    --text: #0F172A;
    --text2: #475569;
    --text3: #94A3B8;
    --accent: #4F46E5;
    --accent-h: #4338CA;
    --danger: #DC2626;
    --success: #16A34A;
    --warning: #D97706;
  }
}

/* Tailwind 4 @theme reads FROM :root — no independent values */
@theme {
  --color-accent: var(--accent);
  --color-accent-h: var(--accent-h);
  --color-bg: var(--bg);
  --color-surface: var(--surface);
  --color-raised: var(--raised);
  --color-border: var(--border);
  --color-border2: var(--border2);
  --color-text: var(--text);
  --color-text2: var(--text2);
  --color-text3: var(--text3);
  --color-danger: var(--danger);
  --color-success: var(--success);
  --color-warning: var(--warning);
  --radius-porter: var(--radius);
}
```

**Embedded HTML pages in porter.py:** Each embedded page (`<style>` block) must inline the full `:root { ... }` variable block plus the `@media (prefers-color-scheme: light)` block. The three-state JS toggle (system/dark/light) applies only after the React app loads. Pre-auth pages (login, register, landing) use system preference only — no JS override.

**Three-state toggle:**
- States: `system` (default) | `dark` (forced) | `light` (forced)
- Storage: `localStorage` key `porter_theme`
- Application: `data-theme` attribute on `<html>` element
- `system` = remove `data-theme` attribute entirely

## Component Patterns

These are the interaction patterns for Phase 1 views. No new components are introduced — these define how existing patterns must behave after the CSS audit.

### Nav Sidebar (Sidebar.tsx)

- Active item: `background: var(--surface)`, 2px right border `var(--accent)`, text `var(--text)`
- Inactive item: text `var(--text3)`, hover text `var(--text2)`, hover background `var(--surface)` at 40% opacity
- Icon size: 16px (w-4 h-4) — do not change
- Nav item vertical padding: py-3 (12px) — on all sidebar nav items
- Collapsed width: 56px (matches 44px touch target + 6px padding each side)
- No hardcoded `bg-neutral-*` or `border-neutral-*` classes — replace with CSS variable references or Tailwind color tokens mapped to variables

### Theme Toggle

- Placed in sidebar footer area (below nav items, above version string)
- Three-state cycle: system → dark → light → system
- Icon representations: Monitor (system), Moon (dark), Sun (light) — from lucide-react
- Size: 16px icon, 32px touch target
- No label — icon only, with tooltip on hover

### Landing Page Placeholder

- Full viewport: centered wordmark + tagline + single login button
- Background: `var(--bg)`
- Wordmark: configurable product name from config, 20px, weight 600, `var(--text)`
- Tagline: "Your autonomous agent platform", 14px, `var(--text2)`
- Login button: accent background, white text, 8px radius, 16px horizontal padding, 12px vertical padding
- No illustrations, no hero images, no animations

### Login / Register Pages

- Centered card: max-width 400px, `var(--surface)` background, 8px radius, 24px padding
- Input fields: `var(--raised)` background, `var(--border)` border, 8px radius, 14px text
- Focus ring: 1px `var(--accent)` outline, no box-shadow glow
- Submit button: full width, `var(--accent)` background, `var(--text)` color (white in dark), 8px radius
- Error message: `var(--danger)` text, 12px, below the errored field

### Boot Sequence Status

- Full-page overlay or dedicated `/setup` route (Claude's discretion — both acceptable)
- Step list: each step shows icon + label + status chip
- Status chips: "Checking..." (`var(--text3)`), "OK" (`var(--success)`), "Missing" (`var(--warning)`), "Failed" (`var(--danger)`)
- Progress indicator: simple vertical step list, no animated progress bars
- Credential prompt: appears inline below the step that requires input

## Interaction States

Every interactive element must handle these states:

| State | Treatment |
|-------|-----------|
| Default | Base `var(--text2)` for secondary elements, `var(--text)` for primary |
| Hover | Lighten background by one step (e.g. `var(--raised)` → `var(--border)` tint), text to `var(--text)` |
| Active / Pressed | `var(--accent)` left/right border, `var(--surface)` background |
| Focus | 1px `var(--accent)` outline, 2px offset |
| Disabled | `var(--text3)` text, `var(--border)` background, `cursor: not-allowed` |
| Error | `var(--danger)` border, error text below element |
| Loading | Opacity 0.5 on the triggering element, no separate spinner for inline states |
