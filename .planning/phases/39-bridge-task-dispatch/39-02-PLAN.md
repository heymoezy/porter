**In-memory task tracking:** Maintain a `Map<string, AbortController>` at module level (`runningTasks`) for cancellation support.

**POST /dispatch endpoint:**
- Validate body: prompt (string, required, max 50_000 chars), cwd (string, required), gateway (string, optional — must be in TASK_CAPABLE_TYPES if provided), agent_id (string, optional), project_id (string, optional), timeout_ms (number, optional, 60_000-600_000 range)
- Call validateCwd(body.cwd) — catch and return 400 with INVALID_CWD code
- If gateway specified: validate it's in TASK_CAPABLE_TYPES, look up the gateway row from DB (`SELECT * FROM gateways WHERE type = $1 AND status = 'active' AND enabled = 1`), create adapter via createAdapter()
- If gateway not specified ('auto'): query all active+enabled gateways, filter to TASK_CAPABLE_TYPES, pick highest priority one
- Extract binaryPath from gateway metadata (adapter.row.metadata.binary_path or default binary name)
- Create task row: INSERT INTO bridge_tasks (id, gateway_type, model_name, prompt, cwd, status, username, agent_id, project_id, created_at) VALUES (...)
- Create AbortController, store in runningTasks Map
- Call runTaskInBackground(taskId, gatewayType, binaryPath, body, abortController) — do NOT await
- Return 202: { task_id, status: 'queued', gateway_type, model }

**runTaskInBackground(taskId, gatewayType, binaryPath, body, ac) async function:**
- Update status to 'running', set started_at: `UPDATE bridge_tasks SET status = 'running', started_at = EXTRACT(EPOCH FROM NOW()) WHERE id = $1`
- Broadcast SSE: broadcast('bridge:task-progress', { task_id: taskId, type: 'started', gateway_type: gatewayType })
- Use getTaskQueue(gatewayType).add() to run the actual execution through the per-gateway task queue (concurrency 1)
- Inside the queue callback: iterate executeTask() async generator
  - For each TaskEvent with type='progress': broadcast('bridge:task-progress', { task_id: taskId, type: 'progress', text: event.text, gateway_type: gatewayType })
  - Accumulate output text
  - For type='result': broadcast('bridge:task-complete', { task_id: taskId, status: event.exitCode === 0 ? 'complete' : 'failed', exit_code: event.exitCode, duration_ms: event.durationMs })
- After generator completes: UPDATE bridge_tasks SET status, output, exit_code, completed_at, duration_ms
- On error: UPDATE bridge_tasks SET status='failed', error=errorMessage, completed_at, duration_ms
- In finally: remove taskId from runningTasks Map, broadcast bridge:task-complete if not already

**GET /:id endpoint:**
- SELECT * FROM bridge_tasks WHERE id = $1
- Return 404 if not found
- Return ok({ id, status, gateway_type, model_name, prompt, cwd, output, error, exit_code, started_at, completed_at, duration_ms, created_at })

**DELETE /:id/cancel endpoint:**
- SELECT status FROM bridge_tasks WHERE id = $1
- If not found: 404
- If status not 'running' and not 'queued': return 409 err('TASK_NOT_CANCELLABLE', 'Task is not running or queued')
- Call runningTasks.get(taskId)?.abort()
- UPDATE bridge_tasks SET status = 'cancelled', completed_at = EXTRACT(EPOCH FROM NOW())
- Broadcast SSE: broadcast('bridge:task-complete', { task_id: taskId, status: 'cancelled' })
- Return ok({ task_id, status: 'cancelled' })

**GET / endpoint (list tasks):**
- Query params: status (optional filter), gateway_type (optional filter), limit (default 50, max 200), offset (default 0)
- SELECT * FROM bridge_tasks WHERE (status filter) AND (gateway_type filter) ORDER BY created_at DESC LIMIT $1 OFFSET $2
- Return ok({ tasks: [...], total: count })
