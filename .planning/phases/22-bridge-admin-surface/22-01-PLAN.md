**GET `/` (ADM-01 — gateway dashboard cards):**
```sql
SELECT g.id, g.type, g.name, g.url, g.auth_method, g.status,
       g.source, g.priority, g.capabilities, g.metadata,
       g.enabled, g.last_health_at, g.circuit_state,
       COUNT(m.id) FILTER (WHERE m.is_active = 1) AS model_count
FROM gateways g
LEFT JOIN models m ON m.gateway_id = g.id
GROUP BY g.id
ORDER BY g.priority ASC, g.created_at ASC
```
Import `getBreakerState` from `../../../services/bridge/circuit-breaker-registry.js`. Map each row through `maskGatewayRow()`, then add:
- `model_count: parseInt(gw.model_count) || 0`
- `circuit_state: getBreakerState(gw.id) ?? gw.circuit_state ?? 'closed'`
- `status_indicator`: derive from status field — 'active' maps to 'healthy', 'stale' maps to 'degraded', 'unavailable' maps to 'unavailable', else 'unknown' (DS-02)
- `briefing_slot: null` (DS-02 — reserved for future Bridge agent narratives)

Add summary object (DS-01/DS-02):
```typescript
const summary = {
  total_gateways: gateways.length,
  healthy: gateways.filter(g => g.status_indicator === 'healthy').length,

  unavailable: gateways.filter(g => g.status_indicator === 'unavailable').length,
  last_activity: Math.max(...gateways.map(g => g.last_health_at || 0)) || null,

```
Return `ok({ gateways, summary })`.

**GET `/models` (ADM-02 — unified model catalog):**
Accept optional query params: `gateway_id` (filter by gateway), `capability` (filter by capability tag).
```sql
SELECT m.*, g.name AS gateway_name, g.type AS gateway_type, g.status AS gateway_status
FROM models m
JOIN gateways g ON g.id = m.gateway_id
WHERE m.is_active = 1
  [AND m.gateway_id = $N if gateway_id provided]
ORDER BY g.priority ASC, m.model_name ASC
```
If `capability` query param provided, filter rows in JS: `rows.filter(r => (r.capabilities || []).includes(capability))`.
Add summary: `{ total_models: models.length, by_gateway: Object.groupBy(models, m => m.gateway_type) counted }`.
Return `ok({ models, summary })`.

**GET `/dispatch-log` (ADM-03 — paginated routing decisions):**
Pagination: `page` (default 1), `limit` (default 50, max 100). Calculate `offset = (page - 1) * limit`.
Optional filters: `gateway_type`, `model_name`, `agent_id` as query params. Build WHERE clause dynamically.
```sql
SELECT id, gateway_id, gateway_type, model_name, chosen_reason,
       alternatives, estimated_cost_usd, input_tokens, output_tokens,
       cached_tokens, latency_ms, agent_id, project_id, chat_id, rule_id, created_at
FROM bridge_dispatch_log
[WHERE clauses]
ORDER BY created_at DESC
LIMIT $N OFFSET $M
```
Count query: `SELECT COUNT(*) as total FROM bridge_dispatch_log [WHERE clauses]`.
Return `ok({ entries: rows, pagination: { page, limit, total, pages: Math.ceil(total / limit) } })`.

**GET `/costs` (ADM-04 — cost analytics):**
Accept `from` and `to` query params as epoch seconds (floats). Default: `from` = 30 days ago, `to` = now.
Three aggregation queries using `COALESCE(SUM(estimated_cost_usd), 0)` (Pitfall 4):

1. By gateway: GROUP BY gateway_type — return total_cost_usd, dispatch_count, input_tokens, output_tokens
2. By model: GROUP BY model_name, gateway_type — return total_cost_usd, dispatch_count
3. By day: GROUP BY TO_CHAR(TO_TIMESTAMP(created_at), 'YYYY-MM-DD') — return total_cost_usd, dispatch_count

Add summary: `{ total_cost_usd: sum of byGateway totals, total_dispatches: sum of byGateway counts, period_days: Math.ceil((to - from) / 86400) }`.
Return `ok({ byGateway, byModel, byDay, summary, range: { from, to } })`.
