## Decisions Made

- **OLD.source_type early-return:** The trigger function returns OLD/NEW without further checks when `OLD.source_type IS DISTINCT FROM 'moe-direct'`. This is the explicit fix for Risk 6 in RESEARCH.md — memory-pruner's `UPDATE directives SET status='superseded'` on `source_type='system'` rows continues working without modification.
- **Bypass via SET LOCAL, not GUC at session start:** `current_setting('porter.allow_moe_direct_mutation', true)` reads a per-tx setting that auto-clears on COMMIT/ROLLBACK. This forces every legitimate admin mutation to opt in explicitly per tx, eliminating accidental global bypass.
- **Boolean type, not integer 0/1:** schema.ts uses Drizzle's `boolean('enabled')` and the migration uses raw `BOOLEAN NOT NULL DEFAULT TRUE`. Consistent with CONTEXT.md schema lock and idiomatic Postgres.
