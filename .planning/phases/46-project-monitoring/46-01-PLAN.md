1. Create `backend/src/services/watcher-service.ts`:

   Export types:
   ```typescript
   export interface WatcherConfig {
     id: string;
     project_id: string;
     name: string;
     watcher_type: 'web_search' | 'rss_feed' | 'email_monitor' | 'custom';
     schedule_interval_sec: number;
     config: Record<string, unknown>;
     notify_email: string | null;
   }

   export interface WatcherFinding {
     title: string;
     summary: string;
     detail: Record<string, unknown>;
     importance: 'normal' | 'important' | 'critical';
   }
   ```

   Export `executeWatcher(watcherId: string, jobId: string): Promise<void>`:
   - Load watcher config from `project_watchers WHERE id = watcherId`
   - If not found or status != 'active', return early
   - Dispatch to type handler based on `watcher_type`:
     - `web_search`: call `executeWebSearch(config)` 
     - `rss_feed`: call `executeRssFeed(config)`
     - `email_monitor`: call `executeEmailMonitor(config)`
     - `custom`: call `executeCustom(config)`
   - For each finding returned by the handler:
     - INSERT INTO watcher_findings (id, watcher_id, project_id, source_type, title, summary, detail, importance, job_id)
   - UPDATE project_watchers SET last_run_at = NOW(), run_count = run_count + 1, finding_count = finding_count + {N}, next_run_at = NOW() + schedule_interval_sec, last_error = NULL
   - On error: UPDATE project_watchers SET last_error = error.message, status = (if attempt > 3 then 'error' else keep 'active')

   Implement type handlers (all return `WatcherFinding[]`):

   a) `executeWebSearch(config)`:
      - config.query (string) = search query
      - config.max_results (number, default 5)
      - Use Brave Search API if BRAVE_API_KEY is set: `fetch('https://api.search.brave.com/res/v1/web/search?q=...')`
      - If no API key, return single finding: { title: 'Web search unavailable', summary: 'Configure BRAVE_API_KEY to enable web search watchers', detail: {}, importance: 'normal' }
      - Parse results into WatcherFinding[] with title, summary (snippet), detail: { url, published_date, source }

   b) `executeRssFeed(config)`:
      - config.feed_url (string) = RSS/Atom feed URL
      - config.max_items (number, default 10)
      - config.since_hours (number, default 24) = only items newer than this
      - Fetch feed URL with fetch(), parse XML manually (DOMParser not available in Node — use regex to extract <item>/<entry> tags)
      - For each item: extract title, link, description, pubDate
      - Filter items newer than since_hours
      - Return WatcherFinding[] with title, summary (description truncated to 300 chars), detail: { url: link, published: pubDate, raw_description }

   c) `executeEmailMonitor(config)`:
      - config.filter (string) = keyword filter
      - config.hours_back (number, default 24)
      - Query email_messages table: `SELECT * FROM email_messages WHERE created_at > NOW() - hours_back AND (subject ILIKE '%filter%' OR body_text ILIKE '%filter%') ORDER BY created_at DESC LIMIT 20`
      - Return WatcherFinding[] with title: email subject, summary: first 200 chars of body, detail: { from, to, message_id }

   d) `executeCustom(config)`:
      - config.prompt (string) = freeform prompt describing what to watch
      - config.data_source (string, optional) = URL to fetch for context
      - If data_source is set, fetch it and prepend content to prompt
      - Dispatch to Ollama (cheap local model) via a direct fetch to http://127.0.0.1:11434/api/generate with model 'qwen2.5-coder:1.5b'
      - Parse LLM response, try to extract JSON array of findings; if not JSON, wrap as single finding
      - Return WatcherFinding[] with importance based on LLM output (default 'normal')

   Export `scheduleWatcherRuns(): Promise<void>`:
   - Query `SELECT * FROM project_watchers WHERE status = 'active' AND (next_run_at IS NULL OR next_run_at <= EXTRACT(EPOCH FROM NOW()))`
   - For each due watcher: call scheduleSystemJob but with custom insert (not scheduleSystemJob since that's for system source):
     ```sql
     INSERT INTO agent_jobs (id, agent_id, project_id, trigger_type, trigger_data, source, status, scheduled_for, created_at)
     VALUES (uuid, 'system', watcher.project_id, 'watcher_run', '{"watcher_id":"..."}', 'watcher', 'pending', NOW(), NOW())
     ```
   - Dedup guard: skip if pending/running watcher_run job exists for this watcher_id
   - Update next_run_at on the watcher to now + schedule_interval_sec

2. Wire into `backend/src/services/scheduler.ts`:

   a) Add import: `import { executeWatcher, scheduleWatcherRuns } from './watcher-service.js';`
   
   b) Add constant: `const WATCHER_SCHEDULE_INTERVAL = 30; // 30 ticks x 2s = 60s — check for due watchers every minute`

   c) In `tick()`, after the GATEWAY_CHECK_INTERVAL block (around line 374), add:
      ```typescript
      // PMN: Schedule due watcher runs every 60s
      if (tickCount > 0 && tickCount % WATCHER_SCHEDULE_INTERVAL === 0) {
        scheduleWatcherRuns().catch(err =>
          console.error('[scheduler:watcher] schedule error', err));
      }
      ```

   d) In `executeJob()` function, add a new case in the trigger_type dispatch (before the generic fallback):
      ```typescript
      if (job.trigger_type === 'watcher_run') {
        const data = (typeof job.trigger_data === 'string' ? JSON.parse(job.trigger_data) : job.trigger_data) as { watcher_id?: string };
        if (!data.watcher_id) {
          await markJobFailed(job.id, 'Missing watcher_id in watcher_run trigger_data');
          return;
        }
        try {
          await executeWatcher(data.watcher_id, job.id);
          await markJobComplete(job.id, JSON.stringify({ watcher_id: data.watcher_id, status: 'ok' }));
          await logActivity('system', job.id, job.project_id, 'watcher_complete',
            `Watcher ${data.watcher_id.slice(0, 8)} completed`, '{}');
        } catch (e) {
          const msg = e instanceof Error ? e.message : String(e);
          await markJobFailed(job.id, msg);
          await logActivity('system', job.id, job.project_id, 'watcher_failed',
            `Watcher ${data.watcher_id.slice(0, 8)} failed: ${msg.slice(0, 200)}`, '{}');
        }
        return;
      }
      ```
