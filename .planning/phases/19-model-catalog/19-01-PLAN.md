       - `gpt-5.4`: capabilities=['coding','analysis'], contextWindow=128000, pricingInputPerM=null, pricingOutputPerM=null
       - `gemini-2.5-pro`: capabilities=['coding','writing','analysis','vision'], contextWindow=1000000, pricingInputPerM=1.25, pricingOutputPerM=10.0
       - `gemini-2.5-flash`: capabilities=['coding','writing','analysis'], contextWindow=1000000, pricingInputPerM=0.075, pricingOutputPerM=0.30
       - `qwen2.5-coder:1.5b`: capabilities=['coding'], contextWindow=32768, pricingInputPerM=0.0, pricingOutputPerM=0.0
       - Default for unknown Ollama models: capabilities=['chat'], contextWindow=32768, pricingInputPerM=0.0, pricingOutputPerM=0.0

       Helper: `function lookupMetadata(modelName: string)` — exact match first, then prefix match (e.g., `claude-sonnet` matches `claude-sonnet-4-6`), then default for Ollama-style names, then bare default.

    2. **refreshModelsForGateway(pool, gatewayId, gatewayType, adapter)** — async function:
       - Call `adapter.listModels()` (wrapped in try/catch — log error and return on failure)
       - For each model name returned:
         a. Look up metadata via `lookupMetadata(modelName)`
         b. Check if model already exists: `SELECT id, capabilities, context_window FROM models WHERE gateway_id = $1 AND model_name = $2`
         c. If not exists: INSERT with crypto.randomUUID() id, enriched metadata. Also INSERT into model_versions with version_label='initial' and snapshot of the full row as JSONB.
         d. If exists: compare capabilities (JSON.stringify sorted arrays) and context_window. If changed, INSERT new model_versions row with version_label = new Date().toISOString() and snapshot of new values. UPDATE the models row.
         e. If exists and unchanged: UPDATE only updated_at timestamp.
       - After processing all returned models: mark models in DB for this gateway that were NOT in the listModels() response as is_active=0 (only if the gateway status is 'active', not if gateway is stale)
       - Use ON CONFLICT (gateway_id, model_name) DO UPDATE for the upsert pattern

    3. **refreshAllGateways(pool)** — async function:
       - Query: `SELECT id, type, name, url, auth_method, status, source, priority, capabilities, metadata, enabled, masked_display, created_at, updated_at, last_health_at FROM gateways WHERE status IN ('active', 'stale') AND enabled = 1`
       - Map each DB row to a GatewayRow using the same mapGatewayRow pattern from routing-engine.ts (inline helper or import)
       - For each gateway row, call createAdapter(row) from adapters/index.ts
       - If adapter is not null, call refreshModelsForGateway(pool, row.id, row.type, adapter)
       - Wrap each gateway refresh in try/catch — one failing gateway must not block others
       - Log: `[model-catalog] Refreshed models for {gateway.name}: {count} models`

    4. **calculateCostUsd(inputTokens, outputTokens, cachedTokens, modelName, gatewayId, pool)** — async function:
       - If (!inputTokens && !outputTokens) return null
       - Query: `SELECT pricing_input_per_m, pricing_output_per_m FROM models WHERE model_name = $1 AND gateway_id = $2 AND is_active = 1 LIMIT 1` (pass gatewayId to avoid cross-gateway ambiguity per Pitfall 3)
       - If no row or both pricing null: try fallback without gateway filter: `WHERE model_name = $1 AND is_active = 1 ORDER BY updated_at DESC LIMIT 1`
       - If still no pricing: return null
       - inputCost = (inputTokens / 1_000_000) * pricingInputPerM
       - outputCost = (outputTokens / 1_000_000) * pricingOutputPerM
       - cachedCost = (cachedTokens / 1_000_000) * (pricingInputPerM * 0.1)  — cached tokens billed at 10% of input price
       - Return inputCost + outputCost + cachedCost

    Import pg from 'pg', crypto from 'node:crypto', createAdapter from './adapters/index.js', and types from './types.js'. Use pool parameter (not import from db/client.ts) since this may be called from startup-detector.ts context.
