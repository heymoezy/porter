The endpoint:

Implementation steps:

1. Fetch RPG fields from agent_templates:
```sql
SELECT id, name, star_level, level, xp, rarity, shell, intelligence, supports, equipment_slots, passive_tree, elo_rating
FROM agent_templates
WHERE id = $1
```

2. Fetch skills with RPG stats from template_skills:
```sql
SELECT ts.skill_id, ts.sort_order, ts.success_rate_30d, ts.total_uses, ts.last_used
FROM template_skills ts
WHERE ts.template_id = $1
ORDER BY ts.sort_order ASC, ts.skill_id ASC
```

3. Compute skill_slots = 4 + (starLevel - 1), clamped min 4.

4. Parse JSONB fields using parseJson (already defined in the file) — supports, equipment_slots, passive_tree, intelligence all need parseJson with [] or {} fallbacks.

5. Return shape:
```typescript
return ok({
  id: row.id,
  name: row.name,
  star_level: row.star_level ?? 1,
  level: row.level ?? 1,
  xp: row.xp ?? 0,
  rarity: row.rarity ?? 'common',
  shell: row.shell ?? 'builder',
  elo_rating: row.elo_rating ?? 1200,
  intelligence: parseJson(row.intelligence, {}),
  supports: parseJson<SupportEntry[]>(row.supports, []),
  equipment_slots: parseJson(row.equipment_slots, []),
  passive_tree: parseJson(row.passive_tree, []),
  skill_slots: Math.max(4, 4 + ((row.star_level ?? 1) - 1)),
  skills: skills.map(s => ({
    skill_id: s.skill_id,
    sort_order: s.sort_order ?? 0,
    success_rate_30d: s.success_rate_30d ?? 0,
    total_uses: s.total_uses ?? 0,
    last_used: s.last_used ?? null,
  })),
})
```
