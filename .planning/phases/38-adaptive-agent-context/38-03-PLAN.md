# Plan 38-03: Context Pressure Observability

**Phase:** 38 — Adaptive Agent Context
**Requirements:** ACX-05
**Depends on:** Plan 38-01 (directive stats), Plan 38-02 (compression stats)

## Goal

Every dispatch logs context pressure metrics — tokens used, turns elapsed, compression events, directive selection stats — and admin can see this data on dispatch detail and session views. This closes the observability gap: today we know *what* was dispatched but not *how much context budget was consumed or managed*.

## Current State

- `bridge_dispatch_log` has `input_tokens`, `output_tokens`, `cached_tokens`, `latency_ms`
- `session_registry` has `tokens_used`, `token_budget`, `context_pct`
- No visibility into: how many directives were injected vs skipped, how many tool outputs were compressed, cumulative tokens saved by compression
- Admin Bridge panel shows dispatch list but no context pressure metrics

## Design

### Unified Context Stats

Each dispatch produces a `context_stats` JSONB blob:

```json
{
  "memory": {
    "tiers_used": ["identity", "directives", "project_notes"],
    "total_memory_tokens": 1200,
    "budget_tokens": 2000
  },
  "directives": {
    "total_active": 17,
    "injected": 8,
    "skipped": 9,
    "scoring_mode": "task_aware"
  },
  "skills": {
    "candidates": 5,
    "selected": 2,
    "prompt_tokens": 450
  },
  "compression": {
    "tool_outputs_compressed": 1,
    "conversation_turns_compressed": 4,
    "tokens_saved": 2800
  },
  "session": {
    "turn_number": 23,
    "context_pct": 0.72,
    "compression_events": 2,
    "tokens_reclaimed": 4200
  }
}
```

### Admin UI Components

#### Dispatch Detail Enhancement
On the existing dispatch log detail view, add a "Context" section:
- Token budget bar: memory tokens / skill tokens / message tokens (stacked)
- Directive selection: "8/17 directives injected (task-aware scoring)"
- Compression badge: "2 outputs compressed, 2.8K tokens saved"
- Session turn: "Turn 23 of session, 72% context used"

#### Session Detail Enhancement
On session view (if exists) or new session panel:
- Context pressure graph over time (turn number vs context_pct)
- Compression event markers on the graph
- Cumulative tokens saved by compression

## Tasks

### Task 1: Migration — context_stats column
**Files:** `backend/src/db/migrate-acx-v3.ts`
- Add `context_stats JSONB` to `bridge_dispatch_log` (nullable)
- This consolidates what Plans 38-01 and 38-02 need into one column
- Note: if 38-01 or 38-02 already added their own columns, this plan consolidates into one JSONB blob and migrates existing data

### Task 2: Context stats collector
**Files:** `backend/src/services/context-stats-collector.ts` (new, ~60 lines)
- `buildContextStats(opts)` — assembles the JSONB blob from available data:
  - `memoryStats` from `buildMemoryContext()` return value (needs Plan 38-01 to expose this)
  - `directiveStats` from directive scorer
  - `skillStats` from `selectSkills()` result
  - `compressionStats` from compressor events in current dispatch
  - `sessionStats` from `upsertSession()` result
- Pure assembly function — no DB access, just formats data from other services

### Task 3: Wire into logDispatch
**Files:** `backend/src/services/bridge/routing-engine.ts`
- In `logDispatch()`, call `buildContextStats()` with available data
- Write `context_stats` to the dispatch log row
- No behavioral change — purely observability

### Task 4: Admin API endpoint
**Files:** `backend/src/routes/v1/admin/bridge.ts` (or similar)
- `GET /api/v1/admin/dispatches/:id/context` — returns context_stats for a dispatch
- `GET /api/v1/admin/sessions/:id/context-pressure` — returns array of `{ turn, context_pct, compression_event }` for charting
- Both read-only, no writes

### Task 5: Admin UI — dispatch context panel
**Files:** `admin/src/pages/bridge/` (relevant component)
- Add "Context" expandable section to dispatch detail
- Stacked bar for token budget breakdown
- Directive selection summary line
- Compression badge (if any compression occurred)
- Session turn indicator with context % 

### Task 6: Admin UI — session pressure chart
**Files:** `admin/src/pages/bridge/` or `admin/src/components/`
- Line chart: X = turn number, Y = context_pct (0-100%)
- Markers for compression events
- Annotation for session rotation events
- Use existing chart library if available, otherwise lightweight inline SVG

## Verification

1. Dispatch an agent → `bridge_dispatch_log.context_stats` contains valid JSONB with all sections
2. View dispatch in admin → Context section shows token breakdown, directive stats, compression info
3. Run a 20-turn session → session pressure API returns 20 data points with increasing context_pct
4. Compression event during session → visible as marker on pressure chart
5. Dispatch with no compression → `compression` section shows all zeros (not null)
