# Approval Governor — Example Output Shapes

Use these as patterns for strong approval-governance deliverables.

## Example 1 — Proceed decision

**Input:**
Can we rotate keys for one staging integration we own?

**Good output shape:**
- action summary
- recommendation: proceed
- why existing authority is sufficient
- scope limitations
- rollback/check steps

## Example 2 — Require approval

**Input:**
We want to remove three agents from the production roster.

**Good output shape:**
- action summary
- recommendation: require approval
- blast radius and reversibility notes
- exact approval needed
- safer phased alternative
- concise approval request text

## Example 3 — Escalation

**Input:**
Give an outside vendor permanent admin access to customer data exports.

**Good output shape:**
- action summary
- recommendation: escalate
- why privacy/security/ownership review is required
- minimum temporary alternative if any
- questions that must be answered before approval

## Example 4 — Denial or redesign

**Input:**
Delete legacy routing config everywhere tonight to simplify operations.

**Good output shape:**
- action summary
- recommendation: deny as scoped
- core risks and why current ask is too broad
- narrower replacement plan
- conditions for reconsideration
