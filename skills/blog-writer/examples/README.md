# Blog Writer — Example Output Shapes

Use these as patterns for strong blog deliverables.

## Example 1 — SEO explainer post

**Input:**
Write a blog post explaining zero-trust security for IT leaders.

**Good output shape:**
- target audience and search intent
- working headline options
- intro with a clear hook
- structured sections explaining concept, misconceptions, implementation, and pitfalls
- concise conclusion with CTA
- suggested internal links and metadata notes

## Example 2 — Thought-leadership article

**Input:**
Draft a point-of-view post on why AI copilots need better human oversight.

**Good output shape:**
- clear thesis statement
- argument broken into 3 to 5 sections
- examples or evidence supporting each claim
- confident but not inflated tone
- conclusion that reinforces the strategic implication

## Example 3 — Product education blog

**Input:**
Turn our feature notes into a customer-friendly blog post.

**Good output shape:**
- audience and job-to-be-done
- benefit-led framing rather than release-note dump
- screenshots or examples called out where useful
- sections on problem, solution, workflow, and impact
- CTA to try the feature or read docs

## Example 4 — Rewrite of weak draft

**Input:**
This draft is dull and repetitive. Fix it.

**Good output shape:**
- diagnosis of what's weak now
- stronger headline and opening
- improved section order
- tightened rewritten copy
- notes on what was cut and why

## Example 5 — Editorial brief

**Input:**
Create a blog brief for a post targeting finance leaders about usage-based pricing.

**Good output shape:**
- audience pain points
- thesis and angle
- keyword or intent notes if applicable
- proposed outline and evidence points
- CTA recommendation
- risks or claims needing validation
