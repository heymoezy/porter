# Example requests

- Design a graph linking customers, contracts, tickets, product usage, and account owners for account-health and renewal reasoning.
- Create an ontology for papers, authors, institutions, methods, datasets, and findings with citation provenance and time-aware updates.
- Plan entity-resolution rules for a graph built from CRM, support, and billing systems where company names and contacts often conflict.

## Strong deliverable patterns
- Start with the questions the graph must answer.
- Show the minimum viable entity and relation set before expanding.
- Make identity, provenance, confidence, and temporal rules explicit.
- Include example queries or product behaviors that validate the model.
- End with ingestion and governance implications, not just a pretty ontology.
