# Cost Optimizer — Example Output Shapes

Use these as patterns for spend-reduction tasks.

## Example 1 — Cloud spend review

**Input:**
Analyze our AWS bill and suggest the top savings opportunities.

**Good output shape:**
- top spend categories
- 5–10 ranked savings opportunities
- estimated savings range
- effort/risk per item
- quick wins vs structural changes

## Example 2 — SaaS sprawl cleanup

**Input:**
Review our software stack for overlapping tools and unused licenses.

**Good output shape:**
- vendor/tool inventory summary
- overlap or underutilization findings
- renewal-risk highlights
- cancellation/consolidation recommendations
- governance actions for future control

## Example 3 — Architecture tradeoff note

**Input:**
Should we move this always-on batch workload to a cheaper pattern?

**Good output shape:**
- current cost drivers
- alternative architecture options
- savings vs reliability/latency tradeoffs
- migration effort and rollback considerations
- recommendation with rationale

## Example 4 — Unit economics view

**Input:**
Help us understand infrastructure cost per customer for our SaaS platform.

**Good output shape:**
- allocation assumptions
- unit-cost calculation method
- major variance drivers
- risks in the current model
- next instrumentation improvements
