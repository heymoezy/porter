# Examples — ux-writer

## Representative requests

### 1. Payment failure rewrite
“Rewrite our payment failure flow so users understand the issue and know what to do next.”

Expected shape:
- identify the user moment and risk level
- provide revised error, helper, and CTA copy
- explain how the rewrite improves clarity and recovery
- note any missing context or edge states

### 2. New module microcopy set
“Create labels, helper text, empty states, and confirmations for a new CRM workflow.”

Expected shape:
- organize copy by screen or state
- keep terminology and action verbs consistent
- cover normal, empty, warning, success, and error conditions
- include concise rationale where choices are not obvious

### 3. Copy critique
“Critique our onboarding copy. It feels wordy and slightly confusing.”

Expected shape:
- diagnose ambiguity, redundancy, and tone problems
- rewrite the key strings in context
- note where layout or flow issues are being mistaken for copy issues
- end with implementation-ready recommendations

## Output expectation
A strong answer should:
- make the next step obvious without wasting space
- handle risky or confusing states with specific guidance
- preserve consistency across related strings
- be ready for product teams to implement directly
