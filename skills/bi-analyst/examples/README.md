# BI Analyst — Example Output Shapes

Use these as patterns for strong BI deliverables.

## Example 1 — Executive dashboard spec

**Input:**
Design a weekly executive dashboard for a SaaS business.

**Good output shape:**
- audience and decisions supported
- core KPI list with formulas
- recommended sections: growth, retention, revenue, operations
- comparisons to target and prior period
- drill-down or follow-up paths
- caveats on freshness and source trust

## Example 2 — Dashboard critique

**Input:**
Review our current sales dashboard and tell us what to fix.

**Good output shape:**
- what the dashboard currently gets right
- ambiguous or misleading metrics
- redundant or low-value charts to remove
- missing segment views or comparisons
- revised structure recommendation
- highest-priority fixes first

## Example 3 — Competing report discrepancy

**Input:**
Finance says MRR is different from the product dashboard. Find the issue.

**Good output shape:**
- definitions used by each report
- source/grain mismatch analysis
- root cause of the discrepancy
- recommended source-of-truth definition
- remediation steps for affected dashboards

## Example 4 — Operational BI design

**Input:**
Create a dashboard for monitoring onboarding bottlenecks.

**Good output shape:**
- workflow stages and owner actions
- stage-level KPIs and alert thresholds
- segment filters that matter
- charts/tables with clear investigation paths
- notes on latency and incomplete data handling

## Example 5 — Metric governance memo

**Input:**
We have too many inconsistent metrics across teams. Standardize them.

**Good output shape:**
- metric inventory approach
- canonical definitions to adopt
- deprecated or duplicate metrics to retire
- glossary or semantic-layer recommendation
- governance workflow for future KPI changes
