# Examples — Tech Debt Manager

## Representative requests
- Build a technical debt register for a SaaS monorepo and rank what should be fixed this quarter.
- Explain which testing, release, and architecture debt is causing repeated launch delays.
- Turn “the backend is brittle” into a concrete remediation roadmap with owners and sequencing.
- Decide whether an outdated subsystem should be rewritten, strangled, contained, or left alone.
- Prepare a founder-friendly case for spending 20% of engineering capacity on debt reduction.

## Strong output shape
- Distinguish debt from bugs, backlog, and conscious tradeoffs.
- Identify root causes and evidence, not just complaints.
- Rank items by consequence and compounding risk.
- Recommend near-term actions, later actions, and things to leave alone.
- Define success measures so the payoff can be observed.

## Weak output shape
- Laundry lists of code smells with no prioritization.
- Blanket advice to rewrite major systems.
- Abstract claims about maintainability with no delivery or reliability impact.
- Cleanup recommendations that ignore ownership, sequencing, or roadmap pressure.
