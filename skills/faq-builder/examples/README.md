# FAQ Builder — Example Output Shapes

Use these as patterns for strong deliverables.

## Example 1 — Help-center FAQ refresh

**Input:**
Refresh a billing FAQ using recent support tickets, refund policy notes, and search terms from the help center.

**Strong output shape:**
- audience + purpose summary
- merged question set ordered by ticket volume and urgency
- concise answers with refund windows, exceptions, and escalation paths
- contradictions flagged for finance/policy owner review

## Example 2 — Product launch FAQ

**Input:**
Create a launch FAQ for a new AI feature from release notes, privacy review notes, and customer-success objections.

**Strong output shape:**
- launch-specific FAQ categories: access, pricing, privacy, limits, rollout timing
- direct answers in customer language
- explicit notes on beta restrictions and unknowns
- maintenance trigger for GA update

## Example 3 — Internal operations FAQ cleanup

**Input:**
Rewrite an internal HR FAQ with duplicate leave-policy questions and inconsistent wording across regions.

**Strong output shape:**
- duplicate clustering by policy intent
- region-specific splits only where rules differ
- plain-language answers with owner-approved constraints
- list of questions that require legal/HR confirmation before publishing
