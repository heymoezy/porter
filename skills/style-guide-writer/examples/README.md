# Style Guide Writer — Example Output Shapes

Use these patterns to create guides people will actually use.

## Example 1 — Cross-channel voice guide

**Input:**
Create a voice-and-tone guide for a fintech brand covering marketing pages, onboarding emails, support replies, and in-product messaging.

**Good output shape:**
- scope and audience definitions
- 3–5 voice principles translated into rules
- channel-specific flex notes
- terminology, punctuation, and claims guidance
- before/after examples for each channel
- adoption notes for reviewers and contributors

## Example 2 — Style-guide audit

**Input:**
Audit our existing style guide. Teams say it is vague and contradictory.

**Good output shape:**
- top-line verdict on what is broken
- contradictions, loopholes, and missing sections
- examples of guidance that cannot be operationalized
- recommended rewrite structure
- highest-impact fixes first

## Example 3 — Product microcopy standard

**Input:**
Build writing rules for buttons, errors, empty states, confirmations, and onboarding UI.

**Good output shape:**
- microcopy objectives and risk moments
- rules by UI state
- preferred syntax, length, and tone constraints
- say / avoid examples
- notes on accessibility, clarity, and support deflection

## Example 4 — Terminology governance pack

**Input:**
Create naming and terminology rules for a complex B2B platform with overlapping product areas.

**Good output shape:**
- naming hierarchy and ownership logic
- approved, deprecated, and banned terms
- capitalization / abbreviation rules
- disambiguation examples for overlapping concepts
- process for adding or revising terms later
