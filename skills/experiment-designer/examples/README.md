# Experiment Designer — Example Output Shapes

Use these patterns to keep experiment plans causal, measurable, and decision-ready.

## Example 1 — Product A/B test plan

**Input:**
Design an experiment for a new onboarding checklist.

**Good output shape:**
- decision the test should inform
- causal hypothesis and mechanism
- assignment unit and exposure logic
- primary metric, supporting metrics, and guardrails
- sample/duration assumptions
- instrumentation checks before launch
- launch, stop, and readout rules

## Example 2 — Operational pilot design

**Input:**
We want to test a new support escalation policy across regions.

**Good output shape:**
- why randomization may need to happen by team or region
- contamination and fairness risks
- pre/post or staggered rollout options
- operational success and harm metrics
- execution constraints and owner responsibilities
- interpretation caveats

## Example 3 — Marketing experiment framework

**Input:**
Create a test plan for a new pricing page headline.

**Good output shape:**
- exact intervention and control definition
- traffic split and inclusion rules
- primary conversion metric
- downstream guardrails such as bounce, refund, or support-contact rate
- duration logic and seasonality risks
- failure modes that would invalidate the readout

## Example 4 — “Should we run this?” diagnosis

**Input:**
Tell me whether this proposed experiment is valid.

**Good output shape:**
- key threat to validity
- what decision it would and would not answer
- whether the design is salvageable
- recommended redesign or alternative causal approach
- minimum conditions required before launch

## Example 5 — Executive decision memo

**Input:**
Write a concise experiment brief for leadership.

**Good output shape:**
- decision at stake
- hypothesis and business rationale
- risk-managed design summary
- primary success criterion and guardrails
- what different result patterns would mean for rollout
- open risks leadership should accept consciously
