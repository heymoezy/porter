# Policy Researcher — Example Output Shapes

Use these patterns to keep policy work decision-ready.

## Example 1 — Regulatory change brief

**Input:**
Summarize a proposed AI transparency rule for SaaS vendors operating in one region.

**Good output shape:**
- Scope: jurisdiction, regulator, publication date, current status
- What changed:
  - proposal summary
  - main obligations under discussion
  - consultation or adoption timeline
- Who is affected:
  - model developers
  - SaaS integrators
  - enterprise buyers
- Practical implications:
  - documentation burden
  - procurement questions
  - contract / disclosure changes
- What to monitor next:
  - consultation deadline
  - final rule publication
  - enforcement guidance

## Example 2 — Cross-jurisdiction comparison

**Input:**
Compare contractor-classification policy across three markets.

**Good output shape:**
| Jurisdiction | Current status | Main rule/test | Most affected businesses | Immediate action |
|---|---|---|---|---|
| Market A | In force | strict worker test | gig platforms, staffing firms | review model now |
| Market B | Proposed | threshold-based model | multi-market employers | monitor consultation |
| Market C | Court-driven | fact-specific balancing | platform businesses | legal review for edge cases |

Then add:
- differences that materially change operating model choices
- biggest unknowns
- which market needs urgent follow-up first

## Example 3 — Stakeholder and implementation map

**Input:**
Who inside a company will care about a new privacy reporting rule?

**Good output shape:**
- Legal/compliance: interpretation, reporting threshold, recordkeeping
- Product/engineering: logging, data classification, workflow changes
- Security/ops: incident detection and evidence capture
- Sales/procurement: customer questions and contractual assurances
- Leadership: risk appetite, staffing, external messaging

## Example 4 — Policy watch update

**Input:**
Create a weekly update format for payments-regulation tracking.

**Good output shape:**
- Status since last update
- New primary-source developments
- Probability of near-term action
- Implications if adopted as written
- Next dates / hearings / consultation windows
- Items needing legal or operational review
