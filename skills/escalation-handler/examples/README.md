# Escalation Handler — Example Output Shapes

Use these patterns to shape strong escalation-handling deliverables.

## Example 1 — Angry enterprise customer

**Input:**
A major customer is furious because their issue has been open for five days. Draft the response and plan.

**Good output shape:**
- issue summary and severity
- accountable owner and immediate actions
- customer-facing response draft
- next update time / cadence
- internal escalation path
- trust-repair steps after resolution

## Example 2 — Executive escalation

**Input:**
Leadership wants an answer in the next hour about why this rollout failed.

**Good output shape:**
- confirmed facts vs unknowns
- impact statement
- immediate mitigation steps
- owner list and next update time
- executive-ready summary language

## Example 3 — Repeat failure

**Input:**
This is the third time the customer has had the same problem. Handle the escalation.

**Good output shape:**
- acknowledgment of repeated failure
- short explanation of what is different now
- accountable next steps
- prevention / root-cause commitment
- follow-through plan

## Example 4 — Launch-blocking partner issue

**Input:**
A partner integration broke before their launch. We need a controlled response.

**Good output shape:**
- severity framing
- partner communication draft
- engineering / product dependencies
- timing commitments and decision gates
- fallback options if resolution slips

## Example 5 — Internal coordination brief

**Input:**
Summarize this escalation for support, engineering, and success so everyone is aligned.

**Good output shape:**
- concise issue statement
- impact and urgency
- owners by function
- immediate actions and blockers
- communication plan and update cadence

## Example 6 — Post-escalation review

**Input:**
Create the follow-up notes after we resolved the escalation.

**Good output shape:**
- summary of what happened
- root cause if known
- what worked and failed in the response
- stakeholder impact
- prevention actions with owners and due dates
