# Examples — Technology Scout

## Representative requests
- Compare observability platforms for a lean infra team that wants low-ops setup, decent traces, and predictable cost.
- Assess whether this agent framework is mature enough for production, or if we should treat it as a prototype-only tool this year.
- Build a buy-vs-build memo for feature flagging given our compliance requirements and small backend team.
- Create a 12-month watch list of technologies that could matter for telecom reliability and routing intelligence.

## Output expectations
- Define the decision and the evaluation criteria first.
- Compare only the most relevant options.
- Recommend a path: adopt now, pilot, defer, avoid, or watch.
- Include risk notes, lock-in concerns, and the next validation step.

## Good fit signals
- The request involves tool, vendor, framework, or platform choice.
- The team needs tradeoffs, not just feature descriptions.
- Maturity, lock-in, or implementation burden matters.
- There is a real decision horizon.

## Poor fit signals
- The technology has already been chosen and the task is implementation.
- The main work is sourcing workflow or contract process.
- The task is broader corporate strategy with little technology depth.
