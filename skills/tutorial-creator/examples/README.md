# Examples — tutorial-creator

## Representative requests

### 1. Quickstart rewrite
“Rewrite this dense setup document into a 10-minute quickstart for developers.”

Expected shape:
- define the target user and starting assumptions
- surface prerequisites up front
- turn the process into ordered steps with expected outputs
- include the most likely setup errors and fixes

### 2. Beginner product walkthrough
“Create a beginner-friendly tutorial showing users how to build and publish their first automation.”

Expected shape:
- focus on one first successful outcome
- use action-first steps and checkpoints
- keep theory brief and contextual
- end with what to try next after publishing

### 3. Release-to-education conversion
“Turn this product release into a customer education walkthrough with screenshots and checkpoints.”

Expected shape:
- identify the audience and use case
- translate features into a guided task flow
- indicate where screenshots or visuals should support the steps
- include recap, troubleshooting, and next-step suggestions

## Output expectation
A strong answer should:
- make the end state obvious
- give steps that are executable without hidden assumptions
- show what success looks like after key actions
- reduce first-run confusion instead of adding extra theory
