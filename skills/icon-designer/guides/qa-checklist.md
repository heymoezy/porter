# QA Checklist — Icon Designer

Before returning work, verify that:

- Each icon's meaning is recognizable at the stated target size.
- The set uses one coherent grammar for stroke, fill, corners, perspective, and detail density.
- Adjacent concepts are clearly differentiated and collision risks are called out.
- Recommendations are grounded in real UI usage, not only enlarged-canvas aesthetics.
- Visual descriptions are specific enough for a designer or generator to execute without guesswork.
- Expansion rules exist if the task involves a reusable icon family.
- Any dependence on labels, color, or motion is explicitly noted where icon shape alone is insufficient.
