# Handoff Director — Example Output Shapes

Use these as patterns for strong handoff deliverables.

## Example 1 — Debugging handoff

**Input:**
Package this production bug investigation for another agent to continue tomorrow.

**Good output shape:**
- issue, impact, and transfer reason
- current findings and evidence
- what was tried, what failed, and what remains unknown
- exact files, logs, dashboards, and tickets to inspect
- first step for the next owner

## Example 2 — Cross-functional transfer

**Input:**
Hand off this launch package from product to support.

**Good output shape:**
- transfer scope and readiness status
- decisions already locked
- support-owned tasks and deadlines
- known risks, dependencies, and escalation contacts
- acceptance criteria for a successful receipt

## Example 3 — Ready-for-review package

**Input:**
Make this implementation ready for reviewer handoff.

**Good output shape:**
- concise change summary
- areas that deserve extra scrutiny
- test status and evidence
- unresolved questions and risk notes
- clear review ask and definition of done

## Example 4 — End-of-shift ops handoff

**Input:**
Write a clean ops handoff for the next shift.

**Good output shape:**
- active incidents by severity
- what changed this shift
- pending actions in priority order
- blockers, watch items, and escalation path
- immediate first checks for the incoming owner
