# QA Checklist — Roster Curator

Use this before finalizing any catalog recommendation.

## 1. Need quality
- Is the recommendation tied to repeated demand, routing misses, or real confusion?
- Is the difference between a true capability gap and a metadata problem explicit?
- Did the analysis avoid novelty for novelty's sake?

## 2. Boundary clarity
- Are adjacent skills compared directly?
- Can the proposed boundary be explained in one sentence?
- Would another agent know which skill to use from the metadata alone?

## 3. Portfolio discipline
- Does the recommendation reduce complexity or clearly justify adding it?
- Was reuse considered before expansion?
- Were merge, retire, or rename options evaluated honestly?

## 4. Actionability
- Is the decision explicit: reuse, rename, merge, split, retire, or create?
- Are the exact next edits clear?
- Would a maintainer know what changes to make without redoing the analysis?

## 5. Routing impact
- Does the change make routing clearer?
- Are likely trigger phrases or request patterns considered?
- Is it obvious how this improves catalog quality for Porter?

## 6. Writing quality
- Is the recommendation concise, direct, and evidence-based?
- Are tradeoffs visible?
- Would this output help prevent future roster sprawl?
