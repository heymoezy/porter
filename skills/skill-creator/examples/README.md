# Skill Creator — Example Output Shapes

Use these patterns to keep skill-design work sharp and portfolio-safe.

## Example 1 — Full rewrite

**Input:**
This skill is bloated, vague, and collides with two nearby skills. Rewrite the pack.

**Good output shape:**
- diagnosis of trigger and boundary failures
- revised scope and non-scope
- rewritten pack files with aligned worker identity
- metadata cleanup
- brief note on why the new boundary is better

## Example 2 — New skill creation

**Input:**
Create a new skill for vendor risk questionnaires.

**Good output shape:**
- repeated use cases and reason the skill should exist
- adjacent-skill separation
- frontmatter description that routes well
- concise workflow guidance
- examples, QA, and metadata aligned to the same job

## Example 3 — Trigger cleanup

**Input:**
Routing rarely selects this skill even when the task matches.

**Good output shape:**
- why the current description under-triggers
- rewritten description with realistic use cases
- alias and adjacency updates
- any example or QA changes needed to reinforce scope

## Example 4 — Package audit

**Input:**
Audit this skill pack for quality and tell me what to fix first.

**Good output shape:**
- strengths and weaknesses by file
- overlap, bloat, or coherence issues
- highest-priority rewrite actions
- whether to narrow, merge, or split

## Example 5 — Portfolio-safe split or merge

**Input:**
This skill really does two jobs. Propose the cleanest fix.

**Good output shape:**
- center-of-gravity analysis
- what stays, moves, merges, or disappears
- trigger and metadata implications
- implementation notes for the affected packs
