# QA Checklist — Skill Creator

Use this before finalizing any skill-pack work.

## 1. Trigger quality
- Does the frontmatter description clearly say what the skill does and when to use it?
- Would realistic user language trigger the skill?
- Are non-scope boundaries explicit where collisions are likely?

## 2. Scope discipline
- Is the skill coherent enough to behave like one worker?
- Is the scope broad enough to earn existence but narrow enough to stay distinct?
- Did the rewrite reduce overlap instead of hiding it?

## 3. Progressive disclosure quality
- Is the always-loaded description high-signal?
- Is `SKILL.md` concise relative to the job?
- Are details placed in the right file instead of repeated everywhere?

## 4. Package coherence
- Do `SKILL.md`, `prompt.md`, examples, QA, and metadata all describe the same worker?
- Do examples reinforce the stated scope rather than drift beyond it?
- Are aliases, adjacent skills, and outputs consistent with the body?

## 5. Downstream usefulness
- Would another strong agent perform measurably better after loading this pack?
- Are workflows, heuristics, and quality bars practical?
- Does the package remove ambiguity instead of adding it?

## 6. Portfolio health
- Does the skill improve the catalog rather than add noise?
- Are split/merge recommendations surfaced when they are the real fix?
- Would a roster maintainer view this as world-class, maintainable package design?
