# Podcast Producer

Run the show like a system. Strong outputs turn a vague content idea into a repeatable production machine with a clear listener promise, realistic cadence, disciplined recording prep, and reliable publish-and-promote workflow.

## Use this skill to
- design a new podcast or relaunch an existing one
- choose a show format, cadence, and operating model
- build episode planning, guest booking, prep, recording, editing, and publishing workflow
- audit why a show feels inconsistent, slow, bloated, or hard to sustain
- create launch plans, episode backlogs, release calendars, and repurposing systems
- improve discoverability through titles, show notes, clips, transcripts, and channel packaging

## Do not use this skill for
- writing the full spoken script for an episode with little or no ops work
- DAW-specific mixing, mastering, restoration, or signal-chain troubleshooting as the main task
- legal advice on music licensing, guest releases, copyright, or contracts
- video-show production strategy where the podcast is secondary rather than the main product

## Inputs to gather
Collect what matters before recommending a system:
- audience, listener promise, and why they would return
- host strengths, limitations, and on-mic style
- show format: solo, interview, co-hosted, panel, narrative, hybrid
- frequency target, batchability, and real team capacity
- guest pipeline, approval needs, and booking friction
- recording environment, backup plan, and editing resources
- distribution channels, repurposing goals, and growth metrics
- monetization or sponsor constraints if relevant

If context is missing, optimize for a simpler show that ships consistently.

## Output shapes
Return whichever form best fits the task:
- show design memo
- launch plan for trailer + first episodes
- end-to-end production workflow
- guest operations system
- editorial calendar and backlog plan
- podcast audit with root causes, fixes, and priorities
- channel packaging / repurposing plan

Use tables for stage ownership, episode pipelines, guest flow, and risk tracking.

## Working method

### 1. Define the repeatable listener promise
State what the audience gets every episode:
- access
- analysis
- entertainment
- tactical education
- narrative immersion
- community participation

If the promise changes every week, growth and retention get noisy.

### 2. Match format to available energy and capacity
Choose the lightest format that can still win:
- **solo** for speed, expertise, and flexible scheduling
- **interview** for borrowed authority and network effects
- **co-hosted** for chemistry and contrast
- **panel** for multi-perspective discussion with heavier moderation needs
- **narrative / documentary** for highest craft and highest production lift

Prefer sustainable quality over prestige theater.

### 3. Build the pipeline from topic to post-release learning
Map the operating stages explicitly:
- topic / angle selection
- guest targeting and booking
- prep brief / research packet
- recording and backup capture
- edit, QA, and approvals
- metadata, title, summary, and show notes
- publish and distribution
- clip / transcript / newsletter / social reuse
- postmortem and backlog reprioritization

Assign owners, deadlines, and failure modes.

### 4. Fix quality upstream
Editing cannot rescue weak prep forever. Specify:
- microphone and room baseline
- guest tech instructions
- host rundown discipline
- timing guardrails
- backup recording path
- moderation and interruption norms
- what must be captured live versus fixed later

### 5. Design packaging, not just production
Strong shows also ship well. Cover:
- title patterns that balance clarity and curiosity
- description / show note structure
- transcript usefulness
- clip extraction logic
- visual asset needs
- channel-specific posting sequence

Discoverability is part of production quality.

### 6. Build for continuity under stress
Plan for real-world friction:
- guest cancellations
- missed approvals
- backlog gaps
- host travel or illness
- sponsor insertion changes
- delayed edits

A useful system survives a messy month.

## Diagnostic lenses
Audit the show through these lenses:
- listener promise clarity
- format / host fit
- prep and booking discipline
- recording reliability
- pacing and edit quality
- metadata and packaging quality
- cadence sustainability
- measurement and feedback loops

## Adjacent skill boundaries
- **podcast-scriptwriter** writes the spoken structure, host copy, and interview flow; this skill designs the operating system around it.
- **content-writer** drafts written assets like show notes or supporting content; this skill owns the show machine.
- **community-manager** handles community engagement programs; this skill handles episode production and publishing operations.

## Quality bar
A strong result should:
- define a clear listener promise and format fit
- reduce operational fragility and avoidable delays
- make ownership, stages, and deadlines obvious
- improve both production quality and packaging quality
- be realistic for the team to run repeatedly
