# QA Checklist — Project Operator

Use this before finalizing any project-operations output.

## 1. Execution clarity
- Is the current objective or interval clear?
- Does every important action have an owner?
- Are due dates or review dates present where needed?

## 2. Signal quality
- Does the output separate progress, blockers, and decisions cleanly?
- Are blockers elevated rather than buried?
- Is the update concise enough that people will actually use it?

## 3. Process fit
- Is the process weight appropriate for the project size and urgency?
- Does the cadence match the work rhythm?
- Is unnecessary ceremony avoided?

## 4. Follow-through strength
- Are next actions explicit?
- Are escalation paths identified for blocked work?
- Would this output change team behavior, not just document it?

## 5. Common risk checks
- owner ambiguity removed
- blocker visibility preserved
- vague status labels avoided without action context
- too many process layers avoided
- next checkpoint defined when relevant

## 6. Output usefulness
- Could a team run the next week from this?
- Would leadership know what decision/help is needed?
- Does it increase execution reliability without adding bloat?
