# Examples — ux-researcher

## Representative requests

### 1. Onboarding usability test
“Create a usability test for onboarding that checks whether new users can reach first value in 10 minutes.”

Expected shape:
- define the onboarding decision and target participant profile
- specify realistic scenarios and measurable success criteria
- identify confusion, failure, and recovery signals
- show how findings will be prioritized for launch decisions

### 2. Prototype comparison
“Help us compare two dashboard navigation models before we build either one.”

Expected shape:
- choose an appropriate comparative method
- define tasks that expose findability and comprehension differences
- describe what evidence would favor one direction
- end with a decision-ready synthesis frame

### 3. Settings IA diagnosis
“Users say settings are hard to navigate. Plan the research and tell us how to rank the issues.”

Expected shape:
- define the artifact and likely failure points
- recommend method mix if needed, such as tree test plus usability sessions
- explain severity, frequency, and impact criteria
- produce a fix-first reporting structure

## Output expectation
A strong answer should:
- focus on task success, comprehension, and recoverability
- expose the riskiest usability failures quickly
- avoid blending discovery research into a concrete usability study
- give the team specific next design actions
