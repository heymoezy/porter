# Example requests

- Redesign a five-step onboarding flow so new users reach first value in under two minutes without feeling rushed.
- Specify the interaction model for bulk approval, partial failure, retry, and audit history in an operations dashboard.
- Audit an account settings area for confusing permissions, destructive actions, save behavior, and recovery gaps.
- Define how a mobile upload flow should behave across offline mode, background processing, and validation failures.

## Strong deliverable patterns
- Reframe the problem around user intent, not just screens.
- Show the primary path, alternate paths, and failure paths.
- Use state tables or step-by-step rules where behavior could be misimplemented.
- Call out unanswered product decisions separately from solved interaction decisions.
- End with the smallest next step that would validate or ship the design.
