# Whitepaper Author — Example Output Shapes

Use these as patterns for credible whitepaper work.

## Example 1 — Outline first

**Input:**
Draft a whitepaper on AI governance for mid-market enterprises.

**Good output shape:**
- working thesis
- intended audience
- proposed structure:
  1. executive summary
  2. why governance matters now
  3. operational risks and failure modes
  4. governance model options
  5. recommended operating framework
  6. implementation priorities
- key sources to cite
- known evidence gaps

## Example 2 — Executive summary

**Input:**
Write the executive summary for a cloud-cost optimization whitepaper.

**Good output shape:**
- short opening paragraph on the problem and stakes
- evidence snapshot
- recommended direction
- expected business effect
- one sentence on tradeoffs or limits

## Example 3 — Section drafting

**Input:**
Write the section on why fragmented identity systems create enterprise risk.

**Good output shape:**
- claim-led heading
- concise argument
- evidence or examples
- implication for the reader
- transition to the next section

## Example 4 — Revision memo

**Input:**
Review this draft whitepaper and tell me what's weak.

**Good output shape:**
| Section | Issue | Why it hurts credibility | Fix |
|---|---|---|---|
| intro | too generic | reader learns nothing specific | tighten stakes and audience |
| evidence | claims uncited | trust drops | add primary research or official sources |
| solution | reads like product pitch | analysis feels biased | add options and tradeoffs |
| conclusion | drifts | recommendation feels soft | restate decision and remaining limits |
