# Course Creator

Turn expertise into learning that actually transfers.

This skill is for structured instructional design: defining outcomes first, aligning assessments and activities, sequencing content, controlling cognitive load, and packaging materials that a learner or instructor can use.

## Scope

Use this skill for:
- full course and curriculum design
- workshop and cohort program planning
- onboarding or enablement learning paths
- module outlines and lesson sequencing
- learning objectives and assessment design
- exercises, assignments, and practice prompts
- instructor guides and learner handouts
- converting raw expertise into teachable structure

## Use this skill when

Use this skill when the task needs:
- outcome-first course design rather than simple content dumping
- clear learning objectives and competency progression
- alignment between lessons, practice, and assessment
- beginner-to-advanced sequencing
- materials that support actual instruction or self-paced learning

## Do not use this skill when

Do not use this skill for:
- simple documentation writing with no teaching design component
- visual deck polish where design is the main need
- LMS configuration or course-platform administration
- expert-level research synthesis without translating it into pedagogy

## Inputs to gather

Before designing, identify:
- target learners and their starting level
- learning goal, business goal, or credential goal
- delivery mode: live, async, cohort, blended, workshop, etc.
- duration, pacing, and time budget
- prerequisite knowledge or access needs
- assessment constraints and evidence of mastery
- source material, SME input, examples, and tooling
- tone and format constraints for instructors or learners

If the audience or outcome is vague, tighten that before expanding content.

## Output expectations

Return outputs such as:
- course blueprint or syllabus
- module-by-module outline
- learning objectives mapped to lessons and assessments
- session plans or facilitator guide
- learner exercises, quizzes, or projects
- recommendations for sequencing, pacing, and feedback loops

For large builds, prioritize a usable scaffold with enough detail to execute.

## Working method

### 1. Start with outcomes, not topics

Define what learners should know or be able to do.
Write objectives that are observable and assessable.
Useful verbs usually describe action, not vague exposure.

Backward design beats content hoarding.

### 2. Define acceptable evidence of learning

Before building modules, decide how mastery will show up:
- quiz or recall checks
- application exercise
- project or case analysis
- live performance or demonstration
- reflection tied to real use

If you cannot measure it, the objective is probably too fuzzy.

### 3. Sequence for learner progression

Organize from:
- foundations before abstraction
- simple before compound
- recognition before production
- guided practice before independent work
- low-stakes feedback before high-stakes evaluation

Respect cognitive load. Dense is not the same as rigorous.

### 4. Turn content into instruction

For each module, specify:
- objective
- key concepts
- teaching method
- practice activity
- assessment/evidence
- estimated duration
- materials needed

A course is not a pile of notes.

### 5. Design practice, not just explanation

Include opportunities to:
- retrieve knowledge
- apply concepts in realistic scenarios
- compare examples and counterexamples
- receive feedback
- reflect on mistakes and transfer the learning

Learning happens through doing.

### 6. Build for the delivery context

Adapt the design to context:
- live instruction needs timing, facilitation notes, and interaction design
- async delivery needs clear transitions, chunking, and self-checks
- onboarding needs quick job relevance and minimal friction
- expert audiences need less exposition and more challenge

### 7. Package for execution

Deliver artifacts that another person can run:
- syllabus or blueprint
- module map
- lesson plans
- assignment prompts
- rubrics or answer expectations
- notes on revision based on learner feedback

## Adjacent skill boundaries

- **training-developer**: often stronger on enterprise training asset production and rollout execution; this skill owns the instructional design core
- **documentation-writer / technical-writer**: explain products or processes, but do not necessarily design learning progression
- **course-creator** is broader than slide copy or worksheet drafting alone
- **edtech-designer**: deeper learning-product or platform experience design when software/system design dominates
- **tutorial-creator**: narrower task-by-task teaching; this skill handles fuller curriculum architecture

## Quality bar

A strong result should:
- define clear learner outcomes
- align assessments and activities to those outcomes
- sequence material sensibly for the audience
- manage scope, pacing, and cognitive load well
- give an instructor or builder a practical path to execute
