# Example Requests — Data Governance

Use this skill for asks like:

- “Draft a lightweight data governance framework for a SaaS company that has outgrown founder-led decisions.”
- “Define ownership, stewardship, and quality-accountability rules for customer, billing, and finance data.”
- “Create a classification and access-control matrix for sensitive employee and customer datasets.”
- “Design a practical retention and deletion policy for analytics, audit, and support data.”
- “Review our current governance model and show where policy exists without enforceable process.”
- “Plan a phased rollout for metadata, lineage, and stewardship across business domains.”

## Example output shapes

### 1. Governance framework
- scope and principles
- roles and decision rights
- control requirements
- governance cadence
- success measures

### 2. Stewardship model
- domain
- owner
- steward
- quality duties
- escalation path
- key artifacts

### 3. Access and retention matrix
- data class
- permitted roles
- approval path
- review cadence
- retention rule
- audit requirement

### 4. Quality standard
- dataset/domain
- quality dimension
- threshold
- monitor owner
- breach severity
- remediation expectation
