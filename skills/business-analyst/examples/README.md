# Business Analyst — Example Output Shapes

Use these as patterns for strong business-analysis deliverables.

## Example 1 — Requirements clarification

**Input:**
We need to improve enterprise customer onboarding but the teams disagree on what is broken.

**Good output shape:**
- business objective and decision needed
- stakeholder map and key pain points by team
- current-state process summary with bottlenecks
- root-cause hypotheses with evidence level
- prioritized requirements or changes
- risks, dependencies, and next validation steps

## Example 2 — Process redesign recommendation

**Input:**
Analyze our order approval workflow and recommend improvements.

**Good output shape:**
- current-state workflow in plain language
- failure points, delays, and rework loops
- business rules causing friction
- future-state options with tradeoffs
- recommendation with expected impact
- implementation notes and metrics to watch

## Example 3 — KPI framework

**Input:**
Define the right metrics for a support quality initiative.

**Good output shape:**
- objective and operating definition of quality
- leading and lagging KPI set
- metric formulas and caveats
- data source requirements
- reporting cadence and ownership
- misuse risks or interpretation warnings

## Example 4 — Decision memo

**Input:**
Should we automate partner intake now or keep it manual for one more quarter?

**Good output shape:**
- decision statement and context
- assumptions and constraints
- option comparison table
- cost, effort, and risk tradeoffs
- recommendation with rationale
- trigger points for revisiting the decision

## Example 5 — Business rules capture

**Input:**
Document the rules our billing team uses so engineering can build the workflow.

**Good output shape:**
- actors and source systems
- business rules grouped by scenario
- edge cases and exception handling
- ambiguities needing owner signoff
- acceptance criteria for implementation
- unresolved questions list
