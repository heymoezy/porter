# Regulatory Analyst — Example Output Shapes

## Example 1 — Change-impact memo

**Input:**
Explain how the new consumer data portability rule affects our fintech onboarding and account settings flows.

**Good output shape:**
- jurisdiction, regulator, and source status
- applicability analysis by product and customer type
- obligations affecting consent, access, portability format, and response timelines
- impacted workflows, teams, and evidence requirements
- deadline assumptions and rollout priorities
- open interpretation questions for counsel

## Example 2 — Obligations matrix

**Input:**
Map anti-money-laundering obligations for our broker-dealer operating model.

**Good output shape:**
- clause or citation
- obligation summary
- trigger or threshold
- control or workflow required
- owner and evidence artifact
- current-state gap and next action

## Example 3 — Multi-jurisdiction comparison

**Input:**
Compare marketing-consent requirements across the UK, EU, and Singapore for our email acquisition funnel.

**Good output shape:**
- side-by-side jurisdiction table kept clearly separate
- source hierarchy for each jurisdiction
- consent, opt-out, recordkeeping, and proof requirements
- shared implementation options vs jurisdiction-specific exceptions
- unresolved questions requiring local counsel
