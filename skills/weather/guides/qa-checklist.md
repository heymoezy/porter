# QA Checklist — weather

Use this before finalizing any weather-focused output.

## 1. Context integrity
- Is the answer anchored to a real location and time window?
- Is the timezone clear when timing matters?
- Is the decision context explicit?

## 2. Forecast usefulness
- Did the answer identify the few variables that actually matter?
- Did it translate forecast data into practical guidance?
- Did it avoid unnecessary meteorological clutter?

## 3. Source and freshness discipline
- If live data was needed, were current and authoritative sources preferred?
- Were unusual or high-impact claims cross-checked?
- Is the answer careful about stale or uncertain forecast data?

## 4. Uncertainty honesty
- Is confidence stated when relevant?
- Does the output avoid false precision?
- Is there a sensible contingency or re-check recommendation?

## 5. Risk framing
- Are the main hazards clearly named?
- Is severity proportional to the forecast signal?
- If conditions could become dangerous, does the output point to official alerts without pretending to replace them?

## 6. Output quality
- Does it lead with the recommendation or takeaway?
- Would a real user know what to do next?
- Is the answer concise, accurate, and decision-ready?
