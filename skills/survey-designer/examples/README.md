# Survey Designer — Example Deliverable Shapes

Use these as patterns for strong, field-ready outputs.

## Example 1 — Rewrite a bloated satisfaction survey

**Input:**
Shorten and fix a customer satisfaction survey that feels leading, repetitive, and hard to analyze.

**Good output shape:**
- survey objective and respondent context
- biggest instrument problems by severity
- revised question flow with deletions and rewrites
- rationale for scale and wording changes
- launch risks and pretest notes

## Example 2 — Product concept survey

**Input:**
Design a concept-test questionnaire for a new subscription offer.

**Good output shape:**
- decision to be informed and target audience
- recommended module structure: screener, baseline, concept exposure, reactions, intent, open ends
- exact or near-final question wording
- scale choices and logic notes
- what the survey can and cannot claim

## Example 3 — Employee pulse audit

**Input:**
Audit an employee pulse survey for fatigue, ambiguity, and manager-interpretation risk.

**Good output shape:**
- core use case for the survey
- question-by-question critique
- burden and confidentiality risks
- revised items and response options
- interpretation cautions for leadership

## Example 4 — Segmentation instrument

**Input:**
Create a market-segmentation questionnaire that distinguishes behaviors, needs, and attitudes without becoming too long.

**Good output shape:**
- segmentation objective and must-have dimensions
- recommended balance of behaviors, needs, and attitudes
- item set organized by module
- which items are essential vs optional
- coding and analysis-readiness notes

## Example 5 — Screener and routing logic

**Input:**
Build a clean screener and branching plan for a multi-country B2B survey.

**Good output shape:**
- eligibility logic and exclusions
- branch map by respondent type
- wording cautions for routing questions
- places where quota or locale constraints may break interpretation
- handoff notes for programming and QA
