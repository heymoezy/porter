# Content Auditor — Example Output Shapes

Use these as patterns for practical audit outputs.

## Example 1 — Blog post audit

**Input:**
Audit this draft article before publishing.

**Good output shape:**
- audience and purpose summary
- strengths worth keeping
- critical factual or structural issues
- missing examples, proof, or differentiation
- prioritized rewrite order

## Example 2 — Landing page review

**Input:**
Review our homepage copy. Conversions are weak.

**Good output shape:**
- page goal and likely reader intent
- headline/value-prop diagnosis
- friction points and trust gaps
- CTA weaknesses
- section-by-section recommendations by impact

## Example 3 — Newsletter content audit

**Input:**
Check this newsletter for clarity and engagement before send.

**Good output shape:**
- subject line and opening evaluation
- message hierarchy issues
- unnecessary length or repetition
- CTA and click-friction review
- concise edit priorities

## Example 4 — Documentation quality review

**Input:**
Audit this help article. Users still open support tickets.

**Good output shape:**
- missing context or prerequisites
- unclear steps or ambiguous terms
- screenshot/example gaps
- findability and heading issues
- suggested restructuring for self-serve success
