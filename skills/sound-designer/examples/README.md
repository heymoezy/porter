# Sound Designer — Example Output Shapes

Use these patterns to keep sound-design work concrete, restrained, and production-ready.

## Example 1 — Product sonic identity

**Input:**
Define the sound system for a calm but confident fintech app across onboarding, confirmations, warnings, and background states.

**Good output shape:**
- product context and playback realities
- sonic palette and anti-palette
- cue hierarchy by app state
- repetition and fatigue rules
- implementation notes and open questions

## Example 2 — Trailer or film scene direction

**Input:**
Create scene-by-scene sound direction for a suspense trailer with sparse dialogue and one major reveal.

**Good output shape:**
- emotional arc by scene
- ambience, texture, and silence plan
- transition and impact-cue notes
- risks of overplaying the tension
- handoff notes for editor / mixer / composer

## Example 3 — UI sound critique

**Input:**
Audit our current product sounds. Users say the app feels noisy and childish.

**Good output shape:**
- likely causes of the perception problem
- which cues to cut, soften, merge, or redesign
- timing / envelope / tonal issues
- what to preserve if anything still works
- prioritized next tests

## Example 4 — Game ambience and interaction brief

**Input:**
Write an audio brief for a puzzle game that should feel tactile, mysterious, and never stressful.

**Good output shape:**
- world and emotional rules
- interaction sound principles
- ambience behavior by game state
- negative-space guidance
- implementation cautions for repetition and reward loops

## Example 5 — Brand film handoff

**Input:**
Prepare a sound brief so the editor, composer, and mixer align on the same emotional build.

**Good output shape:**
- narrative role of sound in each beat
- palette references and anti-references
- dialogue / effects / music relationship
- moments that need restraint or silence
- final handoff checklist
