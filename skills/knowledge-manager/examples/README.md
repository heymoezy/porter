# Example requests

- Rationalize five overlapping internal wikis into one source-of-truth model with ownership, migration order, and archival rules.
- Design a taxonomy and metadata scheme for support, product, and engineering knowledge so people can find answers without asking Slack first.
- Create a review cadence and governance plan to reduce stale SOPs and duplicate playbooks across operations teams.

## Strong deliverable patterns
- Start with the specific retrieval and trust failures happening now.
- Separate content, governance, search, and org-behavior problems.
- Keep taxonomy and workflow simple enough for teams to maintain.
- Define owners, review triggers, and stale-content handling explicitly.
- End with a prioritized rollout and measurable success criteria.
