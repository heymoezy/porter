# Examples — typography-specialist

## Representative requests

### 1. UI type system
“Choose a typography system for our B2B dashboard. We need good dense-data readability on desktop and tablets.”

Expected shape:
- identify reading context, density, and implementation constraints
- recommend one or two families with fallback logic
- define roles for headings, body, labels, captions, and tables
- note line-height, spacing, and accessibility considerations

### 2. Critique and repair
“Critique this landing page typography. It feels premium, but people say it’s hard to scan.”

Expected shape:
- diagnose hierarchy, line length, spacing, and emphasis problems
- separate tone issues from readability issues
- recommend precise fixes with tradeoffs
- note what should remain unchanged if it supports the brand

### 3. Editorial system
“Build a text-style scale for reports and slide decks that has authority without feeling academic.”

Expected shape:
- define the reading job for reports vs presentation use
- propose text roles, weights, and spacing rhythm
- explain how to handle dense pages and callouts
- include implementation or template notes

## Output expectation
A strong answer should:
- improve readability and hierarchy, not just aesthetics
- provide concrete style recommendations that can be implemented
- explain tradeoffs among tone, accessibility, and density
- stay disciplined about the number of styles and exceptions
