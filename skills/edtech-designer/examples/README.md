# EdTech Designer — Example Deliverable Shapes

Use these patterns to keep outputs buildable and learning-effective.

## 1. Lesson redesign

**Input:**
Our fractions lesson is boring and learners forget it quickly. Redesign it.

**Strong shape:**
- learner and failure-point diagnosis
- outcomes rewritten in observable terms
- improved lesson sequence
- worked examples, guided practice, and independent practice
- formative checks and feedback timing
- retention risks and mitigation

## 2. Course module blueprint

**Input:**
Design a beginner cybersecurity-awareness module.

**Strong shape:**
- module outcomes
- sequence of learning activities
- realistic scenarios and practice tasks
- assessment aligned to behavior, not memorization alone
- reinforcement or spaced follow-up plan
- implementation notes for the delivery environment

## 3. Learning-product onboarding flow

**Input:**
Design the first-session learning flow for a language app.

**Strong shape:**
- target learner state and motivation challenges
- first-win design
- practice loop and feedback system
- progression logic and unlock criteria
- retention instrumentation ideas
- accessibility or cognitive-load cautions

## 4. Assessment review

**Input:**
Review our course assessments and tell us what is weak.

**Strong shape:**
- objective-to-assessment map
- misalignments and blind spots
- redesign recommendations for items or tasks
- formative versus summative balance
- scoring and evidence-quality notes

## 5. Curriculum sequence

**Input:**
Plan a six-week data-literacy curriculum for operations managers.

**Strong shape:**
- week-by-week progression
- prerequisite assumptions
- practice and application tasks
- assessment checkpoints
- workload, support, and reinforcement plan
