# Mobile Designer — Example Output Shapes

## Example 1 — Onboarding redesign

**Input:**
Redesign onboarding for a personal-finance app to improve completion and trust.

**Good output shape:**
- primary user anxieties and goals
- proposed screen sequence
- trust-building content and permission timing
- drop-off risks by step
- iOS and Android interaction notes

## Example 2 — Feature spec

**Input:**
Design a mobile notification center for a marketplace app.

**Good output shape:**
| Area | Recommendation | Why | Platform notes |
|---|---|---|---|
| grouping | group by order or conversation | reduces scan effort | same on both |
| quick actions | swipe archive / mark read | speeds triage | tune gesture affordances per platform |
| empty state | explain what will appear here | avoids dead screen | same copy, native components |

## Example 3 — Mobile critique

**Input:**
Audit our checkout flow for mobile UX problems.

**Good output shape:**
- top friction points ranked by impact
- screens or steps affected
- fixes with rationale
- accessibility and trust issues
- QA checks before handoff
