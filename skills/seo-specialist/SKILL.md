# SEO Specialist

Win search by being the clearest useful result for the query.

## Mission

Use this skill to turn SEO work into practical decisions about intent fit, page type, information architecture, technical crawlability, and content quality. Favor durable organic gains over hacks.

## Work sequence

1. **Define the search job**
   - Identify audience, geography, device context, funnel stage, and likely user intent.
   - Classify the query or page: informational, commercial investigation, transactional, navigational, local, support, or branded.
   - If the SERP likely wants a different page type than the current one, say so early.

2. **Match SERP expectations**
   - Infer what usually wins for the query class: product page, category page, comparison, glossary, tutorial, tool, documentation, local landing page, etc.
   - Evaluate whether the current page format, depth, and angle actually satisfy that expectation.

3. **Audit on-page clarity**
   - Check title tag, H1, subheads, URL, intro, body structure, media, and calls to action.
   - Make the main topic obvious fast.
   - Remove vague positioning, duplicated headings, and keyword-heavy copy that weakens readability.

4. **Assess topical completeness**
   - Identify the subtopics, entities, comparisons, objections, FAQs, examples, or proof points needed for this intent.
   - Improve completeness without padding the page.
   - Recommend consolidation when multiple pages compete for the same intent.

5. **Review technical discoverability**
   - Check crawlability, indexability, canonicals, redirects, rendering, pagination, structured data, mobile usability, and speed constraints.
   - Prioritize blockers that can suppress otherwise strong pages.

6. **Strengthen internal linking and architecture**
   - Map where authority and context should flow.
   - Recommend hub/supporting-page structure, anchor-text improvements, and link paths that help both users and crawlers understand topic relationships.

7. **Prioritize implementation**
   - Rank changes by expected impact, effort, and dependency.
   - Separate immediate fixes from deeper content or architecture work.

## Output requirements

Deliver one or more of these:
- page optimization brief
- SEO audit memo
- content gap / topic cluster map
- internal-linking plan
- technical SEO issue list
- prioritized organic roadmap

For each recommendation, include:
- what to change
- why it matters for this query or page type
- expected impact area: relevance, discoverability, CTR, consolidation, or crawl clarity
- implementation priority

## Heuristics

Prefer:
- intent fit over raw keyword repetition
- page-type correctness over forcing one page to do every job
- clear titles and headings over clever wording
- first-hand usefulness, evidence, and examples over filler
- consolidation when pages cannibalize each other

Avoid:
- ranking guarantees
- keyword stuffing or templated SEO sludge
- duplicative pages chasing the same intent
- metadata-only “optimization” with weak page substance
- advice that ignores crawl, render, or index constraints

## Boundaries

Use adjacent skills instead when needed:
- **copywriter** for final persuasion and brand-voice polish
- **content-writer** for drafting the full piece itself
- **web-designer** or performance-focused engineering skills for implementation-heavy UX / speed changes
- **data-analyst** for broader performance measurement and experiment analysis

## Final check

Before delivering, confirm the recommendation would make sense even if search engines did not exist: it should still make the page clearer, more useful, and easier to navigate.
