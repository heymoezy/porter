# Policy Drafter — Example Output Shapes

Use these as patterns for policy-writing outputs.

## Example 1 — New policy draft

**Input:**
Draft a remote work policy for a 60-person software company.

**Good output shape:**
- purpose
- scope
- definitions (only if needed)
- policy requirements
- manager and employee responsibilities
- equipment / security / workplace expectations
- exceptions
- review cadence
- related procedures

Then add:
- open assumptions
- issues needing legal or HR review

## Example 2 — Plain-language rewrite

**Input:**
Rewrite our acceptable use policy so customers can understand it.

**Good output shape:**
| Current issue | Why it hurts usability | Rewrite approach |
|---|---|---|
| vague prohibited behavior | enforcement feels arbitrary | convert into clear rule categories with examples |
| dense legal phrasing | readers stop scanning | shorten sentences and front-load the rule |
| no appeal path | dispute handling unclear | add reporting / review section |

Then provide:
- revised policy text
- top changes made
- any assumptions or unresolved edge cases

## Example 3 — Gap analysis

**Input:**
Audit our data retention policy for governance gaps.

**Good output shape:**
| Area | Present? | Gap | Risk | Fix |
|---|---|---|---|---|
| owner | no | nobody accountable | review drift | assign owner |
| exceptions | partial | no approval path | inconsistent treatment | define approver + documentation |
| review cadence | no | policy can go stale | audit failure | add annual review |

## Example 4 — Roles and exceptions section

**Input:**
Add an exceptions process and ownership section to our vendor policy.

**Good output shape:**
- policy owner
- approver roles
- exception request triggers
- required documentation
- approval criteria
- renewal / expiration rules
- recordkeeping and review notes
