# Onboarding Specialist — Example Output Shapes

Use these as patterns for activation-first onboarding work.

## Example 1 — Friction audit

**Input:**
Review our SaaS signup and setup flow. Many users create accounts, but few connect their data source.

**Good output shape:**
| Stage | Friction | Likely cause | Impact | Recommended fix |
|---|---|---|---|---|
| Signup | too many required fields | asking for context before value is clear | abandonment | cut to essentials and defer profile setup |
| Verification | email verification blocks setup | unnecessary gate for low-risk users | slower activation | allow limited setup before verification |
| Data connection | permission request appears with little context | trust gap and unclear payoff | connection drop-off | explain why access is needed and preview the result |

Then add:
- top 3 blockers
- quick wins vs structural fixes
- stage-specific metrics to watch

## Example 2 — Activation-first redesign

**Input:**
Design onboarding for a collaborative writing app.

**Good output shape:**
- Target users: solo creator, editor, team lead
- Activation event: publish first shared draft
- Recommended flow:
  1. choose use case
  2. start from template or import draft
  3. make first edit
  4. invite collaborator
  5. confirm successful share/publish
- Defer until later:
  - workspace branding
  - advanced permissions
  - profile completion
- Success metrics:
  - time to first shared draft
  - invite completion rate
  - day-7 team return rate

## Example 3 — Empty-state and checklist copy guidance

**Input:**
Improve our onboarding checklist and empty states.

**Good output shape:**
- Empty-state headline that states the payoff
- One-line explanation of why the next action matters
- Checklist items phrased as outcomes, not admin chores
- Reassurance text for permissions/import steps
- Recovery copy for users who cannot finish setup immediately

## Example 4 — Experiment backlog

**Input:**
What onboarding experiments should we run next quarter?

**Good output shape:**
| Experiment | Stage | Hypothesis | Metric | Effort |
|---|---|---|---|---|
| Persona-based branching | post-signup | tailored setup reduces confusion | setup completion | Medium |
| Sample workspace | first-run | seeded content speeds first value | activation rate | Low |
| Permission explainer | integration step | trust language improves consent | connect completion | Low |
| Deferred billing ask | pre-activation | removing early commitment increases first success | activation rate | Medium |
