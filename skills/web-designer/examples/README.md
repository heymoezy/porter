# Web Designer — Example Output Shapes

Use these as patterns for sharp, buildable web-design guidance.

## Example 1 — Landing page redesign

**Input:**
Redesign our B2B AI homepage so more visitors book demos.

**Good output shape:**
- Goal: clarify value fast and raise demo intent
- Diagnosis:
  - hero headline is generic
  - proof appears too late
  - too many equal-weight buttons
- Proposed structure:
  1. value proposition + single primary CTA
  2. product proof strip or screenshot
  3. trust signals
  4. key use cases / outcomes
  5. objections / FAQ
  6. closing CTA
- Visual direction:
  - larger headline-to-body contrast
  - more whitespace above the fold
  - accent color reserved for action
- Mobile note: compress proof and keep CTA visible early

## Example 2 — Dashboard critique

**Input:**
This analytics dashboard feels messy. Fix it.

**Good output shape:**
| Area | Problem | Recommendation | Why |
|---|---|---|---|
| KPI row | too many equal cards | reduce to 4 primary metrics | faster scan and prioritization |
| filters | visually buried | move near title and summary controls | encourages exploration |
| charts | no narrative grouping | cluster by business question | improves interpretation |
| table | dominates page | collapse advanced columns by default | lowers cognitive load |

Then add breakpoint behavior and what to remove.

## Example 3 — Pricing page plan

**Input:**
Design a pricing page for a mid-market SaaS product.

**Good output shape:**
- section order
- pricing-card structure
- default recommended plan treatment
- enterprise/sales-contact treatment
- FAQ and trust placement
- mobile stacking and comparison fallback

## Example 4 — Quick polish memo

**Input:**
Why does this page feel amateur?

**Good output shape:**
- 3–5 root causes only
- direct fixes for spacing, alignment, type scale, and proof placement
- what to cut, not just what to add
- one paragraph on how the page should feel after cleanup
