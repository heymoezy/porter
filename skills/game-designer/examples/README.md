# Game Designer — Example Output Shapes

Use these as patterns for strong game-designer deliverables.

## Example 1 — Midcore economy redesign

**Input:**
Design a progression and resource system for a midcore strategy game that encourages long-term play without pay-to-win collapse.

**Good output shape:**
clear sinks and faucets, pacing logic, scarcity choices, failure cases, and notes on what telemetry or playtests should validate next

## Example 2 — Co-op run structure

**Input:**
Create a core loop for a co-op roguelite focused on short runs, role synergy, and strong replayability.

**Good output shape:**
player fantasy, session loop, role interactions, meta-progression boundaries, and hypotheses for what makes runs feel fresh instead of repetitive

## Example 3 — Onboarding churn fix

**Input:**
Players leave before understanding the main mechanic in the first ten minutes. Redesign onboarding.

**Good output shape:**
teaching sequence tied to real gameplay, early feedback moments, reduced cognitive overload, and specific playtest questions for first-session comprehension
