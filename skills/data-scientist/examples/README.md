# Data Scientist — Example Output Shapes

Use these as patterns for decision-oriented analytical work.

## Example 1 — Churn modeling plan

**Input:**
Predict which customers are likely to churn next month so success managers can intervene.

**Good output shape:**
- decision and operational capacity
- unit, target, and prediction window
- data and leakage risks
- baseline heuristics
- candidate features and methods
- evaluation: lift, precision@k, recall, calibration
- recommendation and pilot plan

## Example 2 — Experiment design

**Input:**
Should we test a new onboarding flow?

**Good output shape:**
- hypothesis and expected mechanism
- randomization unit and population
- primary and guardrail metrics
- instrumentation requirements
- sample-size or detectable-effect guidance
- decision thresholds and readout caveats

## Example 3 — Forecasting recommendation

**Input:**
Forecast support-ticket volume for the next quarter for staffing.

**Good output shape:**
- planning decision tied to forecast
- trend, seasonality, and event assumptions
- baseline methods
- candidate forecasting approaches
- rolling-backtest design
- base/high/low scenarios and staffing implications

## Example 4 — Model evaluation readout

**Input:**
Summarize whether our fraud model pilot is good enough to ship.

**Good output shape:**
- business objective and threshold options
- baseline comparison
- false-positive and false-negative cost tradeoffs
- calibration and segment-level weaknesses
- rollout recommendation
- monitoring plan and unresolved risks
