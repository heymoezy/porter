# Podcast Scriptwriter — Example Output Shapes

Use these as patterns for podcast-script outputs.

## Example 1 — Solo episode rundown

**Input:**
Write a 20-minute rundown for a solo founder episode on pricing mistakes.

**Good output shape:**
| Beat | Goal | Approx. time | Notes |
|---|---|---|---|
| Hook | surface the painful pricing mistake fast | 0:00–1:00 | start with one vivid example |
| Setup | explain why this matters now | 1:00–3:00 | define audience |
| Lesson 1 | underpricing signal | 3:00–8:00 | include one short story |
| Lesson 2 | packaging vs discounting | 8:00–13:00 | contrast good/bad move |
| Lesson 3 | raise price without chaos | 13:00–18:00 | actionable sequence |
| Close | recap + CTA | 18:00–20:00 | invite listener response |

Then add:
- suggested intro line
- 2 optional transition lines
- where the host can ad-lib

## Example 2 — Full script

**Input:**
Write a full 8-minute educational script explaining passkeys to non-technical listeners.

**Good output shape:**
- cold open
- intro / framing
- sectioned script with spoken-friendly paragraphs
- one analogy that clarifies the concept
- brief recap
- outro CTA
- optional cut lines if runtime runs long

## Example 3 — Interview architecture

**Input:**
Create an interview outline for a startup CFO discussing burn and runway.

**Good output shape:**
| Segment | Purpose | Sample questions | What to avoid |
|---|---|---|---|
| Warm open | establish credibility and ease | “What changed your thinking about runway recently?” | resume recap |
| Diagnose | surface common mistakes | 3–4 specific questions | jargon pileup |
| Frameworks | extract reusable advice | 3 tactical prompts | vague platitudes |
| Closing | crystallize takeaway | “What should founders do this week?” | weak CTA |

## Example 4 — Intro, outro, and sponsor package

**Input:**
Write our weekly intro/outro plus a sponsor read for a calm, smart B2B show.

**Good output shape:**
- 3 intro options with different energy levels
- 2 sponsor-read variants matched to tone
- outro with recap + CTA
- notes on where the host should personalize wording
