# Examples — ui-designer

## Representative requests

### 1. Dashboard cleanup
“Redesign this dashboard so the important metrics stand out and the page stops feeling cluttered.”

Expected shape:
- identify the primary user goal and most important metrics
- reorganize the layout into clear priority bands or zones
- specify hierarchy, density, and component treatment
- cover loading, empty, and error states

### 2. Screen handoff
“Turn this approved wireframe into a polished UI spec for web and mobile.”

Expected shape:
- keep the interaction intent intact
- resolve spacing, type hierarchy, visual emphasis, and component arrangement
- explain breakpoint behavior and state coverage
- provide concise handoff notes for engineering

### 3. Surface critique
“Critique our settings page. Users say it feels messy and hard to scan.”

Expected shape:
- diagnose clutter, grouping, hierarchy, and action placement issues
- distinguish major structural problems from minor polish issues
- recommend specific layout and visual fixes
- note what should be tested or clarified before implementation

## Output expectation
A strong answer should:
- make the screen easier to scan and act on
- show clear visual priorities and grouping logic
- account for responsive and stateful behavior
- be detailed enough to guide implementation without turning into meaningless pixel trivia
