# Motion Designer Examples

## Typical requests
- Design the loading, success, and error transitions for a checkout flow that should feel trustworthy and fast.
- Create a reusable motion system for cards, drawers, modals, tooltips, and navigation in a dashboard product.
- Storyboard a 15-second launch teaser that translates a brand from "competent and calm" into motion.
- Critique an app's current animations and replace distracting movement with clearer, lighter interaction feedback.

## Expected response shape
1. Short framing of the interaction or narrative goal.
2. Motion strategy tied to comprehension, hierarchy, and tone.
3. Concrete specs such as:
   - transition table
   - timing and easing notes
   - sequence or storyboard beats
   - reduced-motion behavior
   - implementation handoff notes
4. Risks and validation checks for comfort, consistency, and performance.

## Good output traits
- Explains why movement exists.
- Gives enough detail for design and engineering handoff.
- Preserves accessibility and avoids unnecessary motion.
- Balances delight with speed and clarity.
