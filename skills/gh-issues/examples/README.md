# GitHub Issues — Example Output Shapes

Use these patterns for strong triage and backlog outputs.

## Example 1 — Bug triage

**Input:**
Several users report export failures in production.

**Good output shape:**
- issue type: bug
- concise problem summary
- current evidence and gaps
- severity and priority rationale
- rewritten title and body
- clarifying questions if repro is incomplete
- suggested labels and owner
- next action

## Example 2 — Feature request shaping

**Input:**
Add bulk archive to the dashboard.

**Good output shape:**
- underlying user job, not just requested button
- constraints and assumptions
- proposed issue framing
- acceptance criteria draft
- possible split between UX, API, and rollout work
- milestone or priority recommendation

## Example 3 — Duplicate cleanup

**Input:**
Review eight issues about login timeout problems.

**Good output shape:**
- duplicate clusters versus truly distinct failures
- recommended canonical issue
- link/close recommendations for duplicates
- risk of incorrect merging
- missing evidence that would separate clusters

## Example 4 — Intake-system improvement

**Input:**
Our issue backlog is full of vague bug reports.

**Good output shape:**
- common intake failures observed
- suggested issue-form fields and validations
- default labels or routing changes
- guidance for support-versus-engineering separation
- concrete cleanup steps for the current backlog
