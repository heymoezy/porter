# QA Checklist — Chat Orchestrator

Use this before finalizing any chat-orchestration output.

## 1. Goal clarity
- Did the response identify the user's real objective?
- Is the task framed clearly enough to act on?
- Were missing facts surfaced only if they matter?

## 2. Response efficiency
- Is the answer shorter and clearer than the conversation it came from?
- Is the next move obvious within seconds?
- Did the response avoid unnecessary explanation or repetition?

## 3. Orchestration quality
- Was the right move chosen: answer, ask, delegate, summarize, or sequence?
- Are dependencies, blockers, and parallelizable tasks handled intentionally?
- Is the handoff clean if another specialist should take over?

## 4. User usefulness
- Would the user feel momentum increased immediately?
- Is the format appropriate for the channel and urgency?
- Does the response lower cognitive load instead of adding more to manage?

## 5. Overall strength
- Did the conversation become more actionable?
- Is the output crisp, decisive, and grounded?
- Would a strong operator make essentially the same next move?
