# Patent Researcher — Example Output Shapes

Use these as response patterns, not rigid templates.

## Example 1 — Quick prior-art screen

**Input:**
Check for prior art around an AI system that codes invoices using prior ledger corrections and confidence-ranked suggestions.

**Good output shape:**
- objective and scope assumptions
- invention decomposed into 4–7 searchable features
- keyword and synonym buckets
- likely CPC / IPC classes to inspect
- 5–10 references with one-line why-it-matters notes
- feature comparison table
- top risk signals, blind spots, and next searches

## Example 2 — Feature-to-reference comparison

**Input:**
Compare our warehouse-picking concept against this patent family.

**Good output shape:**
| Feature | User concept | Reference evidence | Assessment |
|---|---|---|---|
| Vision-guided item localization | yes | disclosed in abstract and claim 1 | overlap likely |
| Dynamic grasp-strategy switching | yes | described in spec examples, not clear in independent claims | partial |
| On-the-fly shelf-map updating | yes | not found in reviewed docs | possible differentiator |
| Human override with learning loop | yes | partial disclosure across two related filings | combination risk |

Then add:
- family / continuation notes
- what to inspect next

## Example 3 — Assignee landscape

**Input:**
Who is filing patents in solid-state battery thermal management?

**Good output shape:**
- scope and date range
- top assignees and what each appears to focus on
- dominant CPC clusters
- filing trend by period
- notable family expansion / continuation behavior
- crowded areas vs tentative white-space hints

## Example 4 — Search strategy memo

**Input:**
Plan a stronger patent search for our sensor-fusion drone inspection method.

**Good output shape:**
- objective
- concept decomposition
- search routes by keyword, function, and class
- seed-reference strategy
- citation chaining plan
- jurisdictions to prioritize
- stop conditions and open questions
