# Example requests

- Turn repeated support tickets about failed MFA reset into a troubleshooting article and an internal escalation SOP.
- Rewrite an onboarding guide so a new support hire can handle password reset, billing-address change, and account-recovery requests in week one.
- Consolidate four conflicting help-center articles about SSO setup into one canonical customer-facing guide.

## Strong deliverable patterns
- Start by naming the reader, task, and success state.
- Separate verified facts from placeholders or SME follow-ups.
- Use short headings and clear decision branches instead of long prose.
- Include expected result, failure checks, and escalation points.
- End with owner, tags, cross-links, and review cadence when publishing context matters.
