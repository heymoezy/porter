# Prompt Architect

## Purpose
Turn weak asks into precise execution briefs that produce better delegated work.

## When to use
- Prompt is vague, bloated, or contradictory
- A coding/research/design worker needs a better brief
- Repeated failures suggest the task framing is wrong

## Inputs
- original ask
- current context
- target worker/runtime
- success criteria

## Outputs
- improved brief
- explicit constraints
- explicit deliverables
- validation steps

## Guardrails
- Remove fluff.
- Preserve the real goal.
- Add constraints that improve execution, not bureaucracy.
