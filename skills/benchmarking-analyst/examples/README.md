# Benchmarking Analyst — Example Output Shapes

Use these as patterns for strong benchmarking deliverables.

## Example 1 — Operational KPI benchmark

**Input:**
Benchmark our support team against industry standards for first-response time and resolution speed.

**Good output shape:**
- metric definitions and decision goal
- peer-set or industry-standard rationale
- normalized comparison table
- where the team is ahead, behind, or incomparable
- likely distortion factors: ticket mix, hours, staffing model
- target recommendations and next actions

## Example 2 — Internal historical benchmark

**Input:**
Compare this quarter’s deployment reliability against the last six quarters.

**Good output shape:**
- trend baseline and time-window logic
- normalized metrics
- whether current performance is materially different or within normal range
- possible seasonality or release-pattern effects
- benchmark-informed action priorities

## Example 3 — Vendor benchmark

**Input:**
Benchmark three logistics vendors on cost, SLA attainment, and issue rates.

**Good output shape:**
- evaluation criteria and weighting logic
- unit normalization approach
- side-by-side comparison table
- where vendor differences are meaningful vs noisy
- recommendation with tradeoffs
- data gaps that weaken confidence

## Example 4 — Invalid comparison critique

**Input:**
Review our benchmark deck and tell us whether these competitor comparisons are fair.

**Good output shape:**
- what is currently miscompared
- missing normalization factors
- which benchmark claims should be removed or softened
- corrected comparison options
- confidence level for each revised benchmark

## Example 5 — Target-setting memo

**Input:**
Set realistic benchmark targets for onboarding completion rates across regions.

**Good output shape:**
- target-setting approach
- benchmark ranges by comparable cohorts
- region-specific caveats
- ambitious vs realistic target bands
- interventions likely needed to reach higher bands
