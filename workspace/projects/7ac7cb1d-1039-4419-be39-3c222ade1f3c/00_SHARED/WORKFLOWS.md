# First Mission — Workflows

## How Work Gets Done
- Porter guides the operator through one starter objective
- a worker is created or approved only when the objective needs one
- durable state and at least one artifact are produced in the project lane

## Handoffs
- Porter remains the orchestrator
- workers execute scoped work only after approval or clear direction

## Success Loop
- approve one worker
- record one directive
- produce one artifact or deliverable
- graduate into a real project
