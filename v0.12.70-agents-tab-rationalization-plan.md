# Porter v0.12.70 Draft — Agents Tab Rationalization Plan

Date: 2026-02-26  
Status: Draft (queued; do not implement until current Claude run completes)  
Owner: Lobster + User review

---

## 1) Problem Statement

Current Agents tab mixes:
- production operator controls,
- internal "agent fleet lifecycle" controls,
- and leftover test agents.

This creates cognitive overload and weak trust in what is truly active.

User intent is clear:
- Primary view should show only real, actively used agents:
  - OpenClaw/Codex
  - Claude
  - Gemini
- Test/internal items should not pollute day-to-day controls.
- Agent identity + usage should be unified into one clean card per agent.

---

## 2) Product Goals

1. Make Agents tab immediately legible for daily operations.
2. Unify agent metadata and usage in one card per agent.
3. Hide internal/test complexity by default.
4. Keep advanced lifecycle controls available (but explicitly gated).

---

## 3) Scope for v0.12.70 (post-Claude)

### A) Primary Agents View (default)
Show only production agents in default list.

Each card includes:
- Agent name + provider/type
- Status (online/offline/unknown)
- Role
- Last seen
- Usage summary (today + recent burn/ops)
- Quick actions (only core-safe actions)

### B) Unified Card Layout
Merge currently separated "Agent info" and "Usage" sections into a single visual module per agent.

### C) Advanced/Internal Section
Move "Agent Fleet Lifecycle" and similar controls into a collapsed area:
- Label: `Advanced / Internal`
- Default: hidden
- Explicit reveal toggle: `Show Advanced`

### D) Test Agent Hygiene
Introduce test/internal classification behavior:
- Test-tagged agents hidden from default Primary view.
- Add management actions for test agents:
  - Archive
  - Delete
  - (Optional) Bulk cleanup action

### E) Display Filtering Rules
Default display filter:
- show only active production agents
- hide archived/test/internal unless Advanced is expanded

---

## 4) UX State Model

### State 1: Clean production setup
- User sees only the 3 primary agents (OpenClaw/Codex, Claude, Gemini)
- One compact card each

### State 2: Test agents exist in config
- Test agents are hidden in primary view
- Advanced section indicates hidden count (e.g., "3 hidden test agents")

### State 3: Advanced mode enabled
- Internal controls and test agents visible with clear badges:
  - `TEST`
  - `INTERNAL`
  - `ARCHIVED`

## 6) Technical Plan (implementation outline)

### Data shaping
At render layer, derive per-agent display fields:
- `is_test`
- `is_internal`
- `is_archived`
- `is_primary`

### Classification strategy (v0.12.70)
Classify as test/internal if any of:
- name contains "test"
- explicit role/tag indicates internal/testing
- manually marked by user action

### UI composition
- Build `renderAgentCard(agent)` to include both identity + usage widgets.
- Replace split sections with card grid/list.
- Add Advanced section render branch.

### Backward compatibility
- No destructive schema migration required in this tranche.
- Classification can be additive at UI level first.

---

## 7) Acceptance Criteria

1. Default Agents tab shows only production agents.
2. Test agents no longer appear in default main view.
3. Agent info + usage are unified in one neat card per agent.
4. Advanced/internal lifecycle controls are accessible but collapsed by default.
5. User can identify and clean up test agents quickly.
